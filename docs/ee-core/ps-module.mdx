# PS 模块

PS 模块提供环境检测、进程类型判断、路径获取和进程管理功能，是其他模块获取运行环境信息的基础依赖。

所有路径信息来自 `boot.ts` 写入 `process.env` 的环境变量。

## 环境检测

| 函数 | 说明 |
|------|------|
| `env()` | 当前环境名称（dev/local/prod） |
| `isDev()` | 是否开发环境（dev 或 local） |
| `isProd()` | 是否生产环境 |
| `isPackaged()` | 是否已打包 |
| `allEnv()` | 获取所有环境变量 |

```javascript
import { env, isDev, isProd, isPackaged } from 'ee-core/ps';

console.log(env());       // 'dev' 或 'prod'
console.log(isDev());     // true 或 false
console.log(isProd());    // true 或 false
console.log(isPackaged()); // true 或 false
```

## 进程类型判断

| 函数 | 说明 |
|------|------|
| `isMain()` | 是否主进程（process.type === 'browser'） |
| `isRenderer()` | 是否渲染进程（process.type === 'renderer'） |
| `isForkedChild()` | 是否 Node 子进程（ELECTRON_RUN_AS_NODE=1） |
| `isChildJob()` | 是否 ChildJob 类型子进程 |
| `isChildPoolJob()` | 是否 ChildPoolJob 类型子进程 |
| `processType()` | 获取进程类型字符串 |

```javascript
import { isMain, isRenderer, isForkedChild, processType } from 'ee-core/ps';

if (isMain()) {
  // 主进程逻辑
}

if (isRenderer()) {
  // 渲染进程逻辑
}

if (isForkedChild()) {
  // 子进程逻辑
}

console.log(processType()); // 'browser' | 'renderer' | 'child'
```

<Note>
`isForkedChild()` 检查 `process.env.ELECTRON_RUN_AS_NODE` 是否为 1，这是 `child_process.fork()` 创建子进程的标志。`isChildJob()` 和 `isChildPoolJob()` 通过检查 `process.argv[2]` 中 JSON 对象的 `type` 字段判断。
</Note>

## 路径获取

### 核心路径函数

| 函数 | 说明 | 开发环境路径 | 生产环境路径 |
|------|------|-------------|-------------|
| `getBaseDir()` | 项目根目录 | `process.env.EE_BASE_DIR` | 同 |
| `getElectronDir()` | Electron 源码目录 | `process.env.EE_ELECTRON_DIR` | 同 |
| `getExecDir()` | 可执行文件目录 | `process.env.EE_EXEC_DIR` | exe 所在目录 |
| `getRootDir()` | 运行数据根目录 | `{baseDir}` | `{appUserData}` |

### 数据与日志路径

| 函数 | 说明 | 开发环境 | 生产环境 |
|------|------|---------|---------|
| `getDataDir()` | 数据目录 | `{baseDir}/data` | `{userHome}/.{appName}/data` |
| `getLogDir()` | 日志目录 | `{baseDir}/logs` | `{userHome}/.{appName}/logs` |
| `getUserHomeHiddenAppDir()` | 隐藏应用目录 | — | `{userHome}/.{appName}/` |
| `getUserHomeAppDir()` | 应用目录 | — | `{userHome}/{appName}/` |

### 特殊路径函数

| 函数 | 说明 |
|------|------|
| `getBundleDir(basePath?)` | 打包输出目录 `{basePath}/public/electron` |
| `getElectronCodeDir(basePath?)` | Electron 源码目录 `{basePath}/electron` |
| `getFrontendCodeDir(basePath?)` | 前端源码目录 `{basePath}/frontend` |
| `getPublicDir()` | 公共资源目录 `{baseDir}/public` |
| `getAppUserDataDir()` | Electron 用户数据目录 |
| `getUserHomeDir()` | 用户主目录 |

### extraResources 路径

`getExtraResourcesDir()` 根据平台和打包状态返回不同路径：

| 环境 | Windows/Linux | macOS |
|------|-------------|-------|
| 打包 | `{execDir}/resources/extraResources` | `{execDir}/../Resources/extraResources` |
| 未打包 | `{execDir}/build/extraResources` | 同 |

<Note>
macOS 打包后的路径结构为 `Contents/MacOS/{exe}` 和 `Contents/Resources/extraResources`，所以需要 `..` 回溯一级。
</Note>

### 端口获取

| 函数 | 说明 |
|------|------|
| `getSocketPort()` | Socket 服务端口 |
| `getHttpPort()` | HTTP 服务端口 |

```javascript
import { getSocketPort, getHttpPort } from 'ee-core/ps';

console.log(getSocketPort());  // 7070 或 0
console.log(getHttpPort());    // 7071 或 0
```

## 进程管理

### exit() — 退出进程

```javascript
import { exit } from 'ee-core/ps';
exit(0);  // 正常退出
exit(1);  // 异常退出
```

### makeMessage() — 格式化 IPC 消息

```javascript
import { makeMessage } from 'ee-core/ps';

const msg = makeMessage({
  channel: 'controller/user/add',
  event: 'jobComplete',
  data: { result: true },
});
// { channel: 'controller/user/add', event: 'jobComplete', data: { result: true } }
```

### exitChildJob() — 退出 ChildJob 子进程

```javascript
import { exitChildJob } from 'ee-core/ps';

// 在子进程代码中调用
exitChildJob(0);  // 仅 ChildJob 类型进程会退出
```

<Note>
`exitChildJob()` 检查 `process.argv[2]` 中的 `type` 字段，只有 `type === 'childJob'` 的子进程才会退出。其他类型的子进程不受影响。
</Note>

## getArgumentByName() — 命令行参数获取

```javascript
import { getArgumentByName } from 'ee-core/ps';

// 从 process.argv 中获取命名参数（--name=value 格式）
const envValue = getArgumentByName('env');     // 'prod' 或 'dev'
const debuggerValue = getArgumentByName('debugger'); // 'true' 或 undefined
```

## process.env 环境变量映射

`boot.ts` 构造函数中写入的完整环境变量列表：

| 变量 | 说明 |
|------|------|
| `EE_ENV` | 运行环境 |
| `EE_APP_NAME` | 应用名称 |
| `EE_APP_VERSION` | 应用版本 |
| `EE_BASE_DIR` | 项目根目录 |
| `EE_ELECTRON_DIR` | Electron 目录 |
| `EE_USER_HOME` | 用户主目录 |
| `EE_APP_DATA` | 系统应用数据目录 |
| `EE_APP_USER_DATA` | Electron 用户数据目录 |
| `EE_EXEC_DIR` | 可执行文件目录 |
| `EE_IS_PACKAGED` | 是否已打包 |
| `EE_SOCKET_PORT` | Socket 端口（启动后写入） |
| `EE_HTTP_PORT` | HTTP 端口（启动后写入） |
