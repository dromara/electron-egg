# Controller 模块

控制器是业务逻辑的核心，每个前端 IPC 请求最终都会路由到对应的控制器方法。Controller 模块负责扫描、加载和包装所有控制器文件。

## ControllerLoader 加载器

### 两种加载方式

| 方法 | 加载方式 | 适用场景 |
|------|---------|---------|
| `load()` | 同步 `require()` | CJS 项目 |
| `loadAsync()` | 异步 `import()` | ESM 项目 |

<Warning>
同步和异步加载不能并发执行。如果异步加载正在进行，调用同步方法会抛出错误。异步加载完成后才可使用 `getControllers()`。
</Warning>

### 两种扫描模式

| 模式 | 条件 | 扫描方式 |
|------|------|---------|
| Bundle 模式 | `globalThis.__EE_CONTROLLER_REGISTRY__` 存在 | 从预注册表读取模块 |
| Dev 模式 | 注册表不存在 | globby 文件系统扫描 |

### 加载流程

1. 检查 `__EE_CONTROLLER_REGISTRY__` 是否存在
2. 扫描 `electron/controller/` 目录下的所有文件
3. 对导出的类执行 `wrapClass()` 包装
4. 对导出的函数直接注册
5. 返回控制器方法映射对象

## wrapClass 类包装

`wrapClass()` 遍历类原型链，将每个方法转换为中间件函数：

- 遍历原型链收集所有层级的方法
- `constructor` 被跳过
- getter/setter 和非函数属性被跳过
- 子类方法不会被父类同名方法覆盖（`hasOwnProperty` 保护）

## methodToMiddleware 并发安全设计

每个控制器方法被包装为中间件函数，**每次调用都会创建新的控制器实例**：

```javascript
function methodToMiddleware(Controller, key) {
  return function classControllerMiddleware(...args) {
    const controller = new Controller();  // 新实例
    const fn = controller[key];
    return fn.call(controller, args, event);
  };
}
```

<Note>
这种设计确保了并发安全 — 当多个请求同时到达时，每个请求拥有独立的控制器实例，避免共享状态导致的竞态条件。
</Note>

## 命名规则 (caseStyle: lower)

控制器文件路径到属性路径的转换使用 `caseStyle: 'lower'`（首字母小写）：

| 文件路径 | 属性路径 |
|---------|---------|
| `controller/user.js` | `controller.user` |
| `controller/admin/login.js` | `controller.admin.login` |
| `controller/user_info.js` | `controller.userInfo` |
| `controller/UserController.js` | `controller.userController` |

<Warning>
文件名只允许包含 `a-z0-9_-` 字符，不合法的命名会在加载时抛出错误。
</Warning>

## IPC 自动注册

`IpcServer` 自动遍历控制器对象树，为每个导出方法注册 IPC 通道：

- 通道命名规则：将属性路径中的 `.` 替换为 `channelSeparator`（默认 `/`）
- 例如 `controller.user.add` → `controller/user/add`
- 每个方法注册两种 IPC 处理器：`ipcMain.on`（同步模型）和 `ipcMain.handle`（异步模型）

## 控制器编写示例

<CodeGroup>

```javascript [类风格 (推荐)]
// electron/controller/user.js
class UserController {
  async add(args) {
    // args: 渲染进程传来的参数
    const result = { id: 1, name: args.name };
    return result;
  }

  async delete(args) {
    // 返回值自动通过 IPC 回传给渲染进程
    return { success: true };
  }
}

module.exports = UserController;
```

```javascript [函数风格]
// electron/controller/user.js
module.exports = {
  add: async (args) => {
    return { id: 1, name: args.name };
  },
  delete: async (args) => {
    return { success: true };
  },
};
```

</CodeGroup>

## 导出结构

加载完成后，控制器映射对象的结构如下：

```javascript
{
  controller: {
    user: {
      add: [Function: classControllerMiddleware],
      delete: [Function: classControllerMiddleware],
    },
    admin: {
      login: {
        auth: [Function: classControllerMiddleware],
      },
    },
  },
}
```

## API

### loadController() / loadControllerAsync()

```javascript
import { loadController, loadControllerAsync } from 'ee-core/controller';

// CJS 项目
const controllers = loadController();

// ESM 项目
const controllers = await loadControllerAsync();
```

### getControllers() / getController()

获取已加载的控制器映射。如果尚未加载，会自动触发同步加载。

```javascript
import { getControllers } from 'ee-core/controller';
const controllers = getControllers();
```

<Warning>
在异步加载进行中调用 `getControllers()` 会抛出错误，需要等待异步加载完成后才能访问。
</Warning>
