# Electron 模块

Electron 模块负责应用生命周期管理、窗口创建和页面内容加载，是框架与 Electron API 的核心桥梁。

## createElectron() — 创建 Electron 应用

`createElectron()` 是框架启动流程的最后一个步骤，由 `Application.run()` 调用。

### 执行流程

```
1. 检查单实例锁 (singleLock)
2. app.whenReady() 等待
3. createMainWindow() 创建主窗口
4. emitLifecycle(Preload) 触发 preload 事件
5. loadServer() 加载页面内容
6. emitLifecycle(ElectronAppReady) 触发就绪事件
```

### 单实例锁

当 `singleLock: true` 时，只允许运行一个应用实例：

```javascript
const gotTheLock = electronApp.requestSingleInstanceLock();
if (singleLock && !gotTheLock) {
  electronApp.quit();  // 第二个实例直接退出
  return;
}
```

第二个实例启动时，已有实例的窗口会被恢复（通过 `restoreMainWindow`）。

### 系统事件监听

| 事件 | 行为 |
|------|------|
| `window-all-closed` | 非 macOS 平台退出应用 |
| `before-quit` | 清理跨进程服务、终止所有子任务 |

```javascript
electronApp.on('before-quit', () => {
  setCloseAndQuit(true);
  eventBus.emitLifecycle(BeforeClose);
  cross.killAll();        // 终止 Go/Python 等后端服务
  killAllJobs();          // 终止所有子任务进程
});
```

## loadServer() — 四种页面加载策略

`loadServer()` 根据配置和环境自动选择页面加载方式：

### 远程模式 (remote.enable = true)

直接加载远程 URL：

```javascript
// config.default.js
config.remote = {
  enable: true,
  url: 'ws://remote-host:port',
};
```

### 开发模式 (isDev)

加载前端开发服务器（默认 `http://localhost:8080`），等待服务就绪：

1. 先显示启动页面（loadingPage）避免白屏
2. 通过 HTTP GET 轮询检查前端服务是否就绪（最多 60 次，间隔 1 秒）
3. 服务就绪后加载前端页面
4. 服务未就绪时显示失败页面

<Note>
开发环境的轮询参数从 `cmd/bin.js` 的 `dev.frontend` 配置中读取。`force: true` 时只重试 3 次，适用于前端服务不依赖 dev server 的场景。
</Note>

### 跨进程接管模式 (mainServer.takeover)

生产环境中，等待跨进程服务（Go/Python 后端）就绪后加载其提供的 URL：

```javascript
// config.prod.js
config.mainServer = {
  takeover: 'goServer',  // 对应 config.cross.goServer.name
};
```

流程：
1. 显示 loadingPage
2. 检查跨进程服务配置是否正确
3. 通过 `cross.getUrl()` 获取服务 URL
4. 轮询检查服务就绪（最多 100 次，间隔 200ms）
5. 就绪后加载服务 URL，未就绪显示跨进程失败页面

### 生产文件模式 (默认)

加载本地打包的 HTML 文件：

```javascript
config.mainServer = {
  protocol: 'file://',
  indexPath: '/public/dist/index.html',
};
```

## windowsOption 配置

```javascript
config.windowsOption = {
  title: 'electron-egg',
  width: 980,
  height: 650,
  minWidth: 400,
  minHeight: 300,
  webPreferences: {
    contextIsolation: false,
    nodeIntegration: true,
  },
  frame: true,
  show: false,       // false: 先加载内容再显示，避免白屏
  icon: path.join(baseDir, 'public/images/logo-32.png'),
};
```

<Warning>
`show: false` 配合窗口创建后手动调用 `win.show()` 是推荐做法，可以避免窗口创建时的白屏闪烁。ee-core 在 `WindowReady` 事件后自动显示窗口。
</Warning>

## 窗口管理 API

### getMainWindow()

获取主窗口实例：

```javascript
import { getMainWindow } from 'ee-core/electron';
const win = getMainWindow();
if (win) {
  win.webContents.send('channel', data);
}
```

### createMainWindow()

创建主窗口，使用 `config.windowsOption` 配置。如果 `openDevTools` 为 `true` 或对象，自动打开开发者工具。

### restoreMainWindow()

恢复主窗口（从最小化状态恢复、显示并聚焦）。当第二个实例启动时，已有实例调用此方法激活窗口：

```javascript
electronApp.on('second-instance', () => {
  restoreMainWindow();
});
```

### setCloseAndQuit() / getCloseAndQuit()

控制窗口关闭后是否退出应用的标志：

```javascript
import { setCloseAndQuit, getCloseAndQuit } from 'ee-core/electron';
setCloseAndQuit(true);   // 窗口关闭后退出应用
console.log(getCloseAndQuit()); // true
```

## loadElectron() — 入口函数

```javascript
import { loadElectron } from 'ee-core/electron';
// 在 Application.run() 流程中自动调用，无需手动调用
loadElectron();  // 内部调用 createElectron()
```

## 导出 API

```javascript
import { electronApp, createElectron, getMainWindow, createMainWindow, restoreMainWindow, loadServer, setCloseAndQuit, getCloseAndQuit } from 'ee-core/electron';
```
