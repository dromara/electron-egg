# Log 模块

Log 模块基于 Pino 日志库，提供三个独立的日志实例和 Proxy 包装的多风格调用接口。

## 三个日志实例

| 实例 | 文件名 | 用途 | 默认级别 |
|------|--------|------|---------|
| `logger` | `ee.log` | 业务代码日志 | 配置的 `level` |
| `coreLogger` | `ee-core.log` | 框架内部日志 | 配置的 `level` |
| `errorLogger` | `ee-error.log` | 仅记录错误 | `error`（固定） |

```javascript
import { logger, coreLogger, errorLogger } from 'ee-core/log';

logger.info('业务操作完成');
coreLogger.info('[controller] loaded');
errorLogger.error(new Error('严重错误'));
```

## EeLogger Proxy — 三种调用风格

通过 Proxy 拦截方法调用，兼容多种日志书写习惯：

### 1. Pino 标准合并模式

第一个参数为对象（合并对象），第二个参数为消息字符串：

```javascript
logger.info({ userId: 123, action: 'login' }, '用户登录');
```

### 2. Pino printf 模式

包含 `%s`、`%d` 等格式占位符：

```javascript
logger.info('用户 %s 登录，耗时 %d ms', '张三', 150);
```

### 3. 拼接模式

多个参数自动拼接为字符串：

```javascript
logger.info('用户登录:', userId, '耗时:', elapsed);
// 等价于: logger.info('用户登录: 123 耗时: 150')
```

<Note>
Proxy 自动判断调用风格：第一个参数是普通对象则走标准模式，包含 `%s/%d` 则走 printf 模式，其余走拼接模式。`Error` 对象会被特殊处理，输出 stack 信息。
</Note>

### child() 子日志

```javascript
const childLogger = logger.child({ module: 'userController' });
childLogger.info('用户操作');  // 自动附带 module 字段
```

## PinoLoggers — 底层实现

三个日志实例基于 Pino 的 `transport` 多目标输出机制：

### pino-roll 文件轮转

每个日志文件使用 `pino-roll` 实现自动轮转：

| 配置项 | 说明 |
|--------|------|
| `frequency` | 轮转频率：`daily`（每天）或 `hourly`（每小时） |
| `dateFormat` | 文件名日期格式，如 `yyyy-MM-dd` |
| `size` | 单文件最大尺寸，如 `10m`（10MB） |

### pino-pretty 开发美化

开发环境自动启用 `pino-pretty` 终端彩色输出：

```javascript
// 默认行为：开发环境自动启用
config.logger = {
  prettyPrint: true,  // 或 false 禁用
};
```

输出效果：

```
2024-01-15 10:30:00 INFO  [ee] 用户登录成功
```

## 配置参考

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `dir` | `string` | `getLogDir()` | 日志文件存储目录 |
| `level` | `string` | `'info'` | 日志级别：trace/debug/info/warn/error/fatal |
| `prettyPrint` | `boolean` | `isDev()` | 开发环境彩色输出 |
| `dateFormat` | `string` | `'yyyy-MM-dd'` | 文件名日期格式 |
| `appLogName` | `string` | `'ee.log'` | 应用日志文件名 |
| `coreLogName` | `string` | `'ee-core.log'` | 核心日志文件名 |
| `errorLogName` | `string` | `'ee-error.log'` | 错误日志文件名 |
| `rotator` | `string` | `'day'` | 轮转策略：day/daily/hour/hourly |
| `redact` | `string[]` | `[]` | 需要脱敏的字段路径列表 |
| `redactCensor` | `string` | `'[Redacted]'` | 脱敏替换值 |
| `timestamp` | `boolean` | `true` | 是否包含时间戳 |
| `name` | `string` | `'ee'` | 日志器名称 |
| `maxSize` | `string` | `'10m'` | 单文件最大尺寸 |
| `serializers` | `object` | `{}` | 自定义序列化器 |
| `customLevels` | `object` | `{}` | 自定义日志级别 |
| `depthLimit` | `number` | `5` | 对象序列化深度限制 |
| `safe` | `boolean` | `true` | 安全模式：日志写入失败不抛异常 |
| `enabled` | `boolean` | `true` | 是否启用日志 |

### 脱敏配置示例

```javascript
config.logger = {
  redact: ['password', 'token', '*.secret'],
  redactCensor: '[REDACTED]',
};
```

<Note>
`redact` 支持 Pino 的路径语法，如 `['password']` 脱敏顶层字段，`['*.secret']` 脱敏所有嵌套的 secret 字段。
</Note>

## createLog() — 自定义日志

`createLog()` 可创建独立的日志实例，不依赖框架系统配置：

```javascript
import { createLog } from 'ee-core/log';

// 使用自定义配置创建日志
const customLoggers = createLog({
  dir: '/custom/log/path',
  level: 'debug',
  appLogName: 'custom.log',
  coreLogName: 'custom-core.log',
  errorLogName: 'custom-error.log',
});
```

<Note>
`createLog()` 返回原始的 `PinoLoggers` 对象（非 Proxy 包装），如果需要 Proxy 风格调用，需自行包装。
</Note>

## loadLog() — 系统日志加载

框架启动时由 `boot.ts` 的 `init()` 调用，使用系统配置创建日志实例。

```javascript
import { loadLog } from 'ee-core/log';
loadLog();  // 自动使用 getConfig() 的 logger 配置
```

## EeLogger 接口

```typescript
interface EeLogger {
  trace(...args: unknown[]): void;
  debug(...args: unknown[]): void;
  info(...args: unknown[]): void;
  warn(...args: unknown[]): void;
  error(...args: unknown[]): void;
  fatal(...args: unknown[]): void;
  child(bindings: pino.Bindings): EeLogger;
}
```
