# Utils 模块

Utils 模块是框架的基础工具集，提供对象合并、环境检测、JSON 操作、CLI 参数解析、路径转换、端口分配等通用功能。

## extend — 深度合并

支持浅合并和深度递归合并，用于配置层叠合并和其他对象合并场景。

### 合并规则

- `deep=true`：纯对象递归合并，非纯对象直接覆盖
- 自动跳过 `__proto__` 属性，防止原型链污染
- 自动跳过 `undefined` 值，不会覆盖已有属性
- 检测自引用（`target === val`），防止无限递归

```javascript
import { extend } from 'ee-core/utils';

// 浅合并
extend(false, { a: 1 }, { a: 2, b: 3 });
// { a: 2, b: 3 }

// 深度合并
extend(true, { a: { x: 1 } }, { a: { y: 2 } });
// { a: { x: 1, y: 2 } }
```

## is — Electron 环境检测

检测当前进程类型和操作系统平台：

| 函数 | 说明 |
|------|------|
| `renderer()` | 是否渲染进程 |
| `main()` | 是否主进程（browser） |
| `osx()` / `macOS()` | 是否 macOS |
| `windows()` | 是否 Windows |
| `linux()` | 是否 Linux |
| `x86()` | 是否 32 位架构 |
| `x64()` | 是否 64 位架构 |
| `sandbox()` | 是否 macOS 沙盒环境 |
| `mas()` | 是否 Mac App Store 构建 |
| `windowsStore()` | 是否 Windows Store 构建 |

组合判断：

```javascript
import * as is from 'ee-core/utils';

// 主进程 + macOS
if (is.all(is.main, is.osx)) {
  // macOS 主进程专属逻辑
}

// 不是 Windows 也不是 Linux
if (is.none(is.windows, is.linux)) {
  // 仅 macOS 环境
}

// 至少一个条件为真
if (is.one(is.main, is.renderer)) {
  // Electron 进程
}
```

## json — JSON 文件读写

提供同步和异步的 JSON 文件读写操作，自动处理目录创建和路径检查：

```javascript
import { readSync, writeSync, read, write, strictParse } from 'ee-core/utils';

// 同步读取
const config = readSync('/path/to/config.json');

// 同步写入（自动创建目录，默认 2 空格缩进）
writeSync('/path/to/data.json', { key: 'value' });

// 异步读取
const data = await read('/path/to/data.json');

// 异步写入
await write('/path/to/output.json', { result: true });

// 严格解析（确保结果是对象类型）
const obj = strictParse('{"key": "value"}');
```

| 函数 | 异步 | 自动创建目录 | 说明 |
|------|------|-------------|------|
| `readSync()` | 否 | — | 同步读取 JSON |
| `writeSync()` | 否 | 是 | 同步写入 JSON |
| `read()` | 是 | — | 异步读取 JSON |
| `write()` | 是 | 是 | 异步写入 JSON |
| `strictParse()` | 否 | — | 严格解析（仅接受对象结果） |

## pargv — CLI 参数解析

轻量级命令行参数解析器，支持短选项、长选项、别名、布尔标志等：

```javascript
import { parseArgv } from 'ee-core/utils';

const result = parseArgv(
  ['--name=foo', '-p', '8080', '--verbose', 'file.txt'],
  {
    boolean: ['verbose'],
    alias: { p: 'port' },
    string: ['name'],
  }
);
// { _: ['file.txt'], name: 'foo', port: 8080, verbose: true }
```

<Note>
`parseArgv` 自动跳过 `__proto__` 和 `constructor` 属性，防止原型链污染。支持嵌套属性（点号分隔）和默认值填充。
</Note>

## wrap — 路径转换

将文件路径转换为属性路径数组，用于 FileLoader 构建嵌套对象结构：

```javascript
import { getProperties, defaultCamelize } from 'ee-core/utils';

// caseStyle: lower（控制器使用）
getProperties('user_info.js', { caseStyle: 'lower' });
// ['userInfo']

// caseStyle: upper
getProperties('UserInfo.js', { caseStyle: 'upper' });
// ['UserInfo']

// caseStyle: camel（默认）
getProperties('foo_bar.js', { caseStyle: 'camel' });
// ['fooBar']

// 自定义函数
getProperties('a/b/c.js', {
  caseStyle: (filepath) => filepath.split('/'),
});
```

## helper — 常用工具

| 函数 | 说明 |
|------|------|
| `fnDebounce()` | 防抖函数工厂 |
| `sleep(ms)` | 异步休眠 |
| `systemSleep(ms)` | 同步阻塞休眠（零 CPU） |
| `mkdir(filepath, opt)` | 递归创建目录 |
| `chmodPath(filepath, mode)` | 递归修改权限 |
| `compareVersion(v1, v2)` | 语义版本比较 |
| `getRandomString()` | 生成随机字符串 |
| `stringify(obj, ignore)` | 序列化对象（排除字段） |
| `validValue(value)` | 检查值是否有效 |
| `replaceArgsValue(argv, key, value)` | 替换命令行参数值 |
| `getValueFromArgv(argv, key)` | 获取命令行参数值 |
| `fileIsExist(filepath)` | 检查文件是否存在 |
| `checkConfig(prop)` | 检查配置文件是否存在 |

```javascript
import { fnDebounce, sleep, systemSleep, compareVersion, mkdir } from 'ee-core/utils';

// 防抖
const debounce = fnDebounce();
debounce(handleResize, 300, false, eventData);

// 异步休眠
await sleep(1000);

// 同步休眠（零 CPU，使用 Atomics.wait）
systemSleep(1000);

// 版本比较
compareVersion('1.2.3', '1.3.0');  // -1（v1 < v2）

// 创建目录
mkdir('/path/to/dir', { mode: 0o755 });
```

<Warning>
`systemSleep()` 使用 Atomics.wait 实现真正的线程阻塞，零 CPU 消耗。适用于子进程初始化等待等同步场景，不可在渲染进程使用（会冻结 UI）。
</Warning>

## port — 端口分配

`getPort()` 查找可用 TCP 端口，内置两级锁机制防止并发冲突：

```javascript
import { getPort, releasePortLocks } from 'ee-core/utils';

// 系统自动分配随机端口
const port = await getPort();

// 尝试指定端口，不可用时自动查找下一个
const port = await getPort({ port: 3000 });

// 尝试多个候选端口
const port = await getPort({ port: [3000, 3001, 3002] });

// 应用退出时释放端口锁
releasePortLocks();
```

<Note>
端口锁使用 old/young 两级 Set，每 15 秒轮转一次（young → old，old 清空释放）。确保端口在分配后至少锁定 15 秒，防止短时间重复分配。
</Note>

## ip — 公网 IP 查询

获取机器的公网 IPv4/IPv6 地址，支持三种查询方式：

```javascript
import { publicIpv4, publicIpv6 } from 'ee-core/utils';

// HTTPS 查询（默认）
const ip = await publicIpv4();

// DNS 查询
const ip = await publicIpv4({ type: 'dns' });

// 组合查询（DNS 优先，HTTPS 兜底）
const ip = await publicIpv4({ type: 'all', timeout: 5000 });

// IPv6
const ipv6 = await publicIpv6();

// 取消查询
const promise = publicIpv4();
promise.cancel();
```

## type_check — 类型检测

运行时值类型检测，支持 TypeScript 类型收窄：

| 函数 | 说明 |
|------|------|
| `isFunction(val)` | 是否函数 |
| `isClass(val)` | 是否 ES6 class |
| `isString(val)` | 是否字符串 |
| `isObject(val)` | 是否纯对象 |
| `isArray(val)` | 是否数组 |
| `isPrimitive(val)` | 是否原始类型 |
| `isGeneratorFunction(val)` | 是否生成器函数 |
| `isAsyncFunction(val)` | 是否异步函数 |

<Note>
`isClass()` 通过检查函数的 `toString()` 输出是否以 `class` 关键字开头来判断。`function` 声明的构造函数不会被识别为类。
</Note>

## Root Exports — 根级导出

从 `ee-core/utils` 直接导出的额外功能：

| 函数 | 说明 |
|------|------|
| `getPackage()` | 获取项目 package.json 内容 |
| `getMAC(iface?)` | 获取 MAC 地址 |
| `isMAC(macAddress)` | 检查是否 MAC 地址格式 |
| `isFileProtocol(protocol)` | 是否 file:// 协议 |
| `isWebProtocol(protocol)` | 是否 http/https 协议 |
| `isJsProject(baseDir)` | 是否 JS 项目（非 TS） |
| `machineIdSync(original?)` | 同步获取机器唯一标识 |
| `machineId(original?)` | 异步获取机器唯一标识 |
| `getPlatform(delimiter, isDiffArch)` | 获取平台标识字符串 |

```javascript
import { getPackage, machineIdSync, getPlatform } from 'ee-core/utils';

const pkg = getPackage();                // package.json 对象
const id = machineIdSync();              // SHA-256 哈希的机器 ID
const id = machineIdSync(true);          // 原始机器 ID
const platform = getPlatform();          // 'macos_apple' 或 'windows'
```
