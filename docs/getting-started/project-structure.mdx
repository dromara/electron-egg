## 目录总览

```
ee-dev/
├── packages/                # pnpm workspace 包
│   ├── ee-core/             # 运行框架（TypeScript，双 CJS/ESM）
│   │   └── src/
│   │       ├── app/         # boot, application, events, dir
│   │       ├── config/      # 配置加载/合并
│   │       ├── controller/  # 控制器加载器（registry + globby）
│   │       ├── core/        # FileLoader, utils
│   │       ├── cross/       # IPC 通信
│   │       ├── electron/    # Electron 应用/窗口管理
│   │       ├── jobs/        # ChildJob, ChildPoolJob, LoadBalancer
│   │       ├── loader/      # loadFile, requireFile, resolveModule
│   │       ├── log/         # Pino 日志（pino-roll + pino-pretty）
│   │       ├── socket/      # socketServer, httpServer, ipcServer
│   │       ├── storage/     # SqliteStorage（better-sqlite3）
│   │       ├── types/       # TypeScript 类型定义
│   │       └── utils/       # extend, is, json, wrap, helper
│   └── ee-bin/              # 构建工具（TypeScript，双 CJS/ESM）
│       └── src/
│           ├── config/      # bin_default.ts
│           ├── plugins/     # controller_registry_plugin.ts（esbuild）
│           ├── tools/       # serve, encrypt, move, iconGen, incrUpdater
│           └── types/       # BinConfig, BundleConfig 等
├── electron/                # 主进程业务代码
│   ├── main.js              # 入口文件
│   ├── config/              # 配置文件
│   ├── controller/          # 业务控制器
│   ├── service/             # 业务服务层
│   ├── preload/             # 预加载脚本
│   └── jobs/                # 子进程任务
├── frontend/                # 前端代码（Vue 3 + Ant Design）
├── public/                  # 静态资源（构建产物输出目录）
├── cmd/                     # 构建配置
├── go/                      # Go 后端（可选）
├── python/                  # Python 后端（可选）
├── build/                   # electron-builder 配置与脚本
└── .npmrc                   # pnpm 配置
└```

## electron/ 目录详解

`electron/` 是**主进程业务代码**目录，开发时直接修改此目录下的文件。

| 子目录/文件 | 用途 | 说明 |
|------------|------|------|
| `main.js` | 入口文件 | 应用启动入口，加载 ee-core 并启动生命周期 |
| `config/` | 配置文件 | 分层配置：`config.default.js`（默认）→ `config.local.js`（开发覆盖）→ `config.prod.js`（生产覆盖） |
| `controller/` | 控制器 | IPC 通道的处理单元，每个控制器类对应 `controller/{name}/` 通道前缀 |
| `service/` | 服务层 | 业务逻辑封装，控制器调用 service 处理复杂逻辑 |
| `preload/` | 预加载脚本 | `bridge.js`（IPC bridge）、`lifecycle.js`（生命周期钩子），通过 BrowserWindow preload 注入渲染进程 |
| `jobs/` | 子进程任务 | `child_process.fork()` 的入口文件，打包时作为独立文件复制（不进入 bundle） |

<Note>
控制器字符串标识至关重要。`ExampleController.toString = () => 'controller/example/'` 定义了 IPC 通道前缀，方法名作为通道后缀，最终通道为 `controller/example/{method}`。
</Note>

## frontend/ 目录

内置示例前端，基于 **Vue 3 + Ant Design + Vite**：

```
frontend/
├── src/
│   ├── views/          # 页面组件
│   ├── components/     # 通用组件
│   ├── router/         # 路由配置
│   └── main.js         # 前端入口
├── package.json
└── vite.config.js
```

<Warning>
前端框架**可替换**。ee-core 不绑定 Vue，你可以使用 React、Svelte 或纯 HTML 作为前端，只需确保渲染进程通过 preload bridge 与主进程通信。
</Warning>

## public/ 目录

`public/` 是静态资源目录，构建产物也输出到此处：

```
public/
├── electron/           # 构建产物输出目录
│   ├── main.js         # bundled 主进程
│   ├── jobs/           # 子进程任务文件（独立复制）
│   └── preload/
│       └── bridge.js   # 预加载脚本（独立复制）
├── html/               # HTML 静态页面
├── images/             # 图片资源
└── ssl/                # SSL 证书文件
```

<Note>
`public/electron/` 是 `pnpm build-electron` 的输出目录，开发时不应手动修改此目录内容。
</Note>

## cmd/ 目录

`cmd/` 包含构建配置文件：

| 文件 | 用途 |
|------|------|
| `bin.js` | ee-bin 项目配置（`build.electron` 部分） |
| `builder*.json` | electron-builder 平台打包配置 |

## packages/ 目录

pnpm workspace 管理的两个核心包：

### ee-core（运行框架）

ee-core 是运行时框架，**不会被 bundle 打包进 main.js**（作为 esbuild external）。运行时从 `node_modules/ee-core/` 加载，这确保了 `child_process.fork()` 能在磁盘上找到子进程入口 `app.js`。

### ee-bin（构建工具）

ee-bin 是 CLI 工具，仅用于构建阶段，**不参与 Electron 应用运行时**。核心功能：

| 命令 | 用途 |
|------|------|
| `dev` | 启动开发模式 |
| `build` | 执行 bundle 打包 |
| `start` | 生产模式启动 |
| `exec` | 执行指定模块 |
| `move` | 移动前端构建产物 |
| `encrypt` | 代码加密 |
| `icon` | 生成应用图标 |
| `updater` | 增量更新 |

## go/ 与 python/ 目录（可选）

这两个目录提供可选的后端服务支持：

- `go/` — Go 后端，基于 gin 框架
- `python/` — Python 后端，基于 FastAPI

后端服务通过 HTTP 或 Socket.IO 与 Electron 主进程通信，适合计算密集型任务或 AI 推理场景。

## 关键文件详解

### electron/main.js

应用入口文件，核心逻辑：

```javascript
const { app } = require('electron');
const EeApp = require('ee-core/app');

class MainApp {
  async ready() {
    // 应用启动回调
  }

  async windowReady() {
    // 窗口创建回调
  }

  async beforeClose() {
    // 关闭前回调
  }
}

EeApp.run(MainApp);
```

### cmd/bin.js

ee-bin 项目构建配置，`build.electron` 部分控制 esbuild 打包行为：

```javascript
module.exports = {
  build: {
    electron: {
      bundleType: 'bundle',   // 'bundle' | 'copy'
      external: [],           // 用户自定义 external
      sourcemap: false,       // false=自动, 'inline', 'external'
      minify: false,          // 生产打包时设为 true
      drop: undefined,        // ['console', 'debugger'] 移除语句
      keepNames: false,       // minify 时保留函数名
      copy: undefined,        // 额外复制文件 ['assets', 'data/db.json']
      format: undefined,      // 'cjs'(推荐) | 'esm'
    }
  }
};
```

框架管理的 externals（无需手动添加，`packages: 'external'` 已覆盖大部分）：`ee-core`, `ee-bin`, `electron`, `better-sqlite3`, `proxy-agent`, `pino-roll`, `pino-pretty`。

### electron/config/config.default.js

默认配置，所有环境的基础配置：

```javascript
module.exports = (appInfo) => {
  const config = {};
  config.devMode = true;
  config.logger = { level: 'INFO' };
  config.httpServer = { enable: true, port: 7070 };
  config.socketServer = { enable: true, port: 7071 };
  return config;
};
```

配置函数接收 `appInfo` 参数，可基于应用信息动态生成配置值。
