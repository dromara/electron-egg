## 启动开发模式

<Steps>
<Step title="启动前端开发服务器">

```bash
pnpm dev-frontend
```

Vite 启动后会在 `http://localhost:5173` 提供前端开发服务器，支持热更新。
</Step>

<Step title="启动 Electron 主进程">

```bash
pnpm dev-electron
```

Electron 窗口随即打开并加载前端页面。
</Step>

<Step title="一键启动（推荐）">

```bash
pnpm dev
```

同时启动前端开发服务器与 Electron 主进程，这是最常用的开发命令。
</Step>
</Steps>

<Note>
开发模式下前端与 Electron 运行在两个独立进程中：Vite dev server 负责前端热更新，Electron 负责主进程逻辑。修改前端代码会触发 HMR，修改 `electron/` 目录下的主进程代码需要重启 Electron。
</Note>

## 体验 IPC 通信

electron-egg 的 IPC 通道遵循 `controller/{name}/{method}` 模式。让我们通过修改一个控制器来体验。

<Steps>
<Step title="查看现有控制器">
打开 `electron/controller/example.js`，你会看到类似以下结构：

```javascript
const { Controller } = require('ee-core');
const Log = require('ee-core/log');

class ExampleController extends Controller {
  async hello(args) {
    Log.info('hello args:', args);
    return 'hello electron-egg';
  }
}

ExampleController.toString = () => 'controller/example/';
module.exports = ExampleController;
```

控制器的字符串标识为 `controller/example/`，对应 IPC 通道 `controller/example/hello`。
</Step>

<Step title="添加新方法">
在 `ExampleController` 中新增一个方法：

```javascript
async getTime(args) {
  const now = new Date().toLocaleString('zh-CN');
  Log.info('getTime:', now);
  return now;
}
```

保存文件后重启 Electron（`pnpm dev-electron`），前端即可通过 IPC 调用 `controller/example/getTime`。
</Step>

<Step title="前端调用 IPC">
在 `frontend/src/` 中，通过 preload bridge 调用：

```javascript
// 渲染进程调用
const result = await window.electron.ipc.send('controller/example/getTime', {});
console.log('服务器时间:', result);
```

</Step>
</Steps>

## 修改配置

<Steps>
<Step title="查看默认配置">
打开 `electron/config/config.default.js`，核心配置项包括：

```javascript
module.exports = (appInfo) => {
  const config = {};

  // 开发模式
  config.devMode = true;

  // 日志级别
  config.logger = {
    level: 'INFO',
    output: 'file',  // 'file' | 'console' | 'both'
  };

  // HTTP 服务
  config.httpServer = {
    enable: true,
    port: 7070,
  };

  // Socket.IO 服务
  config.socketServer = {
    enable: true,
    port: 7071,
  };

  return config;
};
```

</Step>

<Step title="创建本地覆盖配置">
创建 `electron/config/config.local.js` 覆盖默认配置（不影响默认文件）：

```javascript
module.exports = (appInfo) => {
  const config = {};

  // 本地开发覆盖
  config.logger = {
    level: 'DEBUG',
    output: 'both',
  };

  return config;
};
```

<Note>
配置加载遵循分层合并：`config.default.js` → `config.local.js`（开发环境）/ `config.prod.js`（生产环境），后者覆盖前者。本地配置仅影响你自己的开发环境，不应提交到版本库。
</Note>
</Step>

<Step title="重启查看变化">
重启 Electron 后，日志级别变为 DEBUG，输出同时写入文件与控制台。
</Step>
</Steps>

## 核心命令速查

| 命令 | 用途 |
|------|------|
| `pnpm dev` | 同时启动前端 + Electron 开发 |
| `pnpm dev-frontend` | 仅启动前端开发服务器 |
| `pnpm dev-electron` | 仅启动 Electron |
| `pnpm start` | 生产模式启动（需先构建） |
| `pnpm build` | 前端构建 + Electron bundle + 加密 |
| `pnpm build-frontend` | 仅构建前端 |
| `pnpm build-electron` | 仅构建 Electron bundle |
| `pnpm build-w` | 打包 Windows 安装包 |
| `pnpm build-m` | 打包 macOS 安装包 |
| `pnpm build-m-arm64` | 打包 macOS ARM64 安装包 |
| `pnpm build-l` | 打包 Linux 安装包 |
| `pnpm encrypt` | 代码加密（字节码/混淆） |
| `pnpm icon` | 生成应用图标 |

<Warning>
修改 `packages/ee-core` 或 `packages/ee-bin` 的 TypeScript 源码后，需在对应目录执行 `npm run build` 重新编译，否则变更不会生效。
</Warning>

## VS Code 调试

项目预置了两个 VS Code 调试配置（`.vscode/launch.json`）：

- **Debug Electron（F5）**：先执行 bundle 构建（`--env=dev`，inline sourcemap），然后以 `--inspect=9229` 启动 Electron。断点设置在 `electron/` 源码文件，sourcemap 会映射回原始文件。
- **Attach Electron**：附加到已运行的 Electron 进程（端口 9229）。
