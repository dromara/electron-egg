# CLI 入口与命令总览

## 概述

ee-bin 是 electron-egg 框架的 CLI 构建工具，基于 Commander.js 实现，提供 9 个子命令管理开发、构建、加密等全流程。所有命令在入口文件中定义，执行逻辑委托给对应的工具模块。

**源文件**: `packages/ee-bin/src/index.ts`

## 命令列表

| 命令 | 功能 | 委托模块 | 源文件路径 |
|------|------|----------|-----------|
| `dev` | 启动开发模式（前端开发服务器 + Electron 进程） | `serveProcess.dev()` | `packages/ee-bin/src/tools/serve.ts` |
| `build` | 构建多资源（前端、Electron 打包、平台安装包） | `serveProcess.build()` | `packages/ee-bin/src/tools/serve.ts` |
| `start` | 生产模式预览启动 | `serveProcess.start()` | `packages/ee-bin/src/tools/serve.ts` |
| `exec` | 执行用户自定义命令 | `serveProcess.exec()` | `packages/ee-bin/src/tools/serve.ts` |
| `move` | 资源移动/复制（如前端 dist 到 public） | `move()` | `packages/ee-bin/src/tools/move.ts` |
| `encrypt` | 代码加密（混淆/字节码） | `encrypt()` | `packages/ee-bin/src/tools/encrypt.ts` |
| `clean` | 清除加密输出文件 | `cleanEncrypt()` | `packages/ee-bin/src/tools/encrypt.ts` |
| `icon` | 生成应用图标 | `iconGenRun()` | `packages/ee-bin/src/tools/iconGen.ts` |
| `updater` | 生成增量更新包 | `incrUpdater.run()` | `packages/ee-bin/src/tools/incrUpdater.ts` |

## CLI 选项

每个命令支持不同的 CLI 选项，通过 Commander 的 `.option()` 定义：

| 选项 | 适用命令 | 说明 |
|------|---------|------|
| `--config <folder>` | dev, build, start, exec, move, encrypt, updater | 自定义 bin.js 配置文件路径（默认 `./cmd/bin.js`） |
| `--serve <mode>` | dev | 指定启动的服务，逗号分隔（如 `"frontend,electron"`） |
| `--cmds <flag>` | build, exec | 指定执行的构建/命令名称，逗号分隔（如 `"frontend,electron,win64"`） |
| `--env <env>` | build | 构建环境（默认 `"prod"`） |
| `--flag <flag>` | move | 指定执行的移动配置键，逗号分隔 |
| `--out <folder>` | encrypt | 加密输出目录覆盖 |
| `--asar-file <file>` | updater | asar 包文件路径 |
| `--platform <flag>` | updater | 目标平台（如 `"mac"`, `"win64"`） |
| `--force <flag>` | updater | 强制全量更新（`"true"` 启用） |
| `-d, --dir <folder>` | clean | 清除目录（默认 `"./public/electron"`） |
| `-i, --input <file>` | icon | 源图片文件 |
| `-o, --output <folder>` | icon | 输出目录 |
| `-s, --size <flag>` | icon | 图标尺寸 |
| `-c, --clear` | icon | 先清空输出目录 |
| `-m, --images <flag>` | icon | 窗口图标/托盘图标的生成路径 |

## 核心流程

入口文件的核心流程如下：

1. **注册 Commander program**: 设置名称 `ee-bin`、描述 `CLI for ee development`、版本号 `5.0.0`
2. **定义 9 个子命令**: 每个命令通过 `.command()` + `.description()` + `.option()` + `.action()` 注册
3. **解析命令行参数**: `program.parse()` 触发 Commander 解析
4. **无子命令时显示帮助**: 若 `process.argv.slice(2)` 为空，调用 `program.outputHelp()` 并用 `chalk.green()` 将帮助文本渲染为绿色

```
program.parse()
↓
用户输入子命令 → 找到对应 action → 调用委托模块方法
无子命令 → program.outputHelp() → chalk.green() 渲染 → 打印帮助信息
```

## 关键设计决策

### 1. 委托模式而非集中式实现

所有命令的执行逻辑都不在 `index.ts` 中，而是委托给独立的工具模块。`index.ts` 仅负责命令注册和参数解析。这种设计使得：
- 每个工具模块可独立开发和测试
- CLI 入口文件保持简洁，只做路由分发
- 工具模块之间的依赖关系清晰

### 2. serveProcess 作为单例

`serveProcess` 是 `ServeProcess` 类的导出单例（`export const serveProcess = new ServeProcess()`），dev/build/start/exec 四个命令共享同一个实例。单例设计的原因：
- 进程管理需要跨命令共享状态（`execProcess` 子进程追踪表）
- `originalPkgMain` 在 dev 模式下需要在 SIGINT/SIGTERM 时恢复，共享实例确保状态一致
- 避免重复注册信号处理器

### 3. encrypt 和 cleanEncrypt 分离

加密命令（`encrypt`）和清除命令（`cleanEncrypt`）来自同一个模块但作为独立导出函数，而非 ServeProcess 的方法。原因：
- 加密是一次性操作，不需要进程管理
- 独立函数更符合语义——加密与开发/构建是不同生命周期
- 清除操作是加密的逆操作，同模块导出便于维护

### 4. 异步与同步命令混合

`dev`/`build`/`start`/`exec` 通过 `serveProcess` 调用，是异步方法（返回 `Promise<void>`）。`move`/`cleanEncrypt` 是同步函数。`encrypt`/`iconGenRun`/`incrUpdater.run` 是异步函数（action 中使用 `async function`）。Commander 的 action 函数可以是普通函数或 async 函数，ee-bin 根据各命令的实际需求选择。

### 5. 帮助文本绿色渲染

当用户不带任何参数运行 `ee-bin` 时，帮助文本用 `chalk.green()` 渲染为绿色。这通过 `program.outputHelp()` 的回调实现，回调函数在输出后返回原始文本（让 Commander 内部逻辑正常工作），同时在回调中用 `console.log()` 打印绿色版本。

## API 导出

`index.ts` 本身不导出任何 API——它是 CLI 入口，通过 `program.parse()` 直接执行。但它依赖以下模块的导出：

| 导入来源 | 导入项 | 类型 |
|---------|--------|------|
| `commander` | `program`, `Command` | Commander.js 核心对象 |
| `./lib/helpers.js` | `chalk` | ANSI 颜色输出工具 |
| `./tools/serve.js` | `serveProcess` | ServeProcess 单例实例 |
| `./tools/move.js` | `move` | 资源移动函数 |
| `./tools/encrypt.js` | `encrypt`, `cleanEncrypt` | 加密/清除函数 |
| `./tools/iconGen.js` | `run` | 图标生成函数 |
| `./tools/incrUpdater.js` | `incrUpdater` | 增量更新器实例 |

## 模块关系

```
index.ts (CLI 入口)
  ├── commander (命令注册与解析)
  ├── lib/helpers.ts (chalk 颜色输出)
  ├── tools/serve.ts → ServeProcess 单例
  │     ├── dev() ← dev 命令
  │     ├── build() ← build 命令
  │     ├── start() ← start 命令
  │     └── exec() ← exec 命令
  ├── tools/move.ts → move() ← move 命令
  ├── tools/encrypt.ts
  │     ├── encrypt() ← encrypt 命令
  │     └── cleanEncrypt() ← clean 命令
  ├── tools/iconGen.ts → run() ← icon 命令
  └── tools/incrUpdater.ts → incrUpdater.run() ← updater 命令
```

配置加载由各工具模块内部通过 `lib/utils.ts` 的 `loadConfig()` 完成，CLI 的 `--config` 选项透传到对应模块。
