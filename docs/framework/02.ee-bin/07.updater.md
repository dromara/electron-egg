# 增量更新

## 概述

ee-bin 的增量更新模块（incrUpdater）负责为 Electron 应用生成增量更新包——只包含应用包的变更部分而非完整安装包，用户下载增量包后与本地已有完整包合并，大幅减少更新下载体积。

增量更新原理：electron-builder 打包时生成包含版本号、文件列表和 SHA512 哈希的元数据 YAML 文件。incrUpdater 读取该元数据，将应用包（asar 归档或目录）+ 额外资源 + 原生模块打包成 zip，计算 SHA1/SHA512 哈希，并生成 JSON 元数据供客户端校验和下载。

模块支持 `asar: true`（应用是单个 .asar 文件）和 `asar: false`（应用是真实目录）两种打包模式，通过 `builderConfig` 字段自动检测并调整路径。

## 核心流程

### run 入口流程

```
incrUpdater.run(options)
  │
  ├─ 加载配置：loadConfig(config) → binCfg → cfg = binCfg.updater
  │
  ├─ 校验：cfg 和 platform 必须存在
  │     缺失 → 抛出错误
  │
  ├─ 解析 force 参数：force === 'true' → forceUpdate = true
  │
  └─ generateFile(cfg, appFile, platform, forceUpdate)
```

相关源文件：`packages/ee-bin/src/tools/incrUpdater.ts` 中 `IncrUpdater.run()` 方法

### _resolveAppFilePath 应用包路径解析

```
_resolveAppFilePath(cfg, cliAppFile)
  │
  ├─ CLI 优先：cliAppFile || cfg.appFile
  │     无值 → 返回空字符串
  │
  ├─ 构建 homeDir + appFile 的绝对路径
  │
  ├─ 读取 builderConfig（若配置了 builderConfig 字段）
  │     ├─ builderFilePath = path.join(homeDir, cfg.builderConfig)
  │     ├─ readJsonSync(builderFilePath) → builderConfig
  │     ├─ asarEnabled = builderConfig.asar !== false
  │     │
  │     ├─ asar:false 且路径以 .asar 结尾
  │     │     → 去掉 .asar 后缀（app 是目录，不是归档）
  │     │     → 输出日志：builder asar:false, adjusted appFile: ...
  │     │
  │     └─ asar:true 且路径不以 .asar 结尾
  │           → 添加 .asar 后缀
  │           → 输出日志：builder asar:true, adjusted appFile: ...
  │
  └─ 返回调整后的绝对路径
```

相关源文件：`packages/ee-bin/src/tools/incrUpdater.ts` 中 `IncrUpdater._resolveAppFilePath()` 方法

**关键设计**：`appFile` 在配置中始终写 `.asar` 后缀（如 `./out/mac-arm64/ee.app/Contents/Resources/app.asar`），由 `_resolveAppFilePath()` 根据对应 builder JSON 的 `asar` 字段自动调整：
- `builder-mac-arm64.json` 中 `asar: false` → 路径变成 `.../app`（目录）
- `builder-mac.json` 中 `asar: true` → 路径保持 `.../app.asar`（归档）

### generateFile 核心流程

```
generateFile(config, cliAppFile, platform, force)
  │
  ├─ 1. 查找平台配置
  │     cfg = config[platform]
  │     不存在 → 抛出错误
  │
  ├─ 2. 读取元数据 YAML
  │     yaml.load(metadataPath) → Metadata{ version, releaseDate, files[{ url, sha512, size }] }
  │     文件不存在 → 抛出错误
  │
  ├─ 3. 解析应用包路径（自动调整 .asar）
  │     appFilePath = _resolveAppFilePath(cfg, cliAppFile)
  │     路径不存在 → 抛出错误
  │
  ├─ 4. 平台名转换（下划线 → 连字符）
  │     mac_arm64 → mac-arm64（用于文件命名）
  │
  ├─ 5. 生成 zip 包
  │     ├─ 构建文件名：updater-{platform}-{version}.zip
  │     ├─ 删除已有 zip（确保干净输出）
  │     ├─ 创建 compressZip.Stream
  │     ├─ 添加应用包
  │     │     ├─ asar:true → 单文件 addEntry(appFilePath, basename)
  │     │     └─ asar:false → 目录 _addFolderToZip(zipStream, appFilePath, 'app')
  │     ├─ 添加 extraResources（globby 扫描，保留 extraResources/ 路径层级）
  │     ├─ 添加 asarUnpacked 模块（保留 app.asar.unpacked/ 路径层级）
  │     └─ 写入磁盘（pipe → writeStream → finish 事件）
  │
  ├─ 6. 计算哈希
  │     zipSha1 = generateHash(zip, 'sha1', 'hex')    → 快速校验
  │     fullSha512 = generateHash(fullInstaller, 'sha512', 'base64') → 强校验
  │
  ├─ 7. 查找完整安装包信息
  │     metadataObj.files 中第一个非 .zip 条目 → fullFileInfo
  │     不存在 → 抛出错误
  │
  ├─ 8. SHA512 完整性校验
  │     fullFileInfo.sha512 !== generateSha512 → 抛出错误
  │     （元数据记录的哈希必须与本地文件计算值一致，防篡改/损坏）
  │
  ├─ 9. 写入 JSON 元数据
  │     {
  │       version,           // 版本号
  │       file: zipName,     // 增量包文件名
  │       size,              // 增量包大小
  │       sha1: zipSha1,     // 增量包 SHA1（快速校验）
  │       fullFile: {        // 完整安装包信息
  │         fileName, size, sha512
  │       },
  │       force,             // 是否强制更新
  │       releaseDate        // 发布日期
  │     }
  │     写入 updater-{platform}.json
  │
  └─ 10. 可选清理：cfg.cleanCache → 删除各平台临时目录
        mac, mac-arm64, win-unpacked, win-ia32-unpacked, linux-unpacked
```

相关源文件：`packages/ee-bin/src/tools/incrUpdater.ts` 中 `IncrUpdater.generateFile()` 方法

### _addFolderToZip 递归添加流程

```
_addFolderToZip(zipStream, folderPath, zipDir)
  │
  ├─ 读取目录条目：fs.readdirSync(folderPath, { withFileFileTypes: true })
  │
  └─ 遍历每个条目
      ├─ 目录 → _addFolderToZip(zipStream, fullPath, zipDir + '/' + name)（递归）
      └─ 文件 → zipStream.addEntry(fullPath, { relativePath: zipDir + '/' + name })
```

相关源文件：`packages/ee-bin/src/tools/incrUpdater.ts` 中 `IncrUpdater._addFolderToZip()` 方法

**asar:false 时**，应用是目录而非 .asar 归档，`_addFolderToZip()` 将整个 `app/` 目录递归添加到 zip 中，保留完整目录结构。

### generateHash 哈希计算流程

```
generateHash(filepath, algorithm, encoding)
  │
  ├─ filepath 为空 → 抛出错误（防止空哈希写入元数据）
  │
  ├─ filepath 不存在 → 抛出错误（防止虚假校验值）
  │
  └─ 计算哈希
      fs.readFileSync(filepath) → buffer
      crypto.createHash(algorithm).update(buffer).digest(encoding)
      ├─ sha1 + hex → 用于 zip 快速校验
      └─ sha512 + base64 → 用于完整包强校验
```

相关源文件：`packages/ee-bin/src/tools/incrUpdater.ts` 中 `IncrUpdater.generateHash()` 方法

## 关键设计决策

### 1. 增量而非全量更新

核心设计原则：只打包变更的应用包，而非完整安装程序。典型场景下应用包约 10-50MB，而完整安装包 50-200MB，增量更新可减少 60-90% 的下载量。

### 2. SHA512 双重校验（完整性保证）

增量包的 SHA512 必须与元数据 YAML 中记录的 SHA512 一致：

- 元数据由 electron-builder 在打包时生成，包含每个文件的 SHA512
- incrUpdater 重新计算本地完整安装包的 SHA512 并比对
- 不匹配则拒绝生成增量包——说明完整包被篡改或损坏，增量包无法可靠地基于它生成

SHA1 用于增量包本身的快速校验（hex 格式，客户端下载后比对），SHA512 用于完整包的强校验（base64 格式，防篡改）。

### 3. builderConfig 自动检测 asar 模式

`UpdaterConfig` 新增 `builderConfig` 字段，指向目标平台的 electron-builder 配置文件。`_resolveAppFilePath()` 读取该配置的 `asar` 字段，自动调整 `appFile` 路径：

- `asar: true` → 保留 `.asar` 后缀，应用是单文件归档
- `asar: false` → 去掉 `.asar` 后缀，应用是真实目录

配置中 `appFile` 值始终写 `.asar` 后缀（如 `app.asar`），无需根据打包模式手动修改——模块自动适配。

### 4. asar:false 时目录打包而非单文件

当 `asar: false`，打包产物是 `app/` 目录而非 `app.asar` 单文件。`generateFile()` 通过 `fs.statSync()` 判断路径是文件还是目录，选择不同的 zip 添加策略：
- 目录 → `_addFolderToZip(zipStream, appFilePath, 'app')`（递归添加）
- 文件 → `zipStream.addEntry(appFilePath, { relativePath: basename })`（单文件）

### 5. 空路径 / 不存在路径抛出错误而非静默返回

`generateHash()` 对空路径和不存在的路径直接抛出错误，而非返回空字符串或跳过：

- 防止空哈希被写入更新元数据——客户端会因校验失败而无法更新
- 强制开发者确保文件路径正确，避免在发布流程中隐藏问题

### 6. 平台名下划线到连字符转换

配置键使用下划线（`mac_arm64`）遵循 JavaScript 对象键命名惯例，而文件名使用连字符（`mac-arm64`）遵循操作系统命名惯例。模块在生成文件名时自动转换：

```
配置键: updater.macos_apple  →  文件名: updater-macos-apple.json
配置键: updater.macos_intel   →  文件名: updater-macos-intel.json
```

### 7. asarUnpacked 路径层级保留

原生模块（如 better-sqlite3）必须存在于 `app.asar.unpacked/` 目录下——Electron 运行时在该路径查找原生模块。模块在 zip 中保留此路径层级：

```
zip 内路径: app.asar.unpacked/node_modules/better-sqlite3/...
磁盘路径:   resources/app.asar.unpacked/node_modules/better-sqlite3/...
```

### 8. extraResources 路径层级保留

额外资源文件在 zip 中从 `extraResources/` 开始保留路径层级：

```
zip 内路径: extraResources/config/app-config.json
磁盘路径:   extraResources/config/app-config.json
```

通过 `indexOf('extraResources')` 截取路径后段，确保无论源路径多深，zip 内路径始终从 `extraResources/` 开始。

### 9. 可选缓存清理

`cfg.cleanCache` 控制是否在生成后清理各平台临时目录。这些目录是 electron-builder 打包时解压产生的临时文件，通常体积较大。启用清理可节省磁盘空间，但禁用清理允许开发者复查打包内容。

## API 导出

| 导出 | 类型 | 说明 |
|------|------|------|
| `incrUpdater` | IncrUpdater 实例 | 单例导出，直接调用 `.run()` |
| `IncrUpdater` | class | 类导出，可单独实例化 |

**UpdaterOptions 接口**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `config` | `string?` | 自定义 bin.js 配置文件路径 |
| `appFile` | `string?` | CLI 指定的应用包路径（覆盖配置，自动检测 .asar） |
| `platform` | `string?` | 目标平台名（如 `macos_apple`, `windows_64`） |
| `force` | `string?` | 强制更新标志（`'true'` 字符串） |

**UpdaterConfig 接口**（定义于 `packages/ee-bin/src/types/config.ts`）：

| 字段 | 类型 | 说明 |
|------|------|------|
| `metadata` | `string` | 元数据 YAML 文件路径 |
| `appFile` | `string?` | 应用包文件路径（.asar 后缀由 builderConfig 自动调整） |
| `builderConfig` | `string?` | 对应平台的 builder JSON 路径（用于检测 asar:true/false） |
| `output` | `object` | 输出配置（directory, zip, file） |
| `extraResources` | `string[]?` | 额外资源 glob 匹配模式 |
| `asarUnpacked` | `string[]?` | 原生模块列表 |
| `cleanCache` | `boolean?` | 是否清理临时目录 |

**Metadata / MetadataFile 接口**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `version` | `string` | 版本号 |
| `releaseDate` | `string` | 发布日期 |
| `files` | `MetadataFile[]` | 文件列表 |
| `url` | `string` | 文件下载 URL |
| `sha512` | `string` | 文件 SHA512 哈希 |
| `size` | `number` | 文件大小（字节） |

## 模块关系

```
incrUpdater.ts
  ├─ 引入 helpers.ts
  │     └─ chalk → 日志颜色输出
  │
  ├─ 引入 utils.ts
  │     ├─ loadConfig → 加载并合并 bin.js 配置
  │     ├─ readJsonSync → 读取 builder JSON（检测 asar 字段）
  │     └─ writeJsonSync → 写入 JSON 元数据文件
  │
  ├─ 引入 types/config.ts
  │     └─ UpdaterConfig → 更新配置类型定义
  │
  ├─ 外部依赖
  │     ├─ compressing → zip 压缩流
  │     ├─ js-yaml → YAML 解析
  │     ├─ crypto → 哈希计算
  │     └─ globby → 文件扫描（extraResources）
  │
  └─ Node.js 内置
      ├─ fs → 文件读写、statSync（判断文件/目录）、目录操作
      └─ path → 路径拼接和解析
```

**典型调用链**：

```
CLI 命令 → incrUpdater.run({ config, appFile, platform, force })
  → loadConfig() → binCfg.updater → cfg
  → generateFile(cfg, cliAppFile, platform, forceUpdate)
    → _resolveAppFilePath(cfg, cliAppFile)
      → readJsonSync(builderConfig) → builderConfig.asar
      → asar:false → strip .asar / asar:true → keep .asar
    → yaml.load(metadata) → Metadata
    → 确定应用包路径（已自动调整）
    → 平台名转换
    → new compressZip.Stream()
      ├─ asar:true → addEntry(appFilePath, basename)
      ├─ asar:false → _addFolderToZip(zipStream, appFilePath, 'app')
      → globbySync(extraResources) → addEntry（逐个）
      → _addFolderToZip(asarUnpacked modules)
    → pipe → writeStream → finish
    → generateHash(zip, 'sha1', 'hex') → zipSha1
    → generateHash(fullPackage, 'sha512', 'base64') → fullSha512
    → SHA512 完整性校验
    → writeJsonSync(updater-{platform}.json, metadata)
    → cleanCache → rm(tmpAppDirs)
```
