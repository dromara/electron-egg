# bin_default 配置默认值与类型体系

## 概述

ee-bin 的配置系统由两个核心模块构成：`bin_default.ts` 定义所有配置项的默认值，`types/config.ts` 定义完整的 TypeScript 类型体系。用户在 `./cmd/bin.js` 中提供的自定义配置通过 `extend()` 函数深合并覆盖默认值，最终形成完整的 `BinConfig` 实例。

配置结构分六大板块：
- **dev**：开发模式子进程配置（前端 dev server + Electron 主进程）
- **build**：构建配置（前端构建 + Electron esbuild 打包 + 8 个平台打包命令）
- **move**：资源拷贝配置（前端产物 → 公共目录）
- **start**：生产模式启动配置
- **encrypt**：加密配置（前端混淆 + Electron 混淆/字节码）
- **exec**：自定义命令配置（用户自定义，默认为空）

## 核心流程

### 配置合并流程

```
用户 CLI 命令
  │
  ├─ loadConfig(binFile?)
  │     ├─ binFile 未指定 → 默认 './cmd/bin.js'
  │     ├─ loadFile(binFile) → userConfig（用户自定义配置）
  │     └─ extend(true, { ...defaultConfig }, userConfig) → BinConfig
  │
  ├─ 合并规则（extend 函数）
  │     ├─ 纯对象属性 → 递归深合并（用户字段补充/覆盖默认字段）
  │     ├─ 数组属性 → 全量覆盖（用户数组完全替换默认数组）
  │     ├─ 原始类型属性 → 直接覆盖
  │     └─ __proto__/constructor/prototype → 过滤（防原型链污染）
  │
  └─ 合并结果
      默认配置的所有字段都存在
      用户配置的字段值覆盖默认值
      用户新增的字段被保留（如自定义 exec 命令）
```

相关源文件：`packages/ee-bin/src/lib/utils.ts` 中 `loadConfig()`，`packages/ee-bin/src/lib/extend.ts` 中 `extend()`，`packages/ee-bin/src/config/bin_default.ts` 中 `defaultConfig`

### 类型体系层次结构

```
BinConfig（顶层）
  ├─ dev: DevConfig
  │     ├─ frontend: DevFrontendConfig（继承 ExecConfig + 必填字段）
  │     ├─ electron: DevElectronConfig（继承 ExecConfig + 必填字段）
  │     └─ [key: string]: ExecConfig | undefined（允许自定义子进程）
  │
  ├─ build: BuildConfig
  │     ├─ electron: BundleConfig（esbuild 打包配置，唯一必须为 BundleConfig 的键）
  │     └─ [key: string]: ExecConfig | BundleConfig | undefined
  │        frontend, win32, win64, win_e, win_7z, mac, mac_arm64, linux, linux_arm64
  │
  ├─ move: Record<string, MoveConfig>
  │     例: frontend_dist: { src, dest }
  │
  ├─ start: ExecConfig
  │     例: { directory: './', cmd: 'electron', args: ['.', '--env=prod'] }
  │
  ├─ encrypt: Record<string, EncryptConfig>
  │     ├─ frontend: EncryptConfig（前端混淆）
  │     └─ electron: EncryptConfig（Electron 混淆/字节码）
  │
  ├─ exec: Record<string, ExecConfig>
  │     默认 {}，用户自定义
  │
  └─ updater?: Record<string, UpdaterConfig>（可选）
      例: mac: UpdaterConfig, mac_arm64: UpdaterConfig
```

相关源文件：`packages/ee-bin/src/types/config.ts` 中完整类型定义

## 关键设计决策

### 1. 默认配置全覆盖

`bin_default.ts` 为每个配置板块都提供完整的默认值，确保：

- 用户 `./cmd/bin.js` 可以只写需要自定义的字段，其余自动取默认值
- `extend()` 深合并后，所有必需字段都存在，运行时无需处理缺失字段
- 新版本 ee-bin 添加新默认字段时，老用户的 `bin.js` 无需修改即可获得新默认值

### 2. BuildConfig 混合类型设计

`BuildConfig` 的 `electron` 键类型为 `BundleConfig`（esbuild 打包参数），其他键类型为 `ExecConfig`（命令执行参数）。索引签名 `[key: string]: ExecConfig | BundleConfig | undefined` 兼容两种类型：

```typescript
interface BuildConfig {
  electron: BundleConfig;            // 打包配置（非命令执行）
  [key: string]: ExecConfig | BundleConfig | undefined;  // 命令执行或打包配置
}
```

这种设计允许 build 配置中同时包含打包参数和平台打包命令，语义清晰且类型安全。

### 3. ExecConfig 广泛复用

`ExecConfig` 是最通用的配置接口，被 `DevConfig`、`BuildConfig`、`start`、`exec` 共同复用：

- `dev.frontend` → `DevFrontendConfig extends ExecConfig`（增加 protocol/port/hostname 等前端特有字段）
- `dev.electron` → `DevElectronConfig extends ExecConfig`（增加 loadingPage/watch/delay 等开发特有字段）
- `build.frontend` / `build.win64` 等 → `ExecConfig`（直接使用基础字段）
- `start` → `ExecConfig`（直接使用）
- `exec` → `Record<string, ExecConfig>`（用户自定义命令）

继承模式确保每个子配置至少拥有 `directory/cmd` 等基础字段，同时允许添加特定场景的扩展字段。

### 4. EncryptConfig 双策略分离

前端和 Electron 使用不同的加密策略：

| 配置项 | 前端 | Electron |
|--------|------|----------|
| type | `none` / `confusion` | `none` / `bytecode` / `strict` |
| target | `browser` | `node` |
| stringArrayEncoding | `none` | `rc4` |
| bytecodeOptions | 不存在 | `{ electron: true }` |
| specificFiles | 空 | `bridge.js`（preload 脚本必须可读） |
| entryFiles | 不存在 | `main.js`（入口必须保留 .js 后缀） |

设计原因：
- 前端不支持 bytecode：V8 字节码与浏览器渲染进程的 V8 版本不兼容
- Electron 支持 bytecode：主进程 Node.js 环境的 V8 版本与编译时一致
- bridge.js 必须使用混淆而非字节码：BrowserWindow preload 脚本需保持可读格式
- main.js 作为入口文件：字节码模式下必须保留 .js 后缀，实际内容变为 bytenode loader shell

### 5. MoveConfig 路径别名

`MoveConfig` 的 `dist` 和 `target` 字段分别覆盖 `src` 和 `dest`，允许用户在不修改默认配置结构的情况下指定不同路径：

```typescript
interface MoveConfig {
  src: string;     // 源路径（必需）
  dest: string;    // 目标路径（必需）
  dist?: string;   // 源路径别名（覆盖 src）
  target?: string; // 目标路径别名（覆盖 dest）
}
```

默认配置中 `frontend_dist` 的 `src/dest` 为基础值，用户可通过 `target` 字段指定不同的目标目录而保留默认 `src`。

### 6. BundleConfig sourcemap 三态设计

`sourcemap` 字段支持四种值，覆盖不同使用场景：

| 值 | 行为 | 适用场景 |
|----|------|----------|
| `false` / 未设置 | 自动模式：dev → inline，prod → off | 默认，最常用 |
| `'inline'` | 强制内联 sourcemap | 开发调试 |
| `'external'` | 生成独立 `.map` 文件 | 生产环境远程调试 |
| `true` | 等同 `'inline'` | 简写 |

自动模式减少用户心智负担——开发时自动有 sourcemap，生产时自动禁用。

### 7. UpdaterConfig 可选顶层配置

`updater` 是 `BinConfig` 的可选字段（`updater?: Record<string, UpdaterConfig>`），因为：

- 增量更新是高级功能，大多数项目不需要
- 未配置时不影响其他功能（dev/build/move/start/encrypt）
- 只有使用 `pnpm updater` 命令时才需要

## API 导出

### bin_default.ts 导出

| 导出 | 类型 | 说明 |
|------|------|------|
| `config` | `BinConfig` (default export) | 完整默认配置对象 |

默认配置完整字段一览：

| 板块 | 键 | 核心字段 |
|------|-----|----------|
| **dev.frontend** | directory:`./frontend`, cmd:`npm`, args:`['run', 'dev']`, protocol:`http://`, hostname:`localhost`, port:`8080` |
| **dev.electron** | directory:`./`, cmd:`electron`, args:`['.', '--env=local']`, loadingPage:`/public/html/loading.html`, watch:`false`, delay:`1000` |
| **build.frontend** | directory:`./frontend`, cmd:`npm`, args:`['run', 'build']` |
| **build.electron** | BundleConfig: bundleType:`bundle`, format:`cjs`, sourcemap:`false`, minify:`false`, keepNames:`false`, drop:`[]`, legalComments:`none`, define:`{}`, copy:`[]`, external:`[]` |
| **build.win32/win64/win_e/win_7z/mac/mac_arm64/linux/linux_arm64** | 各平台 electron-builder 打包命令 |
| **move.frontend_dist** | src:`./frontend/dist`, dest:`./public/dist` |
| **start** | directory:`./`, cmd:`electron`, args:`['.', '--env=prod']` |
| **encrypt.frontend** | type:`none`, target:`browser`, stringArrayEncoding:`['none']` |
| **encrypt.electron** | type:`none`, target:`node`, stringArrayEncoding:`['rc4']`, bytecodeOptions:`{ electron: true }` |
| **exec** | `{}`（空，用户自定义） |

### types/config.ts 导出

| 类型 | 说明 |
|------|------|
| `BinConfig` | 顶层配置接口 |
| `DevConfig` | 开发模式配置 |
| `DevFrontendConfig` | 开发前端配置（继承 ExecConfig） |
| `DevElectronConfig` | 开发 Electron 配置（继承 ExecConfig） |
| `BuildConfig` | 构建配置（electron 为 BundleConfig，其余为 ExecConfig） |
| `BundleConfig` | esbuild 打包配置 |
| `ExecConfig` | 命令执行配置（最通用接口） |
| `EncryptConfig` | 加密配置 |

| `BytecodeOptions` | bytenode 字节码编译选项 |
| `MoveConfig` | 资源拷贝配置 |
| `UpdaterConfig` | 增量更新配置 |

### ExecConfig 完整字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `directory` | `string` | 进程工作目录 |
| `cmd` | `string` | 执行命令 |
| `args` | `string[]?` | 命令参数 |
| `stdio` | `'inherit'|'pipe'|'ignore'?` | 子进程 IO 模式 |
| `sync` | `boolean?` | 是否同步执行 |
| `protocol` | `string?` | 前端服务协议（`http://` 或 `file://`） |
| `watch` | `boolean?` | 是否监听文件变更并自动重启 |
| `delay` | `number?` | 监听模式防抖延迟（毫秒） |
| `hostname` | `string?` | 前端开发服务器主机名 |
| `port` | `number?` | 前端开发服务器端口 |
| `indexPath` | `string?` | 前端首页文件名 |
| `force` | `boolean?` | 是否强制刷新页面 |
| `loadingPage` | `string?` | Electron 加载页路径 |

### BundleConfig 完整字段

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `bundleType` | `'bundle'|'copy'?` | `'bundle'` | 打包模式：bundle 用 esbuild 打包，copy 直接拷贝 |
| `external` | `string[]?` | `[]` | 用户自定义 esbuild external 包名 |
| `sourcemap` | `boolean|'inline'|'external'?` | `false`(自动) | Sourcemap 配置 |
| `minify` | `boolean?` | `false` | 是否压缩代码 |
| `keepNames` | `boolean?` | `false` | 压缩时保留函数/类名 |
| `drop` | `('console'|'debugger')[]?` | `[]` | 移除的语句类型 |
| `legalComments` | `'inline'|'eof'|'none'?` | `'none'` | 许可注释处理 |
| `define` | `Record<string, string>?` | `{}` | 编译时常量定义 |
| `copy` | `string[]?` | `[]` | 额外拷贝文件/目录 |
| `format` | `'cjs'|'esm'?` | `'cjs'` | 输出格式 |

### EncryptConfig 完整字段

| 字段 | 类型 | 说明 |

## 模块关系

```
bin_default.ts
  ├─ 引入 types/config.ts
  │     └─ BinConfig → 类型标注
  │
  └─ 被 utils.ts 引用
      └─ defaultConfig → loadConfig() 中的默认值基础
        └─ extend(true, { ...defaultConfig }, userConfig) → BinConfig

types/config.ts
  ├─ 被 bin_default.ts 引用 → BinConfig 类型标注
  ├─ 被 utils.ts 引用 → BinConfig 类型标注
  ├─ 被 move.ts 引用 → MoveConfig
  ├─ 被 incrUpdater.ts 引用 → UpdaterConfig
  ├─ 被 encrypt.ts 引用 → EncryptConfig, ConfusionOptions, BytecodeOptions
  └─ 被 serve.ts 引用 → ExecConfig, DevConfig

配置类型 → 默认值 → 合合 → 运行时 的完整链路：
  types/config.ts（类型定义）
    → bin_default.ts（默认值实例化）
      → utils.ts loadConfig()（加载 + 合合）
        → 各功能模块（使用合合后的 BinConfig）
```
