# 图标生成

## 概述

ee-bin 的图标生成模块（iconGen）负责从一张高分辨率源图片生成 Electron 应用所需的全部图标格式，包括 Windows 的 `.ico` 文件、各尺寸的 `.png` 文件，以及按 electron-egg 框架命名约定重命名/拷贝到指定位置的图标资源。

该模块在 `pnpm icon` 命令时被调用，生成的图标用于：

- Windows 安装程序和任务栏图标（`.ico`）
- Linux 各尺寸桌面图标（`.png`）
- macOS dock 图标（`.icns` 由 icon-gen 自动生成）
- 系统托盘图标（`tray.png`）
- 窗口图标（32px `.png`）

## 核心流程

### run 入口流程

```
run(opts)
  │
  ├─ extractParams(opts) → IconGenParams（应用默认值）
  │     input:  opts.input || '/public/images/logo.png'
  │     output: opts.output || '/build/icons/'
  │     size:   opts.size || '16,32,64,256,512'
  │     clear:  opts.clear === true
  │     imagesDir: opts.images || '/public/images/'
  │
  └─ new IconGen(params) → generateIcons()
```

相关源文件：`packages/ee-bin/src/tools/iconGen.ts` 中 `run()` 和 `extractParams()` 函数

### IconGen 构造流程

```
IconGen.constructor(params)
  │
  ├─ 加载 icon-gen 可选依赖
  │     require('icon-gen')
  │     ├─ 成功 → this.icongen = 模块函数（支持 ESM default 提取）
  │     ├─ Cannot find module → this.icongen = null（静默，后续提示安装）
  │     └─ 其他错误 → this.icongen = null + 打印真实错误信息
  │
  ├─ 解析绝对路径
  │     this.input = path.join(cwd, params.input)
  │     this.output = path.join(cwd, params.output)
  │     this.imagesDir = path.join(cwd, params.imagesDir)
  │
  └─ 构建 iconOptions
      ├─ report: false（禁用 icon-gen 详细报告）
      ├─ ico: { name: 'icon', sizes: [256] }  → 生成 icon.ico（256x256）
      └─ favicon: { name: 'logo-', pngSizes: sizeList }
            → 生成 logo-16.png, logo-32.png, logo-64.png, logo-256.png, logo-512.png
```

相关源文件：`packages/ee-bin/src/tools/iconGen.ts` 中 `IconGen` 类构造函数

### generateIcons 主流程

```
IconGen.generateIcons()
  │
  ├─ 1. 检查 icon-gen 可用性
  │     this.icongen === null → 提示安装 pnpm add icon-gen，return
  │
  ├─ 2. 校验源图片存在
  │     fs.existsSync(this.input) === false → 抛出错误
  │
  ├─ 3. 准备输出目录
  │     ├─ 目录不存在 → fs.mkdirSync(output, { recursive: true })
  │     ├─ 目录存在 + params.clear → deleteGenFile(output)（只删 .ico/.png）
  │     └─ 准备 imagesDir → fs.mkdirSync(imagesDir, { recursive: true })
  │
  ├─ 4. 运行 icon-gen 生成
  │     this.icongen(input, output, iconOptions) → results（文件路径数组）
  │     ├─ 成功 → 打印生成结果
  │     └─ 失败 → 抛出错误
  │
  └─ 5. 后处理：_renameForEE(results)
      → 按 electron-egg 命名约定重命名/拷贝图标
```

相关源文件：`packages/ee-bin/src/tools/iconGen.ts` 中 `IconGen.generateIcons()` 方法

### _renameForEE 后处理流程

```
_renameForEE(filesPath)
  │
  ├─ 遍历每个 .png 文件
  │     从文件名提取尺寸数字（例: "logo-16" → "16"）
  │
  ├─ basename === '16'
  │     → 拷贝到 imagesDir/tray.png（系统托盘图标）
  │     → 删除原始 logo-16.png
  │
  ├─ basename === '32'
  │     → 拷贝到 imagesDir/logo-32.png（窗口图标，保留原名）
  │     → 不删除原始文件（build/icons/ 和 imagesDir 各一份）
  │
  └─ 其他尺寸 (64, 256, 512)
      → 在 build/icons/ 内重命名为 NxN.png 格式
        例: logo-64.png → 64x64.png
        例: logo-256.png → 256x256.png
        例: logo-512.png → 512x512.png
```

相关源文件：`packages/ee-bin/src/tools/iconGen.ts` 中 `IconGen._renameForEE()` 方法

### deleteGenFile 清理流程

```
deleteGenFile(dirPath)
  │
  └─ 递归遍历目录
      ├─ 子目录 → deleteGenFile(子目录)（递归）
      └─ 文件 → 扩展名为 .ico 或 .png？
            ├─ 是 → fs.unlinkSync（删除）
            └─ 否 → 保留（不删除其他文件）
```

相关源文件：`packages/ee-bin/src/tools/iconGen.ts` 中 `IconGen.deleteGenFile()` 方法

## 关键设计决策

### 1. icon-gen 作为可选依赖

icon-gen 是一个可选 npm 包，不是 ee-bin 的硬性依赖。模块在构造函数中通过 `try/catch` 加载 icon-gen：

- 未安装时：`this.icongen = null`，`generateIcons()` 打印友好提示（`pnpm add icon-gen`）后返回
- 安装但加载失败时（如原生模块编译错误）：打印真实错误信息供用户排查

这种设计减少了 ee-bin 的默认依赖体积——大多数用户不需要频繁生成图标。

### 2. 分层输出策略

图标资源被分发到两个不同目录，各有用途：

| 目录 | 用途 | 文件 |
|------|------|------|
| `build/icons/` | electron-builder 打包使用 | `icon.ico`, `64x64.png`, `256x256.png`, `512x512.png` |
| `public/images/` | Electron 运行时使用 | `tray.png`（16px 托盘图标），`logo-32.png`（窗口图标） |

### 3. .ico 只生成 256 尺寸

`iconOptions.ico.sizes` 硬编码为 `[256]`，只生成一个包含 256x256 图标的 `.ico` 文件。Windows `.ico` 文件内部可包含多尺寸图层，但 icon-gen 库的处理方式使得单一 256 尺寸足够覆盖 Windows 安装程序和任务栏需求。

### 4. deleteGenFile 只删除图标文件

清理输出目录时（`--clear` 参数），只删除 `.ico` 和 `.png` 文件，保留其他类型的文件。这避免误删用户在 `build/icons/` 中手动放置的非图标资源。

### 5. 16px 托盘图标删除原始文件

16px 图标拷贝到 `public/images/tray.png` 后，原始的 `build/icons/logo-16.png` 被删除。因为 16px 只用于系统托盘，不需要在 `build/icons/` 中保留。而 32px 窗口图标在两个目录各保留一份——`build/icons/` 供打包使用，`public/images/` 供运行时使用。

## API 导出

| 导出 | 类型 | 说明 |
|------|------|------|
| `run(opts)` | async function | 图标生成入口函数 |

**IconGenParams 接口**：

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `input` | `string` | `/public/images/logo.png` | 源图片路径 |
| `output` | `string` | `/build/icons/` | 输出目录 |
| `size` | `string` | `16,32,64,256,512` | 逗号分隔的尺寸列表 |
| `clear` | `boolean` | `false` | 是否在生成前清空输出目录 |
| `imagesDir` | `string` | `/public/images/` | 托盘/窗口图标输出目录 |

**DEFAULT_PARAMS 常量**：定义所有参数的默认值，`extractParams()` 中未提供的 CLI 选项从此处取值。

## 模块关系

```
iconGen.ts
  ├─ 引入 helpers.ts
  │     └─ chalk → 日志颜色输出
  │
  ├─ 外部依赖（可选）
  │     └─ icon-gen → 图标生成库（npm 包，非硬依赖）
  │
  ├─ Node.js 内置
  │     ├─ fs → 文件系统操作（目录创建、文件拷贝/重命名/删除）
  │     └─ path → 路径拼接和解析
  │
  └─ 产出文件分布
      ├─ build/icons/icon.ico          → Windows 安装包图标
      ├─ build/icons/64x64.png         → Linux 桌面图标
      ├─ build/icons/256x256.png       → 通用大尺寸图标
      ├─ build/icons/512x512.png       → macOS/高 DPI 图标
      ├─ public/images/tray.png        → 系统托盘图标（16px）
      └─ public/images/logo-32.png     → 窗口图标（32px）
```

**典型调用链**：

```
CLI 命令 pnpm icon → run({ input, output, size, clear, images })
  → extractParams() → IconGenParams
  → new IconGen(params)
    → require('icon-gen') → this.icongen
    → 解析路径 → this.input/output/imagesDir
    → 构建 iconOptions → { ico: {...}, favicon: {...} }
  → IconGen.generateIcons()
    → 校验 + 准备目录
    → this.icongen(input, output, iconOptions) → 文件列表
    → _renameForEE(results) → 重命名/拷贝
```
