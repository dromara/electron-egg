# Storage 模块

Storage 模块基于 better-sqlite3 提供 SQLite 数据库操作能力，自动处理数据库文件路径、目录创建和安全防护。

## SqliteStorage 类

### 四种存储模式

| 模式 | 数据库名格式 | 文件存储位置 | 说明 |
|------|-------------|-------------|------|
| memory | `:memory:` | 内存中 | 不持久化，进程退出数据丢失 |
| onlyName | `app.db` | `{dataDir}/db/app.db` | 仅文件名，存储到默认目录 |
| relative | `sub/app.db` | `{dataDir}/db/sub/app.db` | 相对路径，基于默认目录 |
| absolute | `/tmp/app.db` | `/tmp/app.db` | 绝对路径，直接使用指定路径 |

<Note>
`dataDir` 在开发环境为 `{baseDir}/data`，生产环境为 `{userHome}/.{appName}/data`。
</Note>

### 模式推断逻辑

`getMode()` 根据数据库名自动推断存储模式：

1. 名为 `:memory:` → memory 模式
2. 包含路径分隔符（`/` 或 `\`）：
   - 绝对路径 → absolute 模式
   - 相对路径 → relative 模式
3. 不包含路径分隔符 → onlyName 模式

### 安全防护

构造函数验证数据库名称，防止路径遍历攻击：

```javascript
// 禁止包含 ".." 的名称
new SqliteStorage('../etc/passwd.db');
// AssertionError: db name ../etc/passwd.db contains path traversal
```

<Warning>
路径遍历检查在构造函数中执行，空名称也会被拒绝。任何包含 `..` 的名称都会抛出断言错误。
</Warning>

### 自动目录创建

文件模式下，如果数据库目录不存在，自动创建并设置权限为 `0o755`：

```javascript
// absolute 模式：目录取自数据库名的路径部分
new SqliteStorage('/custom/path/app.db');
// 自动创建 /custom/path/ 目录

// other 模式：使用 {dataDir}/db 作为目录
new SqliteStorage('app.db');
// 自动创建 {dataDir}/db/ 目录
```

## 使用示例

<CodeGroup>

```javascript [名称模式 — 推荐]
import { SqliteStorage } from 'ee-core/storage';

// 仅指定文件名，自动存储到默认位置
const db = new SqliteStorage('myapp.db');

// 数据库文件路径
console.log(db.getFilePath());  // /path/to/data/db/myapp.db
console.log(db.getDbDir());     // /path/to/data/db
console.log(db.fileName);       // myapp.db
console.log(db.mode);           // onlyName

// better-sqlite3 同步操作
const stmt = db.db.prepare('INSERT INTO users (name) VALUES (?)');
stmt.run('张三');

const rows = db.db.prepare('SELECT * FROM users').all();
```

```javascript [内存模式 — 测试用]
import { SqliteStorage } from 'ee-core/storage';

// 内存数据库，不持久化
const db = new SqliteStorage(':memory:');

console.log(db.mode);   // memory
console.log(db.fileName); // :memory:

// 适合单元测试和临时数据处理
db.db.exec('CREATE TABLE temp (id INTEGER, value TEXT)');
```

```javascript [相对路径模式]
import { SqliteStorage } from 'ee-core/storage';

// 相对路径，存储到 {dataDir}/db/sub/ 目录
const db = new SqliteStorage('sub/myapp.db');

console.log(db.mode);     // relative
console.log(db.getFilePath()); // /path/to/data/db/sub/myapp.db
```

```javascript [绝对路径模式]
import { SqliteStorage } from 'ee-core/storage';

// 绝对路径，直接使用指定位置
const db = new SqliteStorage('/tmp/data/myapp.db');

console.log(db.mode);     // absolute
console.log(db.getFilePath()); // /tmp/data/myapp.db
```

</CodeGroup>

## better-sqlite3 特性

better-sqlite3 使用同步 API，所有操作都是阻塞的：

- **无需 async/await** — 简化代码逻辑
- **无需回调** — 直接返回结果
- **性能优异** — 比 async 方式的 node-sqlite3 快很多
- **事务安全** — 支持完整的事务控制

```javascript
// 事务示例
const db = storage.db;

const insert = db.transaction((users) => {
  for (const user of users) {
    db.prepare('INSERT INTO users (name, email) VALUES (?, ?)').run(user.name, user.email);
  }
});

insert([{ name: '张三', email: 'a@test.com' }, { name: '李四', email: 'b@test.com' }]);
```

<Note>
better-sqlite3 的同步操作不会阻塞 Electron 主进程的 IPC 通信，因为 SQLite 操作通常很快（微秒级）。但大量数据操作应放在子进程（ChildJob）中执行。
</Note>

## 构造函数选项

```javascript
const db = new SqliteStorage('myapp.db', {
  timeout: 5000,   // SQLite 超时时间(ms)，默认 5000
  // 其他 better-sqlite3 选项
});
```

<Warning>
better-sqlite3 是原生模块，需要针对 Electron 重新编译。使用 `pnpm re-sqlite` 命令重新编译，或在看到原生模块错误时执行此命令。
</Warning>

## API 参考

| 属性/方法 | 类型 | 说明 |
|----------|------|------|
| `name` | `string` | 数据库名（原始输入） |
| `mode` | `string` | 存储模式：memory/onlyName/relative/absolute |
| `dbDir` | `string` | 数据库目录绝对路径 |
| `fileName` | `string` | 数据库文件名部分 |
| `db` | `Database` | better-sqlite3 实例 |
| `getDbDir()` | `string` | 获取数据库目录路径 |
| `getFilePath()` | `string` | 获取数据库文件完整路径 |
