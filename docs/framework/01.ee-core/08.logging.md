# 日志系统

## 概述

ee-core 的日志系统基于 [pino](https://getpino.io/) 高性能 JSON 日志库构建，提供三个独立的日志实例，分别服务于不同场景：

- **logger**：应用日志，记录业务代码输出
- **coreLogger**：框架核心日志，记录框架内部运行信息
- **errorLogger**：错误日志，仅记录 error/fatal 级别

系统通过 Proxy 包装日志实例，兼容多种调用风格（pino 标准、printf、拼接模式），降低开发者使用门槛。日志文件通过 pino-roll 实现按日/按时轮转，开发环境额外提供 pino-pretty 终端彩色输出。

## 核心流程

### 启动流程

日志系统在框架启动流程中由 `boot.ts` 的 `init()` 调用 `loadLog()` 初始化：

```
Boot → loadException → loadConfig → loadLog → loadDir → ...
```

1. `loadLog()` 调用 `create()` 构建三个 pino 实例
2. `create()` 合并默认配置与系统配置，构建 pino LoggerOptions
3. 为每个日志文件构建 pino-roll + (可选) pino-pretty 传输目标
4. 三个实例存储在 `PinoLoggers` 对象中，赋值给模块级 `loggers` 变量
5. 三个 Proxy 包装的导出对象 (`logger`、`coreLogger`、`errorLogger`) 通过延迟 getter 访问真实 pino 实例

相关源文件：
- `packages/ee-core/src/log/index.ts`：模块入口，提供 `loadLog()`、`getLoggers()` 和三个 Proxy 包装的导出实例
- `packages/ee-core/src/log/logger.ts`：`create()` 函数实现，构建三个 pino 实例

### 调用风格处理流程

`createLoggerProxy()` 使用 Proxy 拦截日志方法调用，按以下优先级处理多参数场景：

```
调用 logger.info(...args)
  │
  ├─ args.length <= 1 → 直接调用 pino 方法
  │
  ├─ 第一个参数是 plain object（非 Error） → pino 标准模式
  │   logger.info({ key: 'val' }, 'msg')
  │
  ├─ 第一个参数包含 %[sdijo] 占位符 → pino printf 模式
  │   logger.info('count: %d', 42)
  │
  └─ 其他 → 拼接模式
      logger.info('msg:', value) → logger.info('msg: value')
      formatArg(): Error→stack, object→JSON.stringify, other→String
```

相关源文件：`packages/ee-core/src/log/index.ts` 中 `createLoggerProxy()` 和 `formatArg()` 函数

### 时区处理流程

`buildTimezoneTimeFn(timeZone)` 为 pino 提供自定义时间戳函数：

```
timezone = 'UTC' → 快速路径：Date.toISOString() → "2026-06-01T21:42:24.123Z"
timezone = 'Asia/Shanghai' → Intl.DateTimeFormat
  → 格式化为带偏移的 ISO 字符串 → "2026-06-01T21:42:24.123+08:00"
```

相关源文件：`packages/ee-core/src/log/logger.ts` 中 `buildTimezoneTimeFn()` 函数

### 传输目标构建流程

`buildTransportTargets()` 构建日志输出目标列表：

1. **始终包含**：pino-roll 文件目标（按频率轮转，支持大小限制）
2. **开发环境额外添加**：pino-pretty 终端彩色输出目标

相关源文件：`packages/ee-core/src/log/logger.ts` 中 `buildTransportTargets()` 函数

## 关键设计决策

### 1. Proxy 包装而非直接导出 pino 实例

pino 的日志方法签名是 `info(mergeObj?, msg, ...args)` 或 `info(format, ...args)`，对不熟悉 pino 的开发者来说容易出错。Proxy 包装实现了"智能参数处理"，让开发者可以用最自然的 `console.log` 风格写日志，同时保留 pino 标准调用方式的兼容性。

设计权衡：Proxy 带来微小性能开销，但日志是 I/O 密集操作（写文件/终端），Proxy 开销相对可忽略。

### 2. 延迟 getter（Lazy initialization）

导出的 `logger`、`coreLogger`、`errorLogger` 是模块级常量，但内部通过 getter 函数 `_getLoggerBy()` 延迟获取真实 pino 实例。这意味着：
- 模块可以在 `loadLog()` 调用之前被导入（getter 在首次访问时触发自动加载）
- 避免了循环依赖问题（日志模块被 config 模块依赖，但 config 模块也可能使用日志）

### 3. 三个独立日志实例而非单实例

- **logger**：业务日志，开发时可看全部信息
- **coreLogger**：框架日志，name 后缀 `-core`，便于区分来源
- **errorLogger**：固定 `error` 级别，独立写入 `ee-error.log`，生产环境只关注严重问题时直接查看此文件

三个实例各写独立文件，互不干扰，便于运维按需查看。

### 4. 时区自定义而非依赖系统 TZ

pino 默认输出 UTC 时间戳（`Z` 后缀）。`buildTimezoneTimeFn()` 允许配置任意 IANA 时区（如 `Asia/Shanghai`），确保日志文件中的时间始终是配置的时区，不受服务器系统 TZ 影响。UTC 路径使用 `Date.toISOString()` 快速路径保证性能。

### 5. 安全模式（safe: true）

pino 的 `safe` 选项默认开启，确保日志写入失败时不会抛出异常，避免因日志问题导致应用崩溃。

## API 导出

### `packages/ee-core/src/log/index.ts`

| 导出 | 类型 | 说明 |
|------|------|------|
| `EeLogger` | interface | 日志接口，定义 trace/debug/info/warn/error/fatal 六级 + child() |
| `createLog(config?)` | function | 使用自定义配置创建日志实例，返回 PinoLoggers |
| `loadLog()` | function | 使用系统配置加载日志系统，由 boot.ts 调用 |
| `getLoggers()` | function | 获取日志实例集合，未加载时自动触发加载 |
| `logger` | EeLogger | 应用日志实例（Proxy 包装） |
| `coreLogger` | EeLogger | 框架核心日志实例（Proxy 包装） |
| `errorLogger` | EeLogger | 错误日志实例（Proxy 包装，仅 error/fatal 级别） |

### `packages/ee-core/src/log/logger.ts`

| 导出 | 类型 | 说明 |
|------|------|------|
| `PinoLoggers` | interface | 三个 pino.Logger 实例的集合类型 |
| `create(config?)` | function | 创建三个日志实例的核心实现 |

**EeLogger 接口方法**：

```typescript
interface EeLogger {
  trace(...args: unknown[]): void;
  debug(...args: unknown[]): void;
  info(...args: unknown[]): void;
  warn(...args: unknown[]): void;
  error(...args: unknown[]): void;
  fatal(...args: unknown[]): void;
  child(bindings: pino.Bindings): EeLogger;  // 返回新的 Proxy 包装子日志
}
```

**支持的调用风格**：

```typescript
// 1. 单参数：直接调用
logger.info('simple message');

// 2. pino 标准：第一个参数为合并对象
logger.info({ action: 'login', userId: 123 }, 'user logged in');

// 3. pino printf：包含格式占位符
logger.info('request count: %d, latency: %dms', 42, 15);

// 4. 拼接模式：多参数自动拼接
logger.info('connection failed:', err);  // err.stack 被自动提取

// 5. 子日志：绑定上下文
const childLogger = logger.child({ service: 'socket' });
childLogger.info('server started');
```

## 模块关系

```
log/index.ts
  ├─ imports → log/logger.ts (create, PinoLoggers)
  ├─ imports → pino (类型：Bindings)
  ├─ 被导入 → app/boot.ts (loadLog)
  ├─ 被导入 → exception/index.ts (coreLogger 记录异常)
  ├─ 被导入 → cross/crossProcess.ts (coreLogger 记录进程事件)
  ├─ 被导入 → config/index.ts (getConfig 提供系统配置)
  └─ 被导入 → 全局业务代码 (logger 记录业务日志)

log/logger.ts
  ├─ imports → config/index.ts (getConfig 获取系统配置)
  ├─ imports → config/default_config.ts (默认日志配置)
  ├─ imports → utils/extend.ts (extend 合并配置)
  ├─ imports → ps/index.ts (getLogDir, isDev)
  ├─ imports → pino (核心日志库)
  └─ imports → pino-roll (文件轮转传输)
  └─ imports → pino-pretty (终端彩色输出)
```

### 配置项

日志配置位于 `config.default.js` 的 `logger` 节，可通过 `config.local.js` / `config.prod.js` 覆盖：

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `dir` | string | getLogDir() | 日志文件目录 |
| `level` | string | 'info' | 日志级别（大小写不敏感） |
| `appLogName` | string | 'ee.log' | 应用日志文件名 |
| `coreLogName` | string | 'ee-core.log' | 框架核心日志文件名 |
| `errorLogName` | string | 'ee-error.log' | 错误日志文件名 |
| `prettyPrint` | boolean | isDev() | 是否启用终端彩色输出 |
| `rotator` | string | 'day' | 轮转策略：'day'/'hour' |
| `maxSize` | string/number | '10m' | 单文件最大大小 |
| `dateFormat` | string | 'yyyy-MM-dd' | 文件名日期格式 |
| `timezone` | string | 'UTC' | 时区（IANA 名称） |
| `redact` | string[] | [] | 脱敏字段路径 |
| `redactCensor` | string | '[Redacted]' | 脱敏替换文本 |
| `name` | string | 'ee' | 日志器名称 |
| `safe` | boolean | true | 安全模式（写入失败不抛异常） |
| `enabled` | boolean | true | 是否启用日志 |
| `serializers` | object | {} | 自定义序列化器 |
| `customLevels` | object | {} | 自定义日志级别 |
