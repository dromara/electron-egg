# 04. Electron 管理（app创建、窗口策略、页面加载）

## 概述

Electron 管理模块是 ee-core 框架的终端启动环节，负责 Electron 应用的完整生命周期管理——从单实例锁定、窗口创建、页面加载策略到应用退出清理。该模块在框架启动流程 `Application.run()` 中作为最后一步被调用（`loadElectron()`），确保配置、日志、控制器、通信服务等先完成初始化，再启动 Electron 界面。

模块由两个子模块组成：
- **app 子模块**（`packages/ee-core/src/electron/app/index.ts`）：应用生命周期管理，单实例锁定，系统事件注册
- **window 子模块**（`packages/ee-core/src/electron/window/index.ts`）：窗口创建，页面加载策略，开发/生产环境适配

入口文件 `packages/ee-core/src/electron/index.ts` 整合两者，导出 `loadElectron()` 函数。

## 核心流程

### createElectron() 应用创建流程

源文件：`packages/ee-core/src/electron/app/index.ts`

```
createElectron()
  │
  ├─ 1. 读取 singleLock 配置
  │     └─ electronApp.requestSingleInstanceLock()
  │     ├─ [singleLock=true 且未获得锁] → electronApp.quit() 退出（第二个实例）
  │     └─ [获得锁 / singleLock=false] → 继续
  │
  ├─ 2. electronApp.whenReady().then()
  │     ├─ createMainWindow()               创建主窗口
  │     ├─ eventBus.emitLifecycle(Preload)   发出 Preload 生命周期事件
  │     ├─ loadServer()                      加载页面（异步，含等待逻辑）
  │     └─ eventBus.emitLifecycle(ElectronAppReady)  发出 AppReady 生命周期事件
  │
  ├─ 3. 注册 window-all-closed 事件
  │     └─ 非 macOS → electronApp.quit()
  │     └─ macOS → 保持运行（符合 macOS 使用习惯）
  │
  └─ 4. 注册 before-quit 事件
       ├─ setCloseAndQuit(true)              设置退出标志
       ├─ eventBus.emitLifecycle(BeforeClose) 发出 BeforeClose 生命周期事件
       ├─ cross.killAll()                     终止所有跨进程服务（Go/Python 等）
       └─ killAllJobs()                       终止所有子任务进程
```

**关键细节**：
- `singleLock` 配置为 `true` 时，通过 `electronApp.requestSingleInstanceLock()` 防止同时运行多个实例。第二个实例启动时直接 `quit()`。
- `before-quit` 中的清理顺序是：先终止跨进程服务（`cross.killAll()`），再终止子任务（`killAllJobs()`），确保所有外部进程不会泄漏。
- macOS 下 `window-all-closed` 不退出应用，符合系统惯例（用户可通过 Cmd+Q 退出）。

### 页面加载策略（loadServer）

源文件：`packages/ee-core/src/electron/window/index.ts`

`loadServer()` 根据配置和环境选择四种加载策略：

```
loadServer()
  │
  ├─ [策略1] Remote 模式 (remote.enable=true)
  │   └─ loadMainUrl('remote', remote.url)
  │     直接加载远程 URL，无等待逻辑
  │
  ├─ [策略2] 开发环境 (isDev())
  │   ├─ 加载 cmd/bin.js 获取开发配置
  │   ├─ 合并默认前端配置 (protocol/host/port/directory/indexPath)
  │   ├─ 判断协议类型：
  │   │   ├─ file:// 协议 → 直接加载本地文件，无需等待
  │   │   └─ http:// 协议 → 进入等待流程：
  │   │       ├─ [可选] 显示 loadingPage（配置了 electron.loadingPage 时）
  │   │       ├─ waitForUrl() 轮询前端服务：
  │   │       │   ├─ force=true → 最多 3 次
  │   │       │   └─ force=false/未设置 → 最多 60 次（每次间隔1秒，超时1秒）
  │   │       ├─ [成功] loadMainUrl('spa', url, 'url')
  │   │       └─ [失败且非force] → 加载 failure.html 失败页面
  │   └─ loadMainUrl('spa', url, load方式)
  │
  ├─ [策略3] 生产环境 + cross takeover
  │   └─ crossTakeover()
  │       ├─ [可选] 显示 loadingPage
  │       ├─ 检查 cross.${service} 配置是否正确（enable=true）
  │       ├─ cross.getUrl(entityName) 获取服务 URL
  │       ├─ waitForUrl() 轮询跨进程服务：
  │       │   ├─ 开发环境 → 20次，间隔1秒
  │       │   └─ 生产环境 → 100次，间隔200ms
  │       ├─ [成功] loadMainUrl('spa', url)
  │       └─ [失败] → 加载 cross-failure.html
  │
  └─ [策略4] 生产环境默认
      ├─ 判断 protocol：
      │   ├─ file:// → loadMainUrl('spa', indexPath, 'file')
      │   └─ 其他协议 → loadMainUrl('spa', protocol+indexPath, 'file')
```

### waitForUrl() 轮询机制

源文件：`packages/ee-core/src/electron/window/index.ts`

```ts
async function waitForUrl(url, { retries, intervalMs, timeoutMs })
  while (!ready && count < retries) {
    await sleep(intervalMs)
    try { await axios.get(url, { timeout: timeoutMs, proxy: false }) → ready = true }
    catch { 继续重试 }
    count++
  }
  return ready
```

核心特点：
- 使用 axios GET 请求检测服务是否就绪，禁用代理（`proxy: false`）
- 每次请求前先等待 `intervalMs`（给服务启动时间）
- 开发环境前端等待：默认 60 次 x 1 秒间隔 = 最多等待 60 秒；`force=true` 时仅 3 次
- 跨进程服务等待：开发 20 次 x 1 秒 = 20 秒；生产 100 次 x 200ms = 20 秒

### loadMainUrl() 页面加载实现

源文件：`packages/ee-core/src/electron/window/index.ts`

```ts
function loadMainUrl(type, url, load = 'url')
  │
  ├─ 记录日志（env、type、url）
  ├─ 注册 did-finish-load 一次性事件：
  │   ├─ openDevTools 为对象 → openDevTools(options)
  │   ├─ openDevTools 为 true → openDevTools({ mode: 'bottom' })
  │   └─ 使用 once 确保只在最终页面加载完成后打开 DevTools
  │
  └─ 执行加载：
      ├─ load='file' → win.loadFile(url, mainServer.options)
      └─ load='url'  → win.loadURL(url, mainServer.options)
```

**DevTools 策略**：使用 `once('did-finish-load')` 而非 `on`，确保 DevTools 只在最终页面（非中间过渡页如 loadingPage）加载完成后打开，避免中间页面跳转时 DevTools 断连。

## 关键设计决策

### 1. 单实例锁定而非多实例窗口激活

`createElectron()` 在第二个实例启动时直接 `quit()`，而不是激活已有实例的窗口。这是简化设计——单实例锁由框架自动处理，业务层如需"激活已有窗口"行为，可通过监听 Electron 的 `second-instance` 事件自行调用 `restoreMainWindow()`。

### 2. Instance 模块级单例而非类实例

窗口模块使用模块级 `Instance` 对象存储 `mainWindow` 和 `closeAndQuit`，而非用类实例。这是因为窗口在应用生命周期中只有一个，模块级变量更简洁，且导出的函数可直接调用（无需先获取实例）。

### 3. 页面加载策略的渐进等待设计

开发环境采用"先显示 loadingPage + 轮询等待"策略而非"阻塞等待后加载"，原因：
- 避免用户看到长时间空白窗口
- loadingPage 提供视觉反馈（如"正在启动..."提示）
- `force=true` 选项允许快速失败（3次轮询），适合已知服务会快速启动的场景

### 4. closeAndQuit 标志的存在意义

`closeAndQuit` 默认为 `true`，在 `before-quit` 中被设为 `true`。此标志供业务层使用——业务代码可在某些场景下将其设为 `false`，使窗口关闭时不退出应用（如"最小化到托盘"场景）。框架在 `before-quit` 中重置为 `true` 确保最终退出时所有清理逻辑执行。

### 5. 跨进程接管（cross takeover）的独立策略

生产环境下 `mainServer.takeover` 配置允许页面由 Go/Python 等跨进程服务提供。这与 Remote 模式不同——Remote 直接加载已知 URL，而 takeover 需要先等待服务就绪再获取 URL。此设计支持将前端页面托管在本地后端服务上。

## API 导出

### electron/app/index.ts

| 导出 | 类型 | 说明 |
|------|------|------|
| `createElectron()` | `function(): void` | 创建并启动 Electron 应用 |
| `electronApp` | `Electron.App` | 原生 Electron app 模块引用 |

### electron/window/index.ts

| 导出 | 类型 | 说明 |
|------|------|------|
| `createMainWindow()` | `function(): BrowserWindow` | 创建主窗口，发出 WindowReady 事件 |
| `getMainWindow()` | `function(): BrowserWindow \| null` | 获取主窗口实例 |
| `restoreMainWindow()` | `function(): void` | 恢复/聚焦主窗口（用于单实例激活） |
| `setCloseAndQuit(flag)` | `function(boolean): void` | 设置关闭即退出标志 |
| `getCloseAndQuit()` | `function(): boolean` | 获取关闭即退出标志 |
| `loadServer()` | `async function(): void` | 根据策略加载页面内容 |

### electron/index.ts（整合入口）

| 导出 | 类型 | 说明 |
|------|------|------|
| `loadElectron()` | `function(): void` | 调用 `createElectron()`，启动 Electron |
| `createElectron()` | 从 app 子模块导出 | |
| `electronApp` | 从 app 子模块导出 | |
| `getMainWindow` 等 | 从 window 子模块导出 | |

## 模块关系

```
electron/index.ts
  ├─ electron/app/index.ts
  │     ├─ 引用 electron/window → createMainWindow, setCloseAndQuit, loadServer
  │     ├─ 引用 app/events → eventBus, Preload, ElectronAppReady, BeforeClose
  │     ├─ 引用 config → getConfig
  │     ├─ 引用 cross → cross.killAll()
  │     ├─ 引用 jobs/registry → killAllJobs()
  │     └─ 引用 log → coreLogger
  │
  └─ electron/window/index.ts
        ├─ 引用 config → getConfig
        ├─ 引用 app/events → eventBus, WindowReady
        ├─ 引用 ps → env, isDev, getBaseDir
        ├─ 引用 log → coreLogger
        ├─ 引用 loader → loadFile（加载 cmd/bin.js 开发配置）
        ├─ 引用 utils → isFileProtocol
        ├─ 引用 utils/helper → fileIsExist, sleep
        ├─ 引用 utils/extend → extend（合并前端默认配置）
        ├─ 引用 cross → cross.getUrl(), cross.killAll()
        ├─ 引用 html → getHtmlFilepath（获取 failure/cross-failure 页面路径）
        └─ 引用 axios（外部依赖，用于 waitForUrl 轮询）

上游调用者：
  Application.run() → loadElectron() → createElectron()

下游依赖：
  - config 模块：提供 singleLock, windowsOption, remote, mainServer 等配置
  - cross 模块：提供跨进程服务 URL 和 killAll()
  - jobs/registry 模块：提供 killAllJobs()
  - app/events 模块：提供生命周期事件发射
