# 工具集

## 概述

ee-core 的工具集模块（utils）提供框架各模块和业务代码共用的基础工具函数，涵盖对象合并、环境检测、JSON 文件操作、路径转换、通用辅助、命令行解析、类型检查、公网 IP 查询和端口分配九个子模块。

各子模块职责分明：
- **extend**：对象深/浅合并，配置分层合并的核心实现
- **is**：Electron 运行时环境检测（进程类型、操作系统、架构）
- **json**：JSON 文件读写，自动目录创建和格式化
- **wrap**：文件路径到属性路径转换，控制器/配置注册表的命名规则
- **helper**：通用辅助函数（防抖、休眠、版本比较、文件操作、参数处理）
- **pargv**：命令行参数解析器，支持短选项、长选项、别名、嵌套属性
- **type_check**：运行时类型守卫（函数、类、对象、原始类型判断）
- **ip**：公网 IPv4/IPv6 地址查询（DNS/HTTPS）
- **port**：可用 TCP 端口分配，双层锁防并发冲突

## 核心流程

### extend 深合并流程

```
extend(deep, target, ...sources)
  │
  ├─ 参数解析
  │     deep: boolean → 直接使用
  │     deep: ExtendOptions → deep.deep 提取布尔值
  │
  ├─ 逐个遍历 source 对象
  │     source 为 null/undefined → 跳过
  │
  └─ 逐个遍历 source 的 key
      ├─ key === '__proto__' → 跳过（防原型链污染）
      ├─ val === target → 跳过（防自引用无限递归）
      ├─ deep=true && isPlainObject(val) → 递归合并
      │     target[key] 是 plain object → 在原对象上递归合并
      │     target[key] 不是 plain object → 创建新 {} 再合并
      ├─ val !== undefined → 直接覆盖
      └─ val === undefined → 跳过（不覆盖已有属性）
```

相关源文件：`packages/ee-core/src/utils/extend.ts` 中 `extend()` 和 `isPlainObject()` 函数

### isPlainObject 判断流程

```
isPlainObject(obj)
  │
  ├─ obj 为空 或 toString !== '[object Object]' → false
  │
  ├─ constructor 存在 且不是自有属性 且 prototype 无 isPrototypeOf → false
  │   （排除 Date, RegExp, Buffer 等内置类型实例和自定义类实例）
  │
  └─ 最后可枚举属性是自有属性 → true
     （利用 JS 引擎先遍历自有属性再遍历继承属性的优化）
```

相关源文件：`packages/ee-core/src/utils/extend.ts` 中 `isPlainObject()` 函数

### wrap 路径转换流程

```
getProperties(filepath, { caseStyle })
  │
  ├─ caseStyle 是函数 → 委托自定义转换逻辑
  │     返回值必须是数组，否则抛异常
  │
  └─ caseStyle 是字符串 → 使用 defaultCamelize()
     ├─ 移除扩展名，按 '/' 分割为路径段
     ├─ 每段：下划线/连字符转驼峰 (foo_bar → fooBar)
     ├─ 验证：仅允许 a-z0-9_- 字符
     └─ 根据 caseStyle 调整首字母：
        │ 'lower' → 首字母小写 (FooBar → fooBar)
        │ 'upper' → 首字母大写 (fooBar → FooBar)
        │ 'camel' → 保持原样
```

相关源文件：`packages/ee-core/src/utils/wrap.ts` 中 `getProperties()` 和 `defaultCamelize()` 函数

### pargv 解析流程

```
parseArgv(args, opts)
  │
  ├─ 1. 初始化阶段：构建标志映射
  │     bools: boolean 类型参数标记
  │     strings: string 类型参数标记
  │     aliases: 双向别名映射
  │
  ├─ 2. 分割阶段：以 '--' 分割选项和非选项
  │
  ├─ 3. 逐项解析阶段：
  │     --key=value → 长选项等号赋值
  │     --no-key   → 否定形式 (boolean false)
  │     --key value → 长选项带值
  │     -abc       → 短选项组合 (每个字母独立选项)
  │     bare value → 存入 argv._ 数组
  │
  ├─ 4. 默认值填充：为未出现的键填充默认值
  │
  └─ 5. 非选项处理：
       opts['--']=true → argv['--'] 数组
       opts['--']=false → argv._ 数组
```

相关源文件：`packages/ee-core/src/utils/pargv.ts` 中 `parseArgv()` 函数

### port 分配流程

```
getPort(options?)
  │
  ├─ 1. 惰性创建旋转定时器（首次调用时）
  │     每 15 秒：old = young, young = new Set()
  │     定时器 unref() 防止阻止进程退出
  │
  ├─ 2. 构建端口检查序列
  │     指定端口 → 逐个尝试
  │     无指定 → 端口 0 (OS 自动分配)
  │
  ├─ 3. 逐个端口检查可用性
  │     创建临时 TCP server → 尝试 listen
  │     成功 → 端口可用
  │     失败 → EADDRINUSE/EACCES → 尝试下一个
  │
  ├─ 4. 检查端口是否被锁定
  │     lockedPorts.old 或 young 中 → 已分配
  │     指定端口被锁 → 抛 Locked 异常
  │     OS 分配端口被锁 → 重新获取
  │
  └─ 5. 加入 young 锁定集 → 返回可用端口
```

相关源文件：`packages/ee-core/src/utils/port/index.ts` 中 `getPort()` 函数

### ip 查询流程

```
publicIpv4(options?) / publicIpv6(options?)
  │
  ├─ type='http' → queryHttps()
  │     逐个尝试 icanhazip / ipify HTTPS 端点
  │     返回 IP 字符串，验证格式
  │     支持自定义 fallbackUrls
  │
  ├─ type='dns' → queryDns()
  │     使用 dns-socket 查询 OpenDNS
  │     查询 myip.opendns.com 的 A(v4)/AAAA(v6) 记录
  │     逐个尝试多个 OpenDNS 服务器
  │     验证返回的 IP 格式
  │
  └─ type='all' → queryAll()
     先尝试 DNS → 失败则回退 HTTPS
     确保受限网络环境也能获取公网 IP
```

相关源文件：`packages/ee-core/src/utils/ip.ts` 中 `publicIpv4()`、`publicIpv6()` 函数

### type_check 判断流程

```
isClass(val) 判断 ES6 类
  │
  ├─ typeof val !== 'function' → false
  │
  ├─ 结构性检查（主要方式）
  │     Object.getOwnPropertyDescriptor(val, 'prototype')
  │     writable === false → ES6 class（class 的 prototype 不可写）
  │
  └─ 字符串检查（回退方式）
     /^\s*class\b/.test(val.toString())
     兼容结构性检查遗漏的边界情况
```

相关源文件：`packages/ee-core/src/utils/type_check.ts` 中 `isClass()` 函数

### helper 核心函数流程

```
fnDebounce() 防抖工厂
  │
  ├─ 创建 Map<fn, {delayTime, timer}> 存储定时器
  │
  └─ 返回防抖包装函数
     ├─ delayTime=0 或 isImedite=true → 立即执行，不防抖
     ├─ 已有定时器 → clearTimeout + 重新设置（核心防抖逻辑）
     └─ 定时器触发 → 执行函数 + 从 Map 删除引用

systemSleep(ms) 同步阻塞休眠
  │
  ├─ 类型检查：ms 必须是 number，范围 [0, Infinity)
  │
  ├─ SharedArrayBuffer + Atomics 可用 → sleepAtomic()
  │     Atomics.wait(new Int32Array(new SharedArrayBuffer(4)), 0, 0, ms)
  │     零 CPU 消耗，真正线程阻塞，超时自动解除
  │
  └─ 不可用 → sleepNaive() while 循环忙等待
     CPU 自旋但同样实现阻塞效果

replaceArgsValue(argv, key, value) 参数值替换
  │
  ├─ 查找 key= 格式的参数
  │
  └─ 替换值部分，保留 key= 前缀
     ['--port=8080', '--env=dev'], 'port', '9090'
     → ['--port=9090', '--env=dev']
```

相关源文件：`packages/ee-core/src/utils/helper.ts` 中各函数

## 关键设计决策

### 1. extend 的原型链污染防护

`extend()` 明确跳过 `__proto__` 属性，防止通过对象合并注入恶意属性到原型链。这是安全编码的必要措施，因为 `{ __proto__: { admin: true } }` 这种恶意输入在不加防护的 merge 函数中会影响所有 Object 实例。

### 2. extend 的 undefined 值跳过

`undefined` 值不覆盖 target 已有属性。这避免了配置合并中的常见问题：`extend(true, { port: 8080 }, { port: undefined })` 应保留 `port: 8080` 而非变成 `undefined`。

### 3. extend 的自引用检测

`target === val` 时跳过合并，防止 `extend(true, obj, obj)` 导致无限递归。这在复杂配置合并场景中可能出现。

### 4. isClass 的结构性检测优先于字符串检测

`isClass()` 首先检查 `prototype.writable === false`（结构性特征），其次才用 `toString()` 检查源文本。这确保：
- 代码经过 minify/混淆后仍能正确识别类
- 字节码(.jsc)编译后的类也能正确识别
- 构造函数用 `function` 声明的不被误判为类（这是有意设计）

### 5. is 模块的组合器（all/none/one）

`all()`、`none()`、`one()` 接受多个条件函数，支持组合判断如 `all(main, osx)`（主进程 + macOS）。每个条件函数延迟执行，短路求值——`all()` 在第一个 false 时返回，`one()` 在第一个 true 时返回，避免不必要的检测调用。

### 6. port 的双层锁旋转机制

端口分配使用 old + young 两个 Set：
- young：最近分配的端口
- old：即将释放的端口
- 每 15 秒旋转：young → old，old 被清空释放

双层设计确保端口至少被锁定 15 秒（一次旋转周期），防止同一端口在短时间内被连续分配给不同服务。`unref()` 确保旋转定时器不阻止进程退出。

### 7. pargv 的原型链污染防护

`setKey()` 在设置嵌套属性时跳过 `constructor` 和 `__proto__`，防止通过命令行参数注入原型链。`isConstructorOrProto()` 检查确保 `--__proto__.admin=true` 这类恶意参数被安全忽略。

### 8. systemSleep 的双策略实现

优先使用 `Atomics.wait`（零 CPU、真正阻塞），不可用时回退到 while 循环忙等待。`Atomics.wait` 基于 SharedArrayBuffer，在 Node.js 环境中可用，但在某些受限环境（如 renderer 进程）可能不支持。

### 9. ip 查询的 DNS + HTTPS 组合模式

`type='all'` 先 DNS 后 HTTPS 的设计确保：
- 正常网络环境下 DNS 查询更快（无 HTTP 连接开销）
- DNS 受限的网络环境（如防火墙阻止 UDP 53）仍可通过 HTTPS 获取 IP
- 所有 Promise 附带 `cancel()` 方法，允许随时取消查询

## 模块关系

```
utils/extend.ts
  ├─ imports → types/index.ts (ExtendOptions 类型)
  ├─ 被导入 → config/configLoader.ts (配置分层合并)
  ├─ 被导入 → cross/cross.ts (运行时选项合并)
  ├─ 被导入 → log/logger.ts (日志配置合并)
  └─ 被导入 → electron/app/index.ts (窗口配置合并)

utils/is.ts
  ├─ 被导入 → ps/index.ts (平台差异路径处理)
  ├─ 被导入 → cross/crossProcess.ts (Windows 平台判断)
  ├─ 被导入 → electron/app/index.ts (平台特定逻辑)
  └─ 可独立导入 → 业务代码 (环境判断)

utils/json.ts
  ├─ 被导入 → storage/sqliteStorage.ts (配置文件读写)
  └─ 可独立导入 → 业务代码 (JSON 操作)

utils/wrap.ts
  ├─ imports → utils/type_check.ts (isFunction 判断自定义 caseStyle)
  ├─ 被导入 → core/fileLoader.ts (控制器/配置路径到属性名转换)
  └─ 被导入 → plugins/controller_registry_plugin (构建注册表属性路径)

utils/helper.ts
  ├─ imports → utils/pargv.ts (parseArgv 参数解析)
  ├─ 被导入 → cross/cross.ts (getValueFromArgv, replaceArgsValue)
  ├─ 袍导入 → cross/crossProcess.ts (getValueFromArgv, getRandomString)
  └─ 袍导入 → 全局业务代码 (通用工具函数)

utils/pargv.ts
  ├─ 袍导入 → utils/helper.ts (getValueFromArgv 内部使用)
  ├─ 袍导入 → cross/crossProcess.ts (getArgsObj 参数解析)

utils/type_check.ts
  ├─ 袍导入 → loader/index.ts (isFunction, isClass 加载行为判断)
  └─ 袍导入 → utils/wrap.ts (isFunction 自定义 caseStyle 判断)

utils/ip.ts
  ├─ imports → dgram (UDP Socket)
  ├─ imports → dns-socket (DNS 查询)
  ├─ imports → urllib (HTTPS 请求)
  └─ 可独立导入 → 业务代码 (公网 IP 查询)

utils/port/index.ts
  ├─ imports → net (TCP 临时服务器)
  ├─ imports → os (网络接口信息)
  ├─ 袍导入 → cross/cross.ts (动态端口分配)
  └─ 袍导入 → socket/ (服务端口分配)
```
