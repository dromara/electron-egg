# 跨进程通信

## 概述

ee-core 的跨进程通信模块（cross）负责创建、管理和终止外部子进程（如 Go/Python 后端），并为它们动态分配端口。模块基于 `cross-spawn` 库封装子进程生命周期，基于 `tree-kill` 实现优雅退出，通过 EventEmitter 事件机制自动清理已终止的进程索引。

核心设计思路：
- **Cross 管理器**：进程生命周期总控，维护 pid 和 name 双索引，支持按 pid/name 查询和终止
- **CrossProcess 子进程实例**：封装单个外部进程的完整生命周期（创建→运行→监听→终止），处理路径解析、平台差异、端口分配

## 核心流程

### 启动流程

跨进程通信在框架启动流程中由 `application.ts` 调用 `cross.create()` 初始化：

```
Boot → loadConfig → loadLog → loadDir → loadController → loadSocket → loadElectron
                                         ↕ loadCross (在 loadElectron 内)
```

1. `cross.create()` 读取 `config.cross` 配置
2. 遍历每个服务配置，对 `enable=true` 的服务调用 `run()`
3. `run()` 合并配置、分配端口、创建 CrossProcess
4. 注册到 `children`（pid 索引）和 `childrenMap`（name 索引）

相关源文件：`packages/ee-core/src/cross/cross.ts` 中 `create()` 和 `run()` 方法

### 子进程创建流程（run）

```
cross.run(service, opt)
  │
  ├─ 1. 从 config.cross 获取服务配置
  │     合并默认配置与运行时选项 (extend deep merge)
  │
  ├─ 2. 解析端口参数
  │     getValueFromArgv(tmpArgs, 'port')
  │     confPort > 0 → getPort({ port: confPort }) 动态分配可用端口
  │     replaceArgsValue(tmpArgs, 'port', String(confPort)) 替换参数中的端口值
  │
  ├─ 3. 创建 CrossProcess
  │     new CrossProcess(this, { targetConf, port: confPort })
  │     _init() 内部：
  │       ├─ 解析可执行文件路径
  │       ├─ 设置 stdio 模式（dev: inherit, prod: ignore）
  │       ├─ cross-spawn 启动子进程
  │       ├─ 监听 exit/error 事件 → 通知 Cross 管理器
  │       └─ 记录 pid 和 name
  │
  ├─ 4. 注册双索引
  │     childrenMap[uniqueName] = pid
  │     children[pid] = { name, entity }
  │     重名服务追加 pid 后缀避免冲突
  │
  └─ 返回 CrossProcess 实例
```

相关源文件：
- `packages/ee-core/src/cross/cross.ts` 中 `run()` 方法
- `packages/ee-core/src/cross/crossProcess.ts` 中 `_init()` 方法

### 路径解析流程（CrossProcess._init）

```
解析可执行文件路径
  │
  ├─ 配置有 cmd → 使用 cmd 作为路径
  │   ├─ 非绝对路径 + 非开发环境 → 拼接 extraResourcesDir
  │   └─ 需要同时配置 directory 属性
  │
  ├─ 配置无 cmd → 使用 extraResourcesDir/{name} 作为路径
  │
  ├─ Windows 平台 → 自动追加 .exe 扩展名
  │   (windowsExtname=true 或 非开发环境时)
  │
  ├─ 解析工作目录 (cwd)
  │   ├─ directory 为绝对路径 → 直接使用
  │   ├─ directory 为相对路径 → 开发环境拼接 baseDir，生产拼接 extraResourcesDir
  │   └─ 无 directory → 使用 extraResourcesDir
  │
  └─ cross-spawn(cmdPath, cmdArgs, { stdio, cwd, detached: false })
```

相关源文件：`packages/ee-core/src/cross/crossProcess.ts` 中 `_init()` 方法

### 进程终止流程（kill）

```
CrossProcess.kill(timeout)
  │
  ├─ tree-kill(pid, SIGINT) → 优雅终止（发送中断信号）
  │   ├─ 成功 → 进程自行退出，触发 exit 事件
  │   └─ 失败 → tree-kill(pid, SIGKILL) 强制终止
  │
  └─ setTimeout(timeout) 安全网
      进程未退出 → _exitElectron() 触发主进程退出
```

相关源文件：`packages/ee-core/src/cross/crossProcess.ts` 中 `kill()` 方法

### 事件清理流程

```
子进程 exit/error 事件
  │
  ├─ CrossProcess → host.emitter.emit(Events.childProcessExit/Error, { pid })
  │
  └─ Cross._initEventEmitter() 监听
      ├─ 从 children[pid] 获取 name
      ├─ delete childrenMap[name]
      └─ delete children[pid]
```

相关源文件：`packages/ee-core/src/cross/cross.ts` 中 `_initEventEmitter()` 方法

## 关键设计决策

### 1. 双索引（pid + name）而非单索引

- `children`：pid → { name, entity }，按进程 ID 快速查找
- `childrenMap`：name → pid，按服务名快速查找和获取 URL

双索引设计使得 `kill(pid)`、`killByName(name)`、`getUrl(name)` 等操作都能 O(1) 完成，同时 name 索引支持按服务名语义管理（更符合业务需求）。

### 2. 重名服务追加 pid 后缀

当同名服务被多次 `run()` 时，自动追加 `-pid` 后缀（如 `goServer-12345`），避免 `childrenMap` 中的 name 冲突。这支持同一服务的多实例运行场景。

### 3. 优雅终止（SIGINT → SIGKILL）

先发送 SIGINT 信号让外部程序有机会清理资源（关闭数据库连接、保存状态等），仅当 SIGINT 失败时才使用 SIGKILL 强制终止。`tree-kill` 库确保整个进程树（包括子进程的子进程）都被终止。

### 4. appExit 配置：子进程退出联动主进程

当外部后端（Go/Python）崩溃时，如果配置 `appExit=true`，Electron 主进程会在延迟后自动退出。这是"关键服务不可用则应用不可用"的设计理念，避免前端连接不到后端后呈现半死状态。

### 5. 动态端口分配而非固定端口

跨进程服务的端口通过 `getPort()` 动态分配，避免端口冲突（特别是多实例场景）。分配后通过 `replaceArgsValue()` 将实际端口注入启动参数，外部程序启动时读取参数绑定到分配的端口。

### 6. stdio 模式按环境切换

- 开发环境：`inherit`（子进程输出直接显示在终端，便于调试）
- 生产环境：`ignore`（静默运行，不污染终端输出）
- 可通过配置 `stdio` 字段自定义

## API 导出

### `packages/ee-core/src/cross/cross.ts`

| 导出 | 类型 | 说明 |
|------|------|------|
| `Cross` | class | 跨进程管理器类 |
| `CrossRunOptions` | interface | run() 方法的运行时选项 |
| `cross` | Cross 单例 | 全局跨进程管理器实例 |

**Cross 类方法**：

| 方法 | 返回值 | 说明 |
|------|--------|------|
| `create()` | Promise<void> | 启动所有 enable=true 的跨进程服务 |
| `run(service, opt?)` | Promise<CrossProcess> | 运行指定服务，返回子进程实例 |
| `killAll()` | void | 终止所有子进程 |
| `kill(pid)` | void | 按 pid 终止子进程 |
| `killByName(name)` | void | 按服务名终止子进程 |
| `getUrl(name)` | string \| undefined | 获取指定服务的 URL |
| `getProcByName(name)` | CrossProcess | 按名称获取子进程实例 |
| `getProc(pid)` | CrossProcess | 按 pid 获取子进程实例 |
| `getPids()` | string[] | 获取所有子进程 pid 列表 |

### `packages/ee-core/src/cross/crossProcess.ts`

| 导出 | 类型 | 说明 |
|------|------|------|
| `CrossProcess` | class | 子进程实例类 |
| `CrossProcessOptions` | interface | 创建 CrossProcess 的选项 |
| `CrossHost` | interface | 主机接口（需包含 emitter 字段） |
| `CrossTargetConfig` | type | 跨进程目标服务配置类型（从 types 重导出） |

**CrossProcess 类属性和方法**：

| 属性/方法 | 类型 | 说明 |
|-----------|------|------|
| `emitter` | EventEmitter | 事件发射器 |
| `pid` | number | 子进程 PID |
| `port` | number | 分配的端口 |
| `name` | string | 服务名称 |
| `config` | CrossTargetConfig | 服务配置 |
| `getUrl()` | string | 获取服务 URL（http/https://host:port） |
| `getArgsObj()` | Record<string, unknown> | 获取解析后的启动参数对象 |
| `setPort(port)` | void | 设置端口 |
| `kill(timeout?)` | void | 终止子进程（SIGINT → SIGKILL） |
| `_generateId()` | string | 生成唯一 ID（node:pid:random） |

## 模块关系

```
cross/cross.ts
  ├─ imports → config/index.ts (getConfig 获取跨进程配置)
  ├─ imports → utils/helper.ts (getValueFromArgv, replaceArgsValue)
  ├─ imports → utils/extend.ts (extend 合并配置与运行时选项)
  ├─ imports → utils/port/index.ts (getPort 动态端口分配)
  ├─ imports → cross/crossProcess.ts (CrossProcess, CrossHost)
  ├─ imports → const/channel.ts (Events 事件常量)
  ├─ imports → types/index.ts (CrossTargetConfig, ProcessExitEventData)
  ├─ 被导入 → app/application.ts (cross.create 启动跨进程服务)
  └─ 被导入 → socket/ipcServer.ts (cross.getUrl 获取服务 URL)

cross/crossProcess.ts
  ├─ imports → log/index.ts (coreLogger 记录进程事件)
  ├─ imports → ps/index.ts (getExtraResourcesDir, isPackaged, isDev, getBaseDir)
  ├─ imports → utils/helper.ts (getValueFromArgv, getRandomString)
  ├─ imports → utils/pargv.ts (parseArgv 解析启动参数)
  ├─ imports → utils/is.ts (is.windows 平台判断)
  ├─ imports → electron/app/index.ts (electronApp.quit 主进程退出)
  ├─ imports → cross-spawn (跨平台进程创建)
  ├─ imports → tree-kill (进程树终止)
  └─ imports → const/channel.ts (Events 事件常量)
```

### 配置项

跨进程配置位于 `config.default.js` 的 `cross` 节：

```javascript
cross: {
  goServer: {
    enable: true,           // 是否启用
    name: 'goServer',       // 服务名称
    cmd: 'go_server',       // 可执行文件名（可选）
    directory: 'go',        // 工作目录
    args: ['--port=7070'],  // 启动参数
    appExit: false,         // 子进程退出是否联动主进程退出
    windowsExtname: true,   // Windows 下自动追加 .exe
    stdio: undefined,       // 自定义 stdio 配置
  },
  pythonServer: {
    enable: true,
    name: 'pythonServer',
    cmd: 'python_server',
    directory: 'python',
    args: ['--port=7071', '--ssl=false'],
  }
}
```
