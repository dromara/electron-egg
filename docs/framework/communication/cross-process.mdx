Cross 模块是 electron-egg 框架的跨进程通信管理器，负责创建、管理和终止外部子进程（如 Go/Python 后端），并动态分配端口。结合 SocketServer 和 IPC 机制，Cross 实现了 Electron 主进程与外部服务的完整协作能力。

## 核心架构

Cross 模块基于双索引管理子进程：

```
children (pid 索引):  { pid → { name, entity } }
childrenMap (name 索引): { name → pid }
```

- **children**：以进程 PID 为键，存储 CrossProcess 实例和名称
- **childrenMap**：以服务名称为键，快速查找进程 PID

<Note>
当服务名称冲突时（同一名称的多个实例），Cross 会自动为重复名称添加 PID 后缀（如 `goServer-12345`），避免索引冲突。
</Note>

## Go/Python 后端启动流程

### 配置定义

```javascript
// config.default.js
module.exports = {
  cross: {
    goServer: {
      enable: true,
      name: 'goServer',
      cmd: 'go_server',          // 可执行文件名（相对于 extraResources 目录）
      directory: 'go',           // 工作目录
      args: ['-port', '7072'],   // 启动参数
      appExit: true,             // 子进程退出时是否同时退出 Electron
    },
    pythonServer: {
      enable: false,
      name: 'pythonServer',
      cmd: 'python_app',
      directory: 'python',
      args: ['-port', '7073'],
      appExit: false,
    },
  },
};
```

### 启动流程

Cross 在应用启动时自动遍历 `config.cross` 配置，对所有 `enable: true` 的服务调用 `run()`：

```
Application.run() → loadSocket() → cross.create()
                                   ↓
                                   遍历 config.cross
                                   ↓
                                   每个 enable=true → cross.run(name)
                                   ↓
                                   解析端口 → 动态分配 → 创建 CrossProcess → 注册索引
```

### 动态端口分配

Cross 支持动态端口分配，避免端口冲突：

```javascript
// run() 方法中的端口处理
// 1. 从启动参数中解析端口值
let confPort = parseInt(getValueFromArgv(args, 'port') || '0'), 10);

// 2. 如果指定了端口，动态获取可用端口
if (confPort > 0) {
  confPort = await getPort({ port: confPort });
  // 3. 替换启动参数中的端口值
  targetConf.args = replaceArgsValue(args, 'port', String(confPort));
}
```

<Note>
动态端口分配确保了多个实例或不同环境下的端口不冲突。实际端口会替换启动参数中的原始端口值，子进程通过解析参数获取实际分配的端口。
</Note>

## 服务接管模式（takeover）

服务接管是 Cross 最强大的功能之一，允许 Electron 应用在窗口中加载外部后端服务提供的 Web 页面。

### 工作流程

```
loadServer() → 检查 mainServer.takeover
              ↓
              显示加载页面（loadingPage）
              ↓
              cross.getUrl(serviceName) → 获取服务 URL
              ↓
              waitForUrl(url) → 轮询检查服务是否就绪
              ↓ 就绪                       ↓ 超时
              loadMainUrl(url)             显示失败页面
```

### 配置示例

```javascript
// config.default.js
module.exports = {
  mainServer: {
    takeover: 'goServer',     // 指定接管服务名称
    loadingPage: '/public/html/loading.html',  // 加载等待页面
    indexPath: 'index.html',
  },
  cross: {
    goServer: {
      enable: true,
      name: 'goServer',
      cmd: 'go_server',
      args: ['-port', '7072', '-ssl', 'false', '-hostname', '127.0.0.1'],
      appExit: true,
    },
  },
};
```

<Warning>
接管服务必须在 cross 配置中设置 `enable: true`，否则应用启动时会抛出错误。确保接管服务的名称与 mainServer.takeover 配置一致。
</Warning>

### URL 构建

CrossProcess 根据启动参数构建服务 URL：

```javascript
getUrl() {
  const ssl = getValueFromArgv(args, 'ssl');
  const hostname = getValueFromArgv(args, 'hostname');
  const protocol = ssl === 'true' || ssl === '1' ? 'https://' : 'http://';
  return protocol + (hostname || '127.0.0.1') + ':' + this.port;
}
```

## 进程生命周期管理

### 创建与运行

```javascript
const { cross } = require('ee-core');

// 启动指定服务
const proc = await cross.run('goServer');

// 获取服务 URL
const url = cross.getUrl('goServer');  // http://127.0.0.1:7072

// 获取进程实例
const entity = cross.getProcByName('goServer');

// 获取进程 PID
const pids = cross.getPids();  // ['12345', '12346']
```

### 终止进程

```javascript
// 终止指定进程
cross.kill(pid);             // 通过 PID 终止
cross.killByName('goServer'); // 通过名称终止

// 终止所有子进程
cross.killAll();
```

<Note>
CrossProcess.kill() 先发送 SIGINT 信号尝试优雅退出，失败后发送 SIGKILL 强制终止。设有超时安全机制，1秒后如果进程仍未退出则触发 _exitElectron。
</Note>

### 应用退出清理

在 Electron `before-quit` 事件中，自动清理所有子进程：

```javascript
// electron/app/index.ts
electronApp.on('before-quit', () => {
  setCloseAndQuit(true);
  eventBus.emitLifecycle(BeforeClose);
  cross.killAll();    // 终止所有跨进程服务
  killAllJobs();      // 终止所有子任务进程
});
```

## appExit 配置

当配置 `appExit: true` 时，子进程退出或出错会触发 Electron 主进程退出：

```javascript
// CrossProcess._exitElectron()
_exitElectron(timeout = 1000): void {
  if (this.config.appExit) {
    setTimeout(() => {
      electronApp.quit();
    }, timeout);
  }
}
```

<Warning>
appExit 适用于关键服务（如核心后端）。当关键服务崩溃时，Electron 主进程也应退出，避免用户在无后端支持的情况下继续操作。非关键服务应设置 appExit: false。
</Warning>

## 与 SocketServer 集成

Cross 启动的 Go/Python 后端可以通过 SocketServer 与主进程通信：

```
Go 后端进程 ←→ SocketServer ←→ Electron 主进程控制器
                  (WebSocket)
```

Go 后端通过 SocketIO 客户端连接 SocketServer，发送 `{ cmd, args }` 格式消息调用主进程控制器方法。

## 与 IPC 集成

跨进程服务的 URL 和状态信息也可以通过 IPC 传递给渲染进程：

```javascript
// 控制器方法示例
class CrossController {
  async getServiceUrl() {
    const { cross } = require('ee-core');
    return cross.getUrl('goServer');
  }
}
```

渲染进程通过 IPC 获取跨进程服务信息：

```javascript
// 前端代码
const url = await ipcRenderer.invoke('controller/cross/getServiceUrl');
```

## CrossProcess 路径解析规则

CrossProcess 根据配置解析子进程可执行文件路径：

| 配置 | 路径规则 | 示例 |
|------|----------|------|
| 有 `cmd` | 使用 cmd 作为路径 | `cmd: 'go_server'` → `extraResources/go_server` |
| 无 `cmd` | 使用 `extraResources/{name}` | `name: 'goServer'` → `extraResources/goServer` |
| 绝对路径 `cmd` | 直接使用 | `cmd: '/usr/local/bin/myapp'` |
| Windows | 自动添加 `.exe` | `go_server` → `go_server.exe` |

<Note>
开发环境中，相对路径基于项目根目录解析。生产环境中，基于 extraResources 目录解析。
</Note>
