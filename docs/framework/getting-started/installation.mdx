## 系统要求

<Steps>
<Step title="Node.js">
版本 **>= 20**。推荐使用 LTS 版本（如 v20.x 或 v22.x）。

```bash
node -v
# 应输出 v20.x.x 或更高
```
</Step>

<Step title="pnpm">
项目**仅支持 pnpm**，不支持 npm 或 yarn。

```bash
pnpm -v
# 应输出 9.x 或更高
```

如未安装 pnpm：

```bash
corepack enable
corepack prepare pnpm@latest --activate
```
</Step>

<Step title="操作系统">
支持 macOS、Windows、Linux。macOS ARM64（M1/M2/M3）需额外注意原生模块编译。
</Step>
</Steps>

## 克隆与安装

<Steps>
<Step title="克隆仓库">

```bash
git clone https://github.com/dromara/electron-egg.git
cd electron-egg
```

</Step>

<Step title="确认 .npmrc 配置">
项目根目录 `.npmrc` 已预置以下配置，一般无需修改：

```ini
# 镜像源
registry=https://registry.npmmirror.com/
disturl=https://registry.npmmirror.com/-/binary/node
electron_mirror=https://npmmirror.com/mirrors/electron/
electron-builder-binaries_mirror=https://registry.npmmirror.com/-/binary/electron-builder-binaries/

# 包管理器
package-manager=pnpm
shamefully-hoist=true
node-linker=hoisted
hardlinks=false

# 常见问题预防
block-exotic-subdeps=false
ignore-build-scripts=false
```

<Note>
`shamefully-hoist=true` 与 `node-linker=hoisted` 是 Electron 兼容性所需，**不要删除**。`block-exotic-subdeps=false` 与 `ignore-build-scripts=false` 防止首次安装被拦截，详见[常见问题](/getting-started/troubleshooting)。
</Note>
</Step>

<Step title="构建 ee-bin（重要）">
ee-bin 是本地 workspace 包，其 `bin` 字段指向构建产物 `./dist/esm/index.js`。**必须先构建 ee-bin 再执行 `pnpm install`**，否则 bin 链接会创建失败。

```bash
cd packages/ee-bin
pnpm run build
cd ../..
```

<Warning>
跳过此步骤会导致 `pnpm install` 时出现 `ENOENT: no such file or directory, open 'packages/ee-bin/dist/esm/index.js'` 错误。详见 [ee-bin 链接失败](/getting-started/troubleshooting)。
</Warning>
</Step>

<Step title="安装依赖">

```bash
pnpm install
```

安装过程中 pnpm 会自动：

1. 链接 workspace 包（ee-core、ee-bin）
2. 下载 Electron 二进制文件（约 100MB+）
3. 编译 better-sqlite3 原生模块
4. 安装 esbuild 平台二进制

首次安装可能需要数分钟，取决于网络速度。
</Step>

<Step title="批准构建脚本（pnpm v9+）">
如果出现 `ERR_PNPM_IGNORED_BUILDS` 错误，需要手动批准关键包的构建脚本：

```bash
pnpm approve-builds better-sqlite3 electron-winstaller electron esbuild
```

批准后重新构建：

```bash
pnpm rebuild
```
</Step>

<Step title="验证安装">

```bash
# 检查 ee-bin 链接
ls node_modules/.bin/ee-bin

# 检查 Electron 二进制
ls node_modules/electron/dist/

# 检查 better-sqlite3 编译
ls node_modules/better-sqlite3/build/
```

以上路径均存在文件则安装成功。
</Step>
</Steps>

## 首次安装常见问题

以下是常见问题的**快速修复**，详细排查步骤请参阅 [pnpm install 常见问题排查指南](/faq/pnpm-troubleshooting)。

| 问题 | 快速修复 |
|------|----------|
| Exotic dependency 被阻止 | `.npmrc` 中添加 `block-exotic-subdeps=false` |
| 镜像源 502 | 重试 `pnpm install`，或临时切换 `--registry https://registry.npmjs.org/` |
| 构建脚本被忽略 | `pnpm approve-builds` → `pnpm rebuild` |
| ee-bin 链接失败 | 先 `cd packages/ee-bin && pnpm run build`，再 `pnpm install` |
| Electron 下载慢 | `.npmrc` 已配置 `electron_mirror`，执行 `pnpm rebuild` |

<Warning>
如果遇到多个问题同时出现，建议按以下顺序修复：1) 确认 `.npmrc` 配置 → 2) 检查 `pnpm-workspace.yaml` → 3) 构建 ee-bin → 4) `pnpm install` → 5) `pnpm rebuild`。完整流程见 [pnnpm 排查指南](/faq/pnpm-troubleshooting)。
</Warning>
