LoadBalancer 是 electron-egg 框架的负载均衡调度器，为 ChildPoolJob 进程池提供 8 种算法选择最优子进程执行任务。基于 electron-re 项目改进，支持动态添加/删除进程目标、权重调整和运行时切换算法。

## 8 种负载均衡算法

| 算法 | 常量名 | 说明 | 适用场景 |
|------|--------|------|---------|
| **轮询** | `polling` | 按顺序依次选择下一个目标 | 所有进程性能相同 |
| **加权** | `weights` | 按权重比例选择 | 某些进程性能更强 |
| **随机** | `random` | 随机选择一个目标 | 无特殊需求 |
| **指定** | `specify` | 选择指定 ID 的目标 | 需要固定到某个进程 |
| **加权轮询** | `weightsPolling` | 结合权重和轮询 | 性能不同且需均匀分配 |
| **加权随机** | `weightsRandom` | 结合权重和随机选择 | 性能不同但无需严格顺序 |
| **最小连接** | `minimumConnection` | 选择当前连接数最少的目标 | 长连接/持续任务场景 |
| **加权最小连接** | `weightsMinimumConnection` | 结合权重和最小连接数 | 性能不同且长连接场景 |

## 算法详解

### polling 轮询

按顺序依次选择进程目标，每次调用 `pickOne()` 后索引递增：

```javascript
// targets: [{ id: 1 }, { id: 2 }, { id: 3 }]
// pickOne() 选择顺序: 1 → 2 → 3 → 1 → 2 → 3 → ...
```

适合所有进程性能相同的简单场景。

### weights 加权

根据权重比例选择目标，权重越高的进程被选中的概率越大：

```javascript
const lb = new LoadBalancer({
  algorithm: 'weights',
  targets: [{ id: 1, weight: 3 }, { id: 2, weight: 1 }],
});
// id=1 被选中的概率是 75%，id=2 是 25%
```

### random 随机

完全随机选择一个目标，不考虑权重和历史选择：

```javascript
const lb = new LoadBalancer({
  algorithm: 'random',
  targets: [{ id: 1 }, { id: 2 }, { id: 3 }],
});
```

### specify 指定

选择指定 ID 的目标，需要通过 `pickOne(id)` 传入目标 ID：

```javascript
const lb = new LoadBalancer({
  algorithm: 'specify',
  targets: [{ id: 1 }, { id: 2 }, { id: 3 }],
});

// 选择 id=2 的进程
lb.pickOne(2);
```

### weightsPolling 加权轮询

结合权重和轮询，权重越高的进程在轮询周期中被选中的次数越多：

```javascript
const lb = new LoadBalancer({
  algorithm: 'weightsPolling',
  targets: [{ id: 1, weight: 3 }, { id: 2, weight: 1 }],
});
// 轮询周期: 1, 1, 1, 2 → 1, 1, 1, 2 → ...
```

### weightsRandom 加权随机

结合权重和随机选择，权重越高的进程被随机选中的概率越大：

```javascript
const lb = new LoadBalancer({
  algorithm: 'weightsRandom',
  targets: [{ id: 1, weight: 3 }, { id: 2, weight: 1 }],
});
```

### minimumConnection 最小连接数

选择当前连接数最少的目标，适合长连接/持续任务场景：

```javascript
const lb = new LoadBalancer({
  algorithm: 'minimumConnection',
  targets: [{ id: 1 }, { id: 2 }, { id: 3 }],
});

// connectionsMap: { 1: 3, 2: 1, 3: 2 }
// pickOne() → 选择 id=2（连接数最少）
```

### weightsMinimumConnection 加权最小连接数

结合权重和最小连接数，在选择时同时考虑权重和当前连接数：

```javascript
const lb = new LoadBalancer({
  algorithm: 'weightsMinimumConnection',
  targets: [{ id: 1, weight: 3 }, { id: 2, weight: 1 }],
});
```

## Scheduler 调度器

LoadBalancer 内部使用 Scheduler 类调度算法选择：

```javascript
class Scheduler {
  calculate(tasks, params) {
    const algo = algorithms[this.algorithm];
    return algo(tasks, ...params);
  }

  setAlgorithm(algorithm) {
    if (algorithm in AlgorithmType) {
      this.algorithm = algorithm;
    } else {
      throw new Error(`Invalid algorithm: ${algorithm}`);
    }
  }
}
```

Scheduler 维护当前算法名称，`calculate()` 方法根据算法类型调用对应的算法函数。

## targets 和 weights 配置

进程池中的每个进程在 LoadBalancer 中表示为一个 target：

```javascript
{
  id: 12345,      // 进程 PID
  weight: 1,      // 权重值（默认 1）
}
```

ChildPoolJob 在创建进程时自动将进程注册到 LoadBalancer：

```javascript
// ChildPoolJob._childCreated()
const lbTask = {
  id: pid,           // 子进程 PID
  weight: this.weights[length - 1] || 1,  // 权重数组中的对应值
};
this.LB.add(lbTask);
```

### 自定义权重

```javascript
// 创建时指定权重
const pool = new ChildPoolJob({
  weights: [3, 2, 1, 1, 1, 1],  // 6 个进程的权重值
});
await pool.create(3);
```

<Note>
权重值越高，进程在高负载时被优先选择。建议为性能更强的机器/进程分配更高的权重值。
</Note>

## 动态管理

### 添加目标

```javascript
lb.add({ id: 12346, weight: 2 });
```

<Warning>
如果添加已存在的 id，LoadBalancer 会输出警告并忽略该操作。
</Warning>

### 删除目标

```javascript
lb.del({ id: 12345, weight: 1 });
```

删除目标时同时清理 params 中的 connectionsMap、cpuOccupancyMap 和 memoryOccupancyMap。

### 运行时切换算法

```javascript
lb.setAlgorithm('weightsPolling');
```

<Warning>
切换算法时会重置 weightIndex 参数为 0。只接受 AlgorithmType 中定义的有效算法名称，否则抛出错误。
</Warning>

### 清理数据

```javascript
// 清理指定进程的数据
lb.clean(12345);

// 清理所有数据
lb.clean();

// 完全重置（清空 targets 和 params）
lb.wipe();
```

## 算法选择指南

<Tabs>
  <Tab title="简单均匀分配">
    当所有进程性能相同且任务类型一致时：

    ```javascript
    // 推荐：polling 轮询
    const lb = new LoadBalancer({
      algorithm: 'polling',
      targets: [{ id: 1 }, { id: 2 }, { id: 3 }],
    });
    ```
  </Tab>
  <Tab title="性能差异分配">
    当部分进程性能更强时：

    ```javascript
    // 推荐：weightsPolling 加权轮询
    const lb = new LoadBalancer({
      algorithm: 'weightsPolling',
      targets: [
        { id: 1, weight: 3 },  // 高性能进程
        { id: 2, weight: 1 },  // 低性能进程
      ],
    });
    ```
  </Tab>
  <Tab title="长任务场景">
    当任务是长连接/持续运行时：

    ```javascript
    // 推荐：minimumConnection 最小连接数
    const lb = new LoadBalancer({
      algorithm: 'minimumConnection',
      targets: [{ id: 1 }, { id: 2 }, { id: 3 }],
    });
    ```
  </Tab>
  <Tab title="指定进程">
    当需要固定到某个进程时：

    ```javascript
    // 使用：specify 指定
    const lb = new LoadBalancer({
      algorithm: 'specify',
      targets: [{ id: 1 }, { id: 2 }],
    });
    lb.pickOne(2);  // 总是选择 id=2 的进程
    ```
  </Tab>
</Tabs>

## pickOne 和 pickMulti

```javascript
// 选择一个进程
const target = lb.pickOne();

// 选择多个进程
const targets = lb.pickMulti(3);  // 选择 3 个进程

// specify 算法需要传入 ID
const target = lb.pickOne(12345);
```

## refreshParams 刷新参数

LoadBalancer 支持从外部刷新进程的 CPU 和内存占用信息：

```javascript
// 更新进程资源占用数据
lb.refreshParams({
  12345: { pid: 12345, cpu: 0.3, memory: 0.4 },
  12346: { pid: 12346, cpu: 0.1, memory: 0.2 },
});
```
