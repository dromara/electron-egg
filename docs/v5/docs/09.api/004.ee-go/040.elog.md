## Introduction

日志模块，基于 zap + lumberjack 实现结构化日志输出。支持 JSON 与 Console 两种格式、日志文件自动切割（按大小与天数）、同时输出到文件与控制台。提供两个全局日志实例：`Logger`（业务日志）与 `CoreLogger`（框架核心日志）。日志目录默认为当前工作目录，可通过 `SetLogDir()` 动态修改。

## Import

```go
import "github.com/wallace5303/ee-go/elog"
```

## Global Variables

### Logger

`*zap.SugaredLogger` — 业务日志实例，由 `InitLog()` 初始化。用于记录业务代码中的日志。

```go
elog.Logger.Info("hello")
elog.Logger.Infof("user id: %d", userId)
elog.Logger.Warn("warning message")
elog.Logger.Errorf("error: %s", err)
```

### CoreLogger

`*zap.SugaredLogger` — 框架核心日志实例，由 `InitCoreLog()` 初始化。用于记录 ee-go 框架内部的日志。

```go
elog.CoreLogger.Infof("[ee-go] load http service")
elog.CoreLogger.Errorf("[ee-go] The port:%s already in use", port)
```

### LogDir

`string` — 日志文件存储目录，默认为当前工作目录（`os.Getwd()`），由 `SetLogDir()` 修改。

### LogName

`string` — 默认日志文件名，值为 `"ee-go.log"`。

## Structs

### LogConfig

日志配置结构体，用于生成 zap Logger 时的参数。

| 字段 | 类型 | 说明 |
|------|------|------|
| OutputJSON | `bool` | 输出格式：`true` 为 JSON 格式，`false` 为 Console 格式（默认） |
| Level | `string` | 最低日志等级，如 `"debug"`、`"info"`、`"warn"`、`"error"`（默认 `"info"`） |
| FileName | `string` | 日志文件绝对路径，默认为 `{LogDir}/ee-go.log` |
| MaxSize | `int` | 单个日志文件最大大小（MB），超过后自动切割，默认 1024 |
| MaxAge | `int` | 保留旧日志文件的最大天数，默认 30 |

## API

### InitLog(cfg)

**描述**: 初始化业务日志实例 `Logger`。根据配置创建 zap Logger 并转换为 SugaredLogger 写入全局 `Logger` 变量。

**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| cfg | `map[string]any` | 是 | 日志配置，支持字段：`output_json`（bool）、`level`（string）、`filename`（string）、`max_size`（float64→int）、`max_age`（float64→int） |

**返回值**: 无

**示例**:

```go
logCfg := econfig.GetLogger()
elog.InitLog(logCfg)

// 使用业务日志
elog.Logger.Info("application started")
elog.Logger.Infof("request: method=%s, path=%s", method, path)
```

**参见**: econfig.GetLogger()

### InitCoreLog(cfg)

**描述**: 初始化框架核心日志实例 `CoreLogger`。与 `InitLog()` 逻辑相同，创建独立的 SugaredLogger 写入全局 `CoreLogger` 变量，用于记录 ee-go 框架内部日志。

**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| cfg | `map[string]any` | 是 | 核心日志配置，字段同 `InitLog()` |

**返回值**: 无

**示例**:

```go
coreLogCfg := econfig.GetCoreLogger()
elog.InitCoreLog(coreLogCfg)

// 使用核心日志
elog.CoreLogger.Infof("[ee-go] http server %s, pid:%d", url, pid)
```

**参见**: econfig.GetCoreLogger()

### GetZLog(cfg)

**描述**: 根据配置创建并返回一个 `*zap.Logger` 实例。这是日志初始化的核心函数，`InitLog()` 和 `InitCoreLog()` 内部均调用此函数。合并默认配置与传入配置，生成 zap Core（编码器 + 写入器 + 级别过滤），返回带调用者信息的 zap Logger。

**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| cfg | `map[string]any` | 是 | 日志配置，字段同 `InitLog()`；传入空 map 时使用默认值 |

**返回值**: `*zap.Logger` — zap Logger 实例，可进一步调用 `.Sugar()` 获取 SugaredLogger

**示例**:

```go
cfg := map[string]any{
    "output_json": false,
    "level":       "info",
    "filename":    "app.log",
    "max_size":    512,
    "max_age":     7,
}
zlog := elog.GetZLog(cfg)
logger := zlog.Sugar()
logger.Info("custom logger message")
```

### SetLogDir(path)

**描述**: 设置日志文件存储目录。修改全局 `LogDir` 变量，影响后续 `GetZLog()` 中日志文件的绝对路径拼接。通常在初始化阶段由 `eboot` 调用，将日志目录指向 `{WorkDir}/logs`。

**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| path | `string` | 是 | 日志目录绝对路径 |

**返回值**: 无

**示例**:

```go
logDir := filepath.Join(eruntime.WorkDir, "logs")
elog.SetLogDir(logDir)
```

**参见**: eruntime.WorkDir

## SugaredLogger Methods

`Logger` 和 `CoreLogger` 均为 `*zap.SugaredLogger`，支持以下常用方法：

| 方法 | 说明 |
|------|------|
| `Info(args ...any)` | 记录 INFO 级别日志 |
| `Infof(template string, args ...any)` | 记录 INFO 级别格式化日志 |
| `Warn(args ...any)` | 记录 WARN 级别日志 |
| `Warnf(template string, args ...any)` | 记录 WARN 级别格式化日志 |
| `Error(args ...any)` | 记录 ERROR 级别日志 |
| `Errorf(template string, args ...any)` | 记录 ERROR 级别格式化日志 |
| `DPanic(args ...any)` | 记录 DPANIC 级别日志（开发环境 panic） |
| `DPanicf(template string, args ...any)` | 记录 DPANIC 级别格式化日志 |
| `Panic(args ...any)` | 记录 PANIC 级别日志并 panic |
| `Panicf(template string, args ...any)` | 记录 PANIC 级别格式化日志并 panic |
| `Fatal(args ...any)` | 记录 FATAL 级别日志并终止进程 |
| `Fatalf(template string, args ...any)` | 记录 FATAL 级别格式化日志并终止进程 |
