## Introduction

HTTP 服务模块，基于 Gin 框架创建和管理 HTTP 服务器。负责服务启动、CORS 跨域、Gzip 压缩、Session 管理、静态资源加载、路由分发、平台检测与端口占用检查。在 ee-go 初始化流程中由 `eboot` 调用 `CreateServer()` 自动启动。

## Import

```go
import "github.com/wallace5303/ee-go/ehttp"
```

## Constants

### PlatformPC

`string` — `"pc"`，表示 Electron 桌面端平台标识。

### PlatformBrowser

`string` — `"browser"`，表示浏览器桌面端平台标识。

### PlatformPhone

`string` — `"phone"`，表示手机端平台标识。

### PlatformPad

`string` — `"pad"`，表示平板端平台标识。

### Conf

`map[string]any` — HTTP 配置全局变量，由 `CreateServer()` 写入，存储 http 配置项（port、network 等）。

## API

### CreateServer(cfg)

**描述**: 创建并启动 HTTP 服务器。基于 Gin 框架，依次注册 CORS、Session、Gzip 中间件，加载调试路由（pprof）、静态资源与视图路由，检测端口占用后监听服务。

**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| cfg | `map[string]any` | 是 | HTTP 配置，包含 `port`（float64→int）、`network`（bool）等字段 |

**返回值**: 无

**示例**:

```go
httpCfg := econfig.GetHttp()
if httpCfg["enable"] == true {
    ehttp.CreateServer(httpCfg)
}
```

**参见**: econfig.GetHttp()、erouter

### GetPlatform(ctx)

**描述**: 根据 HTTP 请求的 User-Agent 头判断当前访问平台类型。Electron 客户端返回 `"pc"`；平板或大屏 Android 返回 `"browser"`；移动端返回 `"phone"`；其余浏览器返回 `"browser"`。

**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| ctx | `*gin.Context` | 是 | Gin 请求上下文 |

**返回值**: `string` — 平台标识，取值为 `PlatformPC`、`PlatformBrowser`、`PlatformPhone` 之一

**示例**:

```go
platform := ehttp.GetPlatform(ctx)
if platform == ehttp.PlatformPC {
    // 桌面端逻辑
} else if platform == ehttp.PlatformPhone {
    // 移动端逻辑
}
```

### IsPortOpen(port)

**描述**: 检查指定 TCP 端口是否已被占用。通过 `net.DialTimeout` 在 1 秒内尝试连接 `127.0.0.1:port`，若连接成功则端口已开放（被占用）。

**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| port | `string` | 是 | 端口号字符串，如 `"7073"` |

**返回值**: `bool` — `true` 表示端口已被占用，`false` 表示端口空闲

**示例**:

```go
port := "7073"
if ehttp.IsPortOpen(port) {
    fmt.Printf("端口 %s 已被占用\n", port)
} else {
    fmt.Printf("端口 %s 可用\n", port)
}
```
