## Introduction

任务队列模块，提供异步任务的添加、去重、执行和定时循环功能。任务按队列顺序依次执行（单线程模型），支持超时控制和唯一任务约束（同一 action 在队列中只能存在一个）。适用于后台异步操作、定时轮询等场景。

## Import

```go
import "github.com/wallace5303/ee-go/etask"
```

## API

### Task

**描述**: 任务结构体，表示队列中的一个异步任务。

| 字段 | 类型 | 说明 |
|------|------|------|
| Action | `string` | 任务标识名称，如 `"task.demo"` |
| Handler | `reflect.Value` | 任务处理函数，通过 `reflect.ValueOf` 包装 |
| Args | `[]interface{}` | 传递给 Handler 的参数列表 |
| Created | `time.Time` | 任务创建时间 |
| Timeout | `time.Duration` | 任务执行超时时间，默认 24 小时 |

### UniqueActions

**描述**: 全局唯一任务 action 列表。在此列表中的任务 action，队列中只能存在一个实例——若当前队列或正在执行的任务中已有相同 action，则 `AddTask` / `AddTaskWithTimeout` 会跳过添加。

**类型**: `[]string`

**示例**:

```go
etask.UniqueActions = []string{"task.syncData", "task.cleanup"}
// 之后添加同名任务，若队列中已存在则自动跳过
```

### AddTask(action, handler, args...)

**描述**: 添加任务到队列。默认超时时间为 24 小时（`24 * time.Hour`）。若应用正在退出（`eruntime.IsExiting == true`）则跳过添加。若任务 action 属于 `UniqueActions` 且队列中已存在同名任务，则跳过添加。

**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| action | `string` | 是 | 任务标识名称，如 `"task.demo"` |
| handler | `interface{}` | 是 | 任务处理函数 |
| args | `...interface{}` | 否 | 传递给 handler 的参数 |

**返回值**: 无返回值

**示例**:

```go
etask.AddTask("task.sendEmail", func(to string) {
    fmt.Println("sending email to:", to)
}, "user@example.com")
```

**参见**: `AddTaskWithTimeout` — 添加带自定义超时的任务

### AddTaskWithTimeout(action, handler, timeout, args...)

**描述**: 添加带自定义超时时间的任务到队列。超时后任务会在日志中标记警告，但不终止 goroutine。其余逻辑与 `AddTask` 相同。

**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| action | `string` | 是 | 任务标识名称 |
| handler | `interface{}` | 是 | 任务处理函数 |
| timeout | `time.Duration` | 是 | 任务执行超时时间 |
| args | `...interface{}` | 否 | 传递给 handler 的参数 |

**返回值**: 无返回值

**示例**:

```go
etask.AddTaskWithTimeout("task.longOperation", func() {
    // 耗时操作
}, 5*time.Minute)
```

### Contain(action, moreActions...)

**描述**: 检查队列中是否包含指定 action 的任务。可同时检查多个 action（去重后判断），只要队列中存在任一匹配即返回 `true`。

**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| action | `string` | 是 | 要检查的任务 action |
| moreActions | `...string` | 否 | 额外要检查的 action 列表 |

**返回值**: `bool` — 队列中存在匹配 action 的任务返回 `true`，否则返回 `false`

**示例**:

```go
if etask.Contain("task.syncData", "task.cleanup") {
    fmt.Println("queue contains syncData or cleanup task")
}
```

### Status()

**描述**: 打印当前任务队列状态信息。输出所有待执行任务和当前正在执行任务的 action 列表及计数。仅在开发环境（`eruntime.IsDev()`）下输出到日志。

**返回值**: 无返回值

**示例**:

```go
etask.Status()
// 开发环境下日志输出:
// task status data:  [{"action":"task.syncData"}, {"action":"task.sendEmail"}]
```

### ExecTask()

**描述**: 从队列头部取出一个任务并执行。若应用正在退出（`eruntime.IsExiting == true`）或队列为空则直接返回。任务执行时使用 `reflect.Value.Call` 调用 handler，并通过 `context.WithTimeout` 控制超时。任务完成后日志输出 `"task [action] done"`，超时则输出 `"task [action] timeout"`。执行过程中若发生 panic，由 `eutil.Recover()` 捕获并记录。

**返回值**: 无返回值

**示例**:

```go
// 通常在主循环中调用
for {
    etask.ExecTask()
    time.Sleep(100 * time.Millisecond)
}
```

### Every(interval, f)

**描述**: 每隔指定时间间隔循环执行函数。启动时先随机睡眠 50~200 毫秒（避免多实例同时启动），然后无限循环：执行函数 → 等待间隔 → 再次执行。函数执行中的 panic 由 `eutil.Recover()` 捕获，不会中断循环。

**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| interval | `time.Duration` | 是 | 循环间隔时间 |
| f | `func()` | 是 | 每次循环执行的函数 |

**返回值**: 无返回值（该函数会无限循环，通常在独立 goroutine 中调用）

**示例**:

```go
// 在独立 goroutine 中启动定时任务
go etask.Every(30*time.Second, func() {
    fmt.Println("health check running")
})
```
