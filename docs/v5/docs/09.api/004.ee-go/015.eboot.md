## Introduction

eboot 是 ee-go 的启动引导模块，负责应用的初始化与启动流程。它解析命令行参数、初始化运行环境（目录、配置、日志、HTTP 服务等），并提供 `Ego` 结构体作为应用的入口对象。`New()` 函数完成全部初始化工作，`Run()` 方法启动信号监听使程序持续运行。

## Import

```go
import "github.com/wallace5303/ee-go/eboot"
```

## API

### Ego

**描述**: ee-go 应用实例类型，作为程序的入口对象。通过 `New()` 创建后，调用 `Run()` 方法启动信号监听。

| 字段 | 类型 | 说明 |
|------|------|------|
| （无公开字段） | — | Ego 为空结构体，所有初始化状态存储在全局模块中 |

---

### New(staticFS)

**描述**: 创建新的 ee-go 应用实例并完成全部初始化流程。包括：解析命令行参数、设置运行环境变量、初始化目录结构、加载配置、读取 package.json、初始化用户目录、初始化日志系统、创建 HTTP 服务。

**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| staticFS | `embed.FS` | 是 | 嵌入的静态文件系统，通过 `//go:embed` 指令将 `public/` 目录内容嵌入到二进制中 |

**返回值**: `*Ego` — 初始化完成的 ee-go 应用实例

**示例**:

```go
package main

import (
    "embed"
    "github.com/wallace5303/ee-go/eboot"
)

//go:embed public
var staticFS embed.FS

func main() {
    ego := eboot.New(staticFS)
    ego.Run()
}
```

---

### Run()

**描述**: 启动程序信号监听，阻塞等待系统信号（SIGINT、SIGTERM、SIGQUIT）。收到信号后自动执行优雅退出流程。内部调用 `eapp.Run()`。

**参数**: 无

**返回值**: 无

**示例**:

```go
ego := eboot.New(staticFS)
ego.Run() // 阻塞运行，直到收到退出信号
```

**参见**: `eapp.Run()`

## CLI Flags

`New()` 函数在初始化时解析以下命令行参数：

| Flag | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `-env` | `string` | `"prod"` | 运行环境，可选 `"dev"` 或 `"prod"` |
| `-basedir` | `string` | `"./"` | 应用基础目录路径 |
| `-port` | `string` | `"0"` | HTTP 服务端口，`"0"` 表示使用配置文件中的端口 |
| `-ssl` | `string` | `"false"` | 是否启用 HTTPS/WSS，可选 `"true"` 或 `"false"` |
| `-debug` | `string` | `"false"` | 是否启用调试模式，可选 `"true"` 或 `"false"` |

**示例**:

```bash
# 开发模式启动，指定端口和调试
go run main.go -env=dev -port=8080 -debug=true

# 生产模式启动，指定基础目录
./app -env=prod -basedir=/opt/myapp -ssl=true
```

## Initialization Flow

`New()` 执行的初始化顺序（不可变）：

1. 解析命令行参数，设置 `eruntime` 全局变量
2. 设置嵌入静态文件系统 (`estatic.StaticFS`)
3. 初始化目录 (`eruntime.InitDir()`)
4. 初始化配置 (`econfig.Init()`)
5. 读取 package.json，获取应用名称 (`eapp.ReadPackage()`)
6. 初始化用户目录（创建 `~/.config/{appName}`、`~/.{appName}`、data、logs、tmp 等目录）
7. 初始化核心日志 (`elog.InitCoreLog()`)
8. 初始化业务日志 (`elog.InitLog()`)
9. 创建 HTTP 服务（当配置中 `http.enable` 为 `true` 时）
