## Introduction

静态资源模块，基于 Go 的 `embed.FS` 提供对嵌入文件系统的访问。支持检查文件是否存在、读取 JSON 配置文件（严格模式和容错模式）。严格模式在读文件失败或解析失败时直接抛出异常并退出，容错模式则返回错误供业务代码处理。

## Import

```go
import "github.com/wallace5303/ee-go/estatic"
```

## API

### StaticFS

**描述**: 嵌入文件系统实例，类型为 `embed.FS`。业务代码通过此变量访问编译时嵌入的静态资源文件。

**类型**: `embed.FS`

**示例**:

```go
data, err := estatic.StaticFS.ReadFile("config/app.json")
if err != nil {
    fmt.Println("read failed:", err)
}
```

### FileIsExist(name)

**描述**: 检查嵌入文件系统中指定文件是否存在。通过调用 `StaticFS.ReadFile` 并判断是否返回错误来确定文件存在性。

**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | `string` | 是 | 嵌入文件系统中的文件路径，如 `"config/app.json"` |

**返回值**: `bool` — 文件存在返回 `true`，不存在返回 `false`

**示例**:

```go
if estatic.FileIsExist("public/html/failure.html") {
    fmt.Println("file exists in embedded FS")
}
```

### ReadJsonStrict(f)

**描述**: 从嵌入文件系统中读取 JSON 文件并解析为 `map[string]any`。严格模式：读取失败或 JSON 解析失败时调用 `eerror.ThrowWithCode` 抛出异常并退出进程（退出码 `ExitConfigFileFS`）。适用于必须存在的配置文件。

**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| f | `string` | 是 | 嵌入文件系统中的 JSON 文件路径 |

**返回值**: `map[string]any` — JSON 文件解析后的键值映射

**示例**:

```go
config := estatic.ReadJsonStrict("config/app.json")
fmt.Println("app name:", config["name"])
```

**参见**: `eutil.ReadJsonStrict` — 从磁盘文件系统读取 JSON（严格模式）

### ReadJson(f)

**描述**: 从嵌入文件系统中读取 JSON 文件并解析为 `map[string]any`。容错模式：读取失败或 JSON 解析失败时返回错误，不退出进程。适用于可选的配置文件。

**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| f | `string` | 是 | 嵌入文件系统中的 JSON 文件路径 |

**返回值**: `(map[string]any, error)` — 成功时返回 JSON 解析结果和 `nil`，失败时返回 `nil` 和错误信息

**示例**:

```go
config, err := estatic.ReadJson("config/optional.json")
if err != nil {
    fmt.Println("optional config not found, using defaults")
    config = map[string]any{}
}
fmt.Println("config:", config)
```

**参见**: `eutil.ReadJson` — 从磁盘文件系统读取 JSON（容错模式）
