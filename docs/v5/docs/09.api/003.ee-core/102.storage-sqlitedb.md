### Introduction
SQLite database storage. Based on better-sqlite3, provides synchronous database operations supporting four storage modes.

::: warning Important Change (v5)
`SqliteStorage` constructor now only computes paths (synchronous). You must call **`await init()`** to lazily load better-sqlite3 and open the database connection. The `opt` parameter (timeout, verbose, etc.) is now passed to `init()`, not the constructor.
:::

### Usage
```javascript
import { SqliteStorage } from 'ee-core/storage';
import { getDataDir } from 'ee-core/ps';
import path from 'path';
```

### API List

### Constructor and init
Create a SqliteStorage instance (path computation only), then call `init()` to open the database.

```javascript
// Create instance (synchronous, no DB opened)
const storage = new SqliteStorage(dbFile);

// Lazily load better-sqlite3 and open the database (async)
await storage.init(sqliteOptions);

// Access the database instance
const db = storage.db;
```

#### Storage modes
| Mode | Name format | Storage location |
|------|-------------|------------------|
| `memory` | `:memory:` | In-memory, not persisted |
| `onlyName` | `app.db` | `{dataDir}/db/app.db` |
| `relative` | `sub/app.db` | `{dataDir}/db/sub/app.db` |
| `absolute` | `/tmp/app.db` | Specified path |

#### init(opt)
- opt [Object] - better-sqlite3 database options, merged with `{ timeout: 5000 }`

Returns `this` for chaining: `const storage = await new SqliteStorage(dbFile).init();`

::: tip Note
`init()` is the only point where better-sqlite3 is loaded. If you never call `init()`, the native module is never loaded.
:::

#### storage.db
The better-sqlite3 Database instance. Available after `init()`.

```javascript
// Insert data
const insert = storage.db.prepare(`INSERT INTO user (name, age) VALUES (@name, @age)`);
insert.run({ name: 'test', age: 25 });
```

#### storage.name
Database name (original input value).

#### storage.mode
Storage mode: `memory`, `onlyName`, `relative`, or `absolute`.

#### storage.getDbDir()
Get the database file directory path.

#### storage.fileName
Formatted database filename.

#### storage.getFilePath()
Get the full database file path (`dbDir` + `fileName`).
