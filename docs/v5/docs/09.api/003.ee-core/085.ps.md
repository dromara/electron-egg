### Introduction
Process utility module. Provides environment detection, process type checking, and path retrieval functions. All path information comes from `process.env` environment variables set during boot.

### Usage
```javascript
import {
  allEnv,
  env,
  isProd,
  isDev,
  getDataDir,
  getLogDir,
  getBaseDir,
  getExtraResourcesDir,
} from 'ee-core/ps';
```

### API List

### allEnv()
All environment variables of the current process.

### env()
Current environment name: `dev`, `local`, or `prod`.

### isProd()
Whether it is a production environment.

### isDev()
Whether it is a development environment (`dev` or `local`).

### isRenderer()
Whether it is a renderer process.

### isMain()
Whether it is the main (browser) process.

### isForkedChild()
Whether it is a Node.js child process (created via `child_process.fork`).

### processType()
Current process type: `browser`, `renderer`, or `child`.

### appName()
Application name.

### appVersion()
Application version.

### getDataDir()
Data storage path. Dev: `{baseDir}/data`, Prod: `{userHome}/.{appName}/data`.
On OpenHarmony: uses custom app directory for permission compatibility.

### getLogDir()
Log storage path. Dev: `{baseDir}/logs`, Prod: `{userHome}/.{appName}/logs`.
On OpenHarmony: uses custom app directory for permission compatibility.

### getBundleDir()
Bundled resource directory: `{basePath}/public/electron`.

### getElectronCodeDir()
Electron source directory: `{basePath}/electron`.

### getFrontendCodeDir()
Frontend source directory: `{basePath}/frontend`.

### getRootDir()
Root directory. Dev: project root (`baseDir`), Prod: app user data directory.

### getBaseDir()
Project root directory (baseDir).

### getElectronDir()
Electron directory (where business code resides).

### getPublicDir()
Public static assets directory: `{baseDir}/public`.

### getExtraResourcesDir()
Extra resources directory. Path differs by platform:
- Windows/Linux: `{execDir}/resources/extraResources`
- macOS: `{execDir}/../Resources/extraResources`
- OpenHarmony: `{execDir}/../Resources/extraResources`

Before packaging: `{execDir}/build/extraResources`.

### getAppUserDataDir()
Application user data directory.

### getExecDir()
Executable directory. After packaging, this is where the exe/dmg/deb is located.

### getUserHomeDir()
User home directory. On OpenHarmony: returns appUserData directory for permission compatibility.

### getUserHomeHiddenAppDir()
Hidden app directory under user home: `{userHome}/.{appName}/`.

### getUserHomeAppDir()
App directory under user home: `{userHome}/{appName}/`.

### getAppCustomDataDir()
Custom data directory for OpenHarmony: `{appUserData}/{appName}/`.

### getSocketPort()
Built-in Socket server port number.

### getHttpPort()
Built-in HTTP server port number.

### isPackaged()
Whether the app is packaged (production environment).

### exit(code)
- code [Number] - Exit code (default: 0)

Process exit.

### exitChildJob(code)
- code [Number] - Exit code (default: 0)

Exit a ChildJob-type child process. Only exits if `process.argv[2]` type is `childJob`.

### isChildJob()
Whether the current process is a ChildJob-type child process.

### isChildPoolJob()
Whether the current process is a ChildPoolJob-type child process.

### getArgumentByName(name)
- name [String] - Argument name (without `--` prefix)

Get command line argument value. Search format: `--name=value`.
