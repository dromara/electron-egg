## Introduction

Process state and path utility module. Provides environment detection, process type checking, and path retrieval functions, serving as the foundational dependency for other modules to obtain runtime environment information. All path information comes from `process.env` environment variables set during boot.

## Import

```javascript
// CJS
const { allEnv, env, isProd, isDev, getDataDir, getLogDir } = require('ee-core/ps');

// ESM
import { allEnv, env, isProd, isDev, getDataDir, getLogDir } from 'ee-core/ps';
```

## API

### allEnv()
**描述**: Get all environment variables of the current process.
**返回值**: `NodeJS.ProcessEnv` — the complete `process.env` object.

### env()
**描述**: Get the current environment name.
**返回值**: `string` — environment name: `'dev'`, `'local'`, `'prod'`, or empty string if not set.

### isProd()
**描述**: Whether this is a production environment (`EE_ENV === 'prod'`).
**返回值**: `boolean` — `true` if production environment.

### isDev()
**描述**: Whether this is a development environment (`EE_ENV === 'dev'` or `'local'`).
**返回值**: `boolean` — `true` if development environment.

### isRenderer()
**描述**: Whether this is a renderer process (`process.type === 'renderer'`).
**返回值**: `boolean` — `true` if renderer process.

### isMain()
**描述**: Whether this is the main (browser) process (`process.type === 'browser'`).
**返回值**: `boolean` — `true` if main process.

### isForkedChild()
**描述**: Whether this is a Node child process created via `child_process.fork` (`ELECTRON_RUN_AS_NODE === 1`).
**返回值**: `boolean` — `true` if forked child process.

### processType()
**描述**: Get the current process type string.
**返回值**: `string` — `'browser'`, `'renderer'`, or `'child'`. Empty string if none match.

### appName()
**描述**: Get the application name from `EE_APP_NAME`.
**返回值**: `string` — application name, or empty string if not set.

### appVersion()
**描述**: Get the application version from `EE_APP_VERSION`.
**返回值**: `string` — application version, or empty string if not set.

### getDataDir()
**描述**: Get the data storage path. In dev mode: `{baseDir}/data`. In production: `{userHome}/.{appName}/data`. On OpenHarmony: uses custom app directory for permission compatibility.
**返回值**: `string` — absolute data directory path.

### getLogDir()
**描述**: Get the log storage path. In dev mode: `{baseDir}/logs`. In production: `{userHome}/.{appName}/logs`. On OpenHarmony: uses custom app directory for permission compatibility.
**返回值**: `string` — absolute log directory path.

### getBundleDir(basePath?)
**描述**: Get the build output directory.
**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| basePath | `string` | 否 | Base path, defaults to `process.cwd()` |

**返回值**: `string` — `{basePath}/public/electron`.

### getElectronCodeDir(basePath?)
**描述**: Get the electron source directory.
**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| basePath | `string` | 否 | Base path, defaults to `process.cwd()` |

**返回值**: `string` — `{basePath}/electron`.

### getFrontendCodeDir(basePath?)
**描述**: Get the frontend source directory.
**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| basePath | `string` | 否 | Base path, defaults to `process.cwd()` |

**返回值**: `string` — `{basePath}/frontend`.

### getRootDir()
**描述**: Get the root directory. In dev mode: project root (`baseDir`). In production: app user data directory.
**返回值**: `string` — absolute root directory path.

### getBaseDir()
**描述**: Get the project root directory (`EE_BASE_DIR`).
**返回值**: `string` — project root directory, or empty string if not set.

### getElectronDir()
**描述**: Get the electron directory where business code resides (`EE_ELECTRON_DIR`).
**返回值**: `string` — electron directory path, or empty string if not set.

### getPublicDir()
**描述**: Get the public static assets directory.
**返回值**: `string` — `{baseDir}/public`.

### getExtraResourcesDir()
**描述**: Get the extra resources directory. Path differs by platform after packaging:
- Windows/Linux: `{execDir}/resources/extraResources`
- macOS: `{execDir}/../Resources/extraResources`
- OpenHarmony: `{execDir}/../Resources/extraResources`

Before packaging: `{execDir}/build/extraResources`.
**返回值**: `string` — absolute extra resources directory path.

### getAppUserDataDir()
**描述**: Get the Electron appUserData directory (`EE_APP_USER_DATA`).
**返回值**: `string` — app user data directory, or empty string if not set.

### getExecDir()
**描述**: Get the executable directory (`EE_EXEC_DIR`). After packaging, this is where the exe/dmg/deb is located.
**返回值**: `string` — executable directory path, or empty string if not set.

### getUserHomeDir()
**描述**: Get the OS user home directory (`EE_USER_HOME`). On OpenHarmony: returns appUserData directory for permission compatibility.
**返回值**: `string` — user home directory path, or empty string if not set.

### getUserHomeHiddenAppDir()
**描述**: Get the hidden app directory under the user home directory. Used for storing data, logs, and other persistent files in production.
**返回值**: `string` — `{userHome}/.{appName}/`.

### getUserHomeAppDir()
**描述**: Get the app directory under the user home directory (not hidden).
**返回值**: `string` — `{userHome}/{appName}/`.

### getAppCustomDataDir()
**描述**: Get the application custom data directory. Used on OpenHarmony where the user home directory is not writable.
**返回值**: `string` — `{appUserData}/{appName}/`.

### getSocketPort()
**描述**: Get the built-in Socket server port number.
**返回值**: `number` — Socket server port, or `0` if not set.
**参见**: socket module — `SocketServer` sets this port during initialization.

### getHttpPort()
**描述**: Get the built-in HTTP server port number.
**返回值**: `number` — HTTP server port, or `0` if not set.
**参见**: socket module — `HttpServer` sets this port during initialization.

### isPackaged()
**描述**: Whether the app is packaged (production environment, `EE_IS_PACKAGED === 'true'`).
**返回值**: `boolean` — `true` if the app is packaged.

### exit(code?)
**描述**: Exit the current process.
**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| code | `number` | 否 | Exit code, defaults to `0` |

**返回值**: `never` — this function never returns (terminates the process).
**示例**:
```javascript
import { exit } from 'ee-core/ps';
exit(1); // exit with code 1
```

### exitChildJob(code?)
**描述**: Exit a ChildJob-type child process. Checks `process.argv[2]` to determine if this is a ChildJob process. Only ChildJob processes will exit; other types of child processes are not affected.
**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| code | `number` | 否 | Exit code, defaults to `0` |

**返回值**: `void`
**参见**: jobs module — `ChildJob` processes set `type: 'childJob'` in argv.
**示例**:
```javascript
import { exitChildJob } from 'ee-core/ps';
exitChildJob(); // only exits if current process is a ChildJob
```

### isChildJob()
**描述**: Determine whether the current process is of ChildJob type. Checks the `type` field in `process.argv[2]`.
**返回值**: `boolean` — `true` if the current process is a ChildJob.
**参见**: jobs module — `ChildJob`.

### isChildPoolJob()
**描述**: Determine whether the current process is of ChildPoolJob type. Checks the `type` field in `process.argv[2]`.
**返回值**: `boolean` — `true` if the current process is a ChildPoolJob.
**参见**: jobs module — `ChildPoolJob`, `LoadBalancer`.

### getArgumentByName(name, args?)
**描述**: Get the value of a named argument from command line arguments. Search format: `--name=value`.
**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| name | `string` | 是 | Argument name (without the `--` prefix) |
| args | `string[]` | 否 | Arguments array, defaults to `process.argv` |

**返回值**: `string | undefined` — argument value, or `undefined` if not found.
**示例**:
```javascript
import { getArgumentByName } from 'ee-core/ps';
// If process.argv contains '--env=prod'
const envValue = getArgumentByName('env'); // 'prod'
```

### makeMessage(msg?)
**描述**: Format an IPC message object, filling in missing fields with default values.
**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| msg | `Partial<{ channel: string; event: string; data: unknown }>` | 否 | Partial message fields |

**返回值**: `{ channel: string; event: string; data: unknown }` — complete message object with defaults (`channel: ''`, `event: ''`, `data: {}`).
