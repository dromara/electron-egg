### Introduction
Electron window management APIs. All window-related functionality wrapped by the framework.

::: warning Important
All Electron-related APIs cannot be used in child processes.
:::

### Usage
```javascript
import {
    getMainWindow,
    createMainWindow,
    restoreMainWindow,
    setCloseAndQuit,
    getCloseAndQuit,
} from 'ee-core/electron';
```

### API List

### getMainWindow()
Get the main window object (`BrowserWindow`, **non-null**). Throws Error if called before `WindowReady` event.

```javascript
const win = getMainWindow();
// All BrowserWindow instance APIs available
win.setSize(800, 600);
win.center();
```

::: tip Note
Changed from v5: `getMainWindow()` now returns `BrowserWindow` (not `BrowserWindow | null`). No need for null checks after `WindowReady` lifecycle event.
:::

### createMainWindow()
Create the Electron main window from config.windowsOption. Emits `WindowReady` event. Used by ee-core internally.

### restoreMainWindow()
Restore the main window if minimized, then show and focus it.

### setCloseAndQuit(flag)
- flag [Boolean] - true means the app will quit after window closes

Set a flag to control window exit behavior. Useful for tray icon scenarios.

### getCloseAndQuit()
Return the closeAndQuit flag (boolean).
