###  Introduction
Loader module. When JS modules/files maintained by developers need to be executed or called in business layers such as controller, service, jobs, preload, etc., there may be path or encryption issues. Using the loader can avoid these problems.
###  Usage
```javascript
const {
  loadFile,
  execFile,
  requireFile,
  resolveModule,
  getFullpath,
} = require('ee-core/loader');
```
###  API List
###  loadFile(filepath)

- filepath [String] - Absolute path of the file

Load a JS file, returns a module.

###  execFile(filepath, ...inject)

- filepath [String] - Absolute path of the file
- inject [] - Injected parameters

Load and run a JS file, returns the result. If the file is a class it will be instantiated, if it is a Function it will be called.

###  resolveModule(filepath)

- filepath [String] - File path

Return the absolute path of the file.

###  requireFile(filepath)

- filepath [String] - File path

Import a module

###  getFullpath(filepath)

- filepath [String] - File path

Get the absolute path of the file
