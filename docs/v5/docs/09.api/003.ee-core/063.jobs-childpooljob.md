### Introduction
ChildPoolJob is based on Node.js **child processes**. It can create a pool of processes that stay resident in memory, executing tasks directly upon receiving them, without the performance overhead of creating and destroying processes.

### Usage
```javascript
const { ChildPoolJob } = require('ee-core/jobs');

const pool = new ChildPoolJob();
```

### API List
### pool.children
Class property, returns the collection of instance objects created via run() / runPromise().

### pool.max
Class property, maximum number of processes

### pool.strategy
Class property, process selection algorithm, default 'polling' (round-robin)

### pool.weights
Class property, weights

### pool.LB
Class property, load balancer

### pool.create(number = 3)

- number [Int] - Number of processes to create

Create a pool, returns pids
```javascript
pool.create(3).then(pids => {
  // ...
});
```

### pool.run(filepath, params = {})

- filepath [String] - Relative path from the ./jobs directory; or an absolute path.
- params [Object] - Parameters to pass

Select a process from the pool, then execute a job file.
```javascript
# ./electron/service/example.js

const timerTask = pool.run('./jobs/example/timer', {jobId});
```

### pool.runPromise(filepath, params = {})
Async syntax for run()

### pool.getPids()
Get the current pids array.

### pool.getChildByPid(pid)

- pid [Int]

Get a child process object by pid

### pool.killAll()
Close all processes.

### EventEmitter
ChildPoolJob inherits from the events module, so instantiated objects can use all EventEmitter methods

### ForkProcess
The process object returned by run() / runPromise().
```javascript
const myProc = pool.run('./jobs/example/timer', {jobId});

# or
pool.runPromise('./jobs/example/timer', {jobId}).then(myProc => {
  let eventName = 'job-timer-progress-' + jobId;
  myProc.emitter.on(eventName, (data) => {
    // ...
  })
});
```

#### myProc.emitter
An instantiated [EventEmitter](https://nodejs.org/docs/latest-v16.x/api/events.html) module object.

- Method list:
```javascript
emitter.addListener(eventName, listener)
emitter.emit(eventName[, ...args])
emitter.eventNames()
emitter.getMaxListeners()
emitter.listenerCount(eventName)
emitter.listeners(eventName)
emitter.off(eventName, listener)
emitter.on(eventName, listener)
emitter.once(eventName, listener)
emitter.prependListener(eventName, listener)
emitter.prependOnceListener(eventName, listener)
emitter.removeAllListeners([eventName])
emitter.removeListener(eventName, listener)
emitter.setMaxListeners(n)
emitter.rawListeners(eventName)
emitter[Symbol.for('nodejs.rejection')](err, eventName[, ...args])
```

- Usage
```javascript
pool.runPromise('./jobs/example/timer', {jobId}).then(myProc => {
  let eventName = 'job-timer-progress-' + jobId;
  myProc.emitter.on(eventName, (data) => {
    // ...
  })
});
```

#### myProc.child
The process object forked via the [child_process](https://nodejs.org/docs/latest-v16.x/api/child_process.html#child_processforkmodulepath-args-options) module, [fork](https://nodejs.org/docs/latest-v16.x/api/child_process.html#child_processforkmodulepath-args-options).

#### myProc.pid
Child process pid

#### myProc.dispatch(cmd, jobPath = '', params = {})
Dispatch a task

#### myProc.kill()

- timeout [Int] - Default 1000ms

Kill the child process

#### myProc.sleep()
sleep (Unix platform only)

#### myProc.wakeup()
wakeup (Unix platform only)
