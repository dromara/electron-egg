## Introduction

Communication service module. Provides three transport modes for controller method routing:

- **SocketServer**: Socket.IO server for third-party software communication (Go, Python backends, etc.) via WebSocket
- **HttpServer**: Koa-based RESTful HTTP/HTTPS server for external API calls
- **IpcServer**: Electron IPC server for main вЖФ renderer process communication via `ipcMain`/`ipcRenderer`

All three share the `resolveControllerFn()` routing mechanism. Channel pattern: `controller/{name}/{method}`.

## SocketServer

Socket.IO server. Provides WebSocket/HTTP polling communication for third-party software (non-Electron renderer processes) to call main process controller methods.

Communication protocol:
- Client connects and sends messages on a specified channel: `{ cmd: 'controller/user/add', args: {...} }`
- Server parses the `cmd` path and calls the corresponding controller method
- Controller execution result is returned to client via callback

Created using factory pattern (`static create()`) because initialization requires async port acquisition. When `enable=false` in config, the service will not start after creation.

### SocketServer.create()
**жППињ∞**: Factory method вАФ create and initialize a SocketServer instance. Acquires an available port, creates a Socket.IO Server instance, and sets up connection listeners. The port number is written to `process.env.EE_SOCKET_PORT` for other modules to access.
**ињФеЫЮеАЉ**: `Promise<SocketServer>` вАФ fully initialized SocketServer instance.

### SocketServer.init()
**жППињ∞**: Initialize the Socket.IO server. If `config.enable === false`, returns immediately without starting. Otherwise acquires a port, creates the Socket.IO Server, and calls `connect()`.
**ињФеЫЮеАЉ**: `Promise<void>`

### SocketServer.connect()
**жППињ∞**: Listen for client connections and handle messages. After a client connects, listens on the configured channel (default `'socket-channel'`). Upon receiving a message, routes via `cmd` path to the corresponding controller method. Results are sent back to client via callback. Errors are returned as `{ __EE_ERROR__: true, message: string }`.
**ињФеЫЮеАЉ**: `void`

### SocketServer.io
**жППињ∞**: The instantiated Socket.IO Server object.
**з±їеЮЛ**: `Server | undefined` вАФ `undefined` before initialization or when `enable=false`.

Properties and methods follow the Socket.IO Server API: [https://socket.io/docs/v4/server-api/](https://socket.io/docs/v4/server-api/)

Key methods:
- `io.emit(eventName[, ...args])` вАФ broadcast to all connected clients
- `io.on(eventName, listener)` вАФ listen for server-level events
- `io.close([callback])` вАФ close the server

### SocketServer.socket
**жППињ∞**: The socket object established through `io.on('connection')`. Represents the currently connected client socket instance.
**з±їеЮЛ**: `ReturnType<Server['on']> | undefined` вАФ `undefined` until a client connects.

Socket.IO Socket API: [https://socket.io/docs/v4/server-api#socket](https://socket.io/docs/v4/server-api#socket)

### SocketServer.config
**жППињ∞**: Socket.IO server configuration object.
**з±їеЮЛ**: `SocketServerConfig` вАФ contains `enable`, `port`, `path`, `channel`, `cors`, `transports`, etc.

### SocketServer.channelSeparator
**жППињ∞**: Channel separator used for controller path routing.
**з±їеЮЛ**: `string` вАФ defaults to `'/'`.

## HttpServer

Koa-based HTTP/HTTPS server. Provides RESTful API services, routing HTTP requests to corresponding controller methods.

Request routing rules:
- URL path maps to controller path, e.g., `/controller/user/add` вЖТ `controller.user.add`
- GET request parameters from `query`, POST request parameters from `body`
- Paths starting with `'controller'` are used directly; otherwise `'controller'` prefix is added automatically
- URIs in `filterRequest` return specified data directly without going through controllers

Middleware loading order:
1. Error handler (`errorHandler`)
2. Pre-middleware (`preMiddleware`)
3. CORS cross-origin middleware
4. `koa-body` request body parsing middleware
5. Core routing dispatch middleware (`_dispatch`)
6. Post-middleware (`postMiddleware`)

Created using factory pattern (`static create()`). When `enable=false` in config, the service will not start after creation.

### HttpServer.create()
**жППињ∞**: Factory method вАФ create and initialize an HttpServer instance. Acquires an available port, creates a Koa application with middleware, and starts HTTP/HTTPS listening. The port number is written to `process.env.EE_HTTP_PORT` for other modules to access.
**ињФеЫЮеАЉ**: `Promise<HttpServer>` вАФ fully initialized HttpServer instance.

### HttpServer.init()
**жППињ∞**: Initialize the HTTP server. If `config.enable === false`, returns immediately. Otherwise acquires a port and calls `_create()`.
**ињФеЫЮеАЉ**: `Promise<void>`

### HttpServer._create()
**жППињ∞**: Create the Koa application and start HTTP/HTTPS service. Loads middleware in order, handles HTTPS/SSL configuration, and starts listening on the configured host and port.
**ињФеЫЮеАЉ**: `Promise<void>`

### HttpServer._dispatch(ctx, next)
**жППињ∞**: Core routing dispatch middleware. Maps HTTP request paths to controller methods:
1. Remove leading `'/'` from path
2. Check if path is in filter list (e.g., `favicon.ico`)
3. Ensure path starts with `'controller'`
4. Find and call controller method via `resolveControllerFn`
5. GET requests pass `query` params, POST requests pass `body` params

Error handling: When controller method throws an exception, returns HTTP 500 + `{ error: 'Internal Server Error' }`.
**еПВжХ∞**:

| еПВжХ∞ | з±їеЮЛ | ењЕе°Ђ | иѓіжШО |
|------|------|------|------|
| ctx | `Koa.Context` | жШѓ | Koa request context |
| next | `Koa.Next` | жШѓ | Next middleware function |

**ињФеЫЮеАЉ**: `Promise<void>`

### HttpServer.getHttpApp()
**жППињ∞**: Get the Koa application instance.
**ињФеЫЮеАЉ**: `Koa | undefined` вАФ Koa instance, or `undefined` if HTTP service is not enabled.
**з§ЇдЊЛ**:
```javascript

const httpApp = getHttpServer().getHttpApp();
```

### HttpServer._setupErrorHandler(app, errorHandler)
**жППињ∞**: Set up Koa error handler. Registers the provided function as a Koa `'error'` event listener.
**еПВжХ∞**:

| еПВжХ∞ | з±їеЮЛ | ењЕе°Ђ | иѓіжШО |
|------|------|------|------|
| app | `Koa` | жШѓ | Koa application instance |
| errorHandler | `((err: Error) => void) | null` | жШѓ | Error handler function |

**ињФеЫЮеАЉ**: `void`

### HttpServer._loadMiddlewares(app, middlewares, type)
**жППињ∞**: Load pre/post middleware. Middleware can be factory functions (called to return Koa middleware) or direct middleware functions. Invalid middleware will be skipped with a warning.
**еПВжХ∞**:

| еПВжХ∞ | з±їеЮЛ | ењЕе°Ђ | иѓіжШО |
|------|------|------|------|
| app | `Koa` | жШѓ | Koa application instance |

| type | `string` | еР¶ | Type identifier (`'pre'` or `'post'`), defaults to `'pre'` |

### HttpServer.config
**жППињ∞**: HTTP server configuration object.
**з±їеЮЛ**: `HttpServerConfig` вАФ contains `enable`, `port`, `host`, `protocol`, `https`, `cors`, `body`, `filterRequest`, `koaConfig`, etc.

### HttpServer.channelSeparator
**жППињ∞**: Channel separator used for controller path routing.
**з±їеЮЛ**: `string` вАФ defaults to `'/'`.

## IpcServer

Electron IPC server. Registers controller methods as `ipcMain` channel handlers, enabling renderer processes to call main process controller methods via `ipcRenderer`.

Supports two communication models:
- **send/on model** (synchronous): Renderer uses `ipcRenderer.sendSync()`, main process returns results via `event.returnValue`. Supports both sync and async controller methods (Electron's `sendSync` enters a nested message loop that processes microtasks).
- **invoke/handle model** (asynchronous, recommended): Renderer uses `ipcRenderer.invoke()`, main process returns a Promise via `ipcMain.handle()`. Supports both synchronous and asynchronous controller methods.

Channel naming rule: Replace `'.'` in controller path with `channelSeparator` (default `'/'`). Example: `controller.user.add` вЖТ `controller/user/add`.

### IpcServer.constructor()
**жППињ∞**: Create and initialize an IpcServer instance. Reads `channelSeparator` from config, then traverses the controller object tree and registers all IPC channels.
**ињФеЫЮеАЉ**: `IpcServer` вАФ initialized instance.

### IpcServer.init()
**жППињ∞**: Initialize вАФ traverse controller tree and register all IPC channel handlers. Called automatically by the constructor.
**ињФеЫЮеАЉ**: `void`

### IpcServer.loop(obj, pathname)
**жППињ∞**: Recursively traverse the controller object tree. Distinguished by `EXPORTS` symbol marker:
- Objects with `EXPORTS` marker вЖТ leaf nodes (controller method collections), call `register()`
- Objects without marker вЖТ intermediate nodes (directory/namespace), continue recursion
**еПВжХ∞**:

| еПВжХ∞ | з±їеЮЛ | ењЕе°Ђ | иѓіжШО |
|------|------|------|------|
| obj | `Record<string, unknown>` | жШѓ | Current object being traversed |
| pathname | `string` | жШѓ | Current property path |

**ињФеЫЮеАЉ**: `void`

### IpcServer.register(exportObj, propertyChain)
**жППињ∞**: Register controller methods as IPC channel handlers. For each method, registers two handlers:
1. `ipcMain.on` (send/on model): Sets `event.returnValue`, supports async methods. Errors are returned as `{ __EE_ERROR__: true, message: string }`.
2. `ipcMain.handle` (invoke/handle model): Returns Promise. Errors cause `ipcRenderer.invoke()` to reject on renderer side.
**еПВжХ∞**:

| еПВжХ∞ | з±їеЮЛ | ењЕе°Ђ | иѓіжШО |
|------|------|------|------|
| exportObj | `Record<string, unknown>` | жШѓ | Controller method collection object |
| propertyChain | `string` | жШѓ | Property path (e.g., `'controller.user'`) |

**ињФеЫЮеАЉ**: `void`

### IpcServer.channelSeparator
**жППињ∞**: Channel separator used for IPC channel name generation.
**з±їеЮЛ**: `string` вАФ defaults to `'/'`.

### IpcServer.directory
**жППињ∞**: Controller directory name, used as the property path prefix.
**з±їеЮЛ**: `string` вАФ `'controller'`.

## Factory Functions

Module-level functions that create, retrieve, and initialize communication service instances.

### loadSocket()
**жППињ∞**: Load and initialize all communication services. Created in order: SocketServer вЖТ HttpServer вЖТ IpcServer. Called by `Application.run()` during the framework startup flow.
**ињФеЫЮеАЉ**: `Promise<void>`
**еПВиІБ**: ps module вАФ `getSocketPort()` / `getHttpPort()` return the ports set during this initialization.

### createSocketServer()
**жППињ∞**: Create a SocketServer instance via `SocketServer.create()` and store it in the module instance registry.
**ињФеЫЮеАЉ**: `Promise<SocketServer>` вАФ the created SocketServer instance.

### createHttpServer()
**жППињ∞**: Create an HttpServer instance via `HttpServer.create()` and store it in the module instance registry.
**ињФеЫЮеАЉ**: `Promise<HttpServer>` вАФ the created HttpServer instance.

### createIpcServer()
**жППињ∞**: Create an IpcServer instance and store it in the module instance registry. Unlike SocketServer and HttpServer, IpcServer creation is synchronous.
**ињФеЫЮеАЉ**: `IpcServer` вАФ the created IpcServer instance.

### getSocketServer()
**жППињ∞**: Get the SocketServer instance. Throws Error if called before the `Ready` lifecycle event (i.e., before `createSocketServer()` has been called).
**ињФеЫЮеАЉ**: `SocketServer` вАФ the stored SocketServer instance (non-null).
**з§ЇдЊЛ**:
```javascript

const server = getSocketServer();
server.io.emit('channel-name', 'data');
```

::: tip
Changed from v5: `getSocketServer()` returns `SocketServer` (not `SocketServer | null`). Throws Error if called before services are created.
:::

### getHttpServer()
**жППињ∞**: Get the HttpServer instance. Throws Error if called before the `Ready` lifecycle event (i.e., before `createHttpServer()` has been called).
**ињФеЫЮеАЉ**: `HttpServer` вАФ the stored HttpServer instance (non-null).
**з§ЇдЊЛ**:
```javascript

```

::: tip
Changed from v5: `getHttpServer()` returns `HttpServer` (not `HttpServer | null`).
:::

### getIpcServer()
**жППињ∞**: Get the IpcServer instance. Throws Error if called before the `Ready` lifecycle event (i.e., before `createIpcServer()` has been called).
**ињФеЫЮеАЉ**: `IpcServer` вАФ the stored IpcServer instance (non-null).
**з§ЇдЊЛ**:

::: tip
