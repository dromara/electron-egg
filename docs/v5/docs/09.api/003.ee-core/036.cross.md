### Introduction
Cross-process service module. Manages external child processes (Go, Python, Java backends) with dynamic port allocation and dual-index tracking (pid and name).

### Usage
```javascript
import { cross, Cross } from 'ee-core/cross';
import type { CrossTargetConfig, CrossRunOptions } from 'ee-core/types';
```

### API List

### run(service, opt)
- service [String] - Service name (key in config.cross)
- opt [CrossRunOptions] - Runtime options, now `Partial<CrossTargetConfig>` (all fields optional)

Create a service (child) process. Returns a CrossProcess entity.

```javascript
import { cross } from 'ee-core/cross';
import type { CrossTargetConfig } from 'ee-core';

const opt: CrossTargetConfig = {
  name: 'goapp',
  cmd: path.join(getExtraResourcesDir(), 'goapp'),
  directory: getExtraResourcesDir(),
  args: ['--port=7073'],
  appExit: true,
};
const entity = await cross.run('go', opt);
```

### kill(pid)
- pid [Int] - Process ID

Kill a process.

### killByName(name)
- name [String] - Program process name

Kill a process by name.

### killAll()
Kill all processes.

### getUrl(name)
- name [String] - Program process name

Program service URL. Returns `string | undefined` if process not found.

### getProcByName(name)
- name [String] - Program process name

Get a process object by name. Throws Error if not found.

### getProc(pid)
- pid [Int | String] - Process ID

Get a process object by pid. Throws Error if not found.

### getPids()
Get all process pids, returns an array of strings.

### CrossProcess (entity)

### entity.pid
Process pid (number).

### entity.name
Process name (string).

### entity.config
Process configuration (CrossTargetConfig).

### entity.child
The process object spawned via [child_process.spawn](https://nodejs.org/api/child_process.html#child_processspawncommand-args-options).

### entity.port
Service port number. Only has a value if the child process uses a port.

### entity.emitter
An [EventEmitter](https://nodejs.org/api/events.html) instance for per-process events.

### entity.kill(timeout)
- timeout [Number] - Delay before force kill (default: 1000ms)

Kill the process. First sends SIGINT, then SIGKILL after timeout.

### entity.getUrl()
Program service URL (string).

### entity.getArgsObj()
Return opt.args as an object.
