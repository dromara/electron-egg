### Introduction
Electron functionality wrapped by the framework. Provides app/window management APIs.

::: warning Important
All Electron-related APIs cannot be used in child processes.
:::

### Usage
```javascript
import {
    electronApp,
    createElectron,
    getMainWindow,
    createMainWindow,
    restoreMainWindow,
    setCloseAndQuit,
    getCloseAndQuit,
    loadServer,
} from 'ee-core/electron';
```

### API List

### electronApp
The Electron `app` module, re-exported for convenience. Use `electronApp.getPath()`, `electronApp.getName()`, etc.

```javascript
import { electronApp } from 'ee-core/electron';
const exePath = electronApp.getPath('exe');
```

### createElectron()
Create Electron app, register system events, single instance lock. Used by ee-core internally.

### getMainWindow()
Return the main window object (`BrowserWindow`, **non-null**). Throws Error if called before window creation.

```javascript
import { getMainWindow } from 'ee-core/electron';
const win = getMainWindow();
win.setMenuBarVisibility(false);
```

::: tip Note
`getMainWindow()` returns `BrowserWindow` (not `BrowserWindow | null`). If called before `WindowReady` lifecycle event, it throws an Error indicating a lifecycle ordering bug.
:::

### createMainWindow()
Create the Electron main window from config.windowsOption. Used by ee-core internally.

### restoreMainWindow()
Restore, show, and focus the main window. Used when a second instance starts.

### setCloseAndQuit(flag)
- flag [Boolean] - true means the app will quit after window closes

Set a flag to control window exit behavior.

### getCloseAndQuit()
Return the closeAndQuit flag (boolean).

### loadServer()
Load page content based on environment (remote/dev/cross-takeover/production). Used by ee-core internally.
