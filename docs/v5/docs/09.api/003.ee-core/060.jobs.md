## Introduction
Background task module. Provides child process task management and load balancing capabilities for running tasks outside the main Electron process.

::: warning Important
All Electron-related APIs cannot be used in child processes.
:::

Core components:
- **ChildJob**: Single task execution, creates a new child process each time. Acquire on demand, release when done. Suitable for infrequent tasks. When creating tasks frequently and in large quantities, there is process startup overhead — use ChildPoolJob instead.
- **ChildPoolJob**: Process pool, maintains resident child processes, reuses them via load balancing. Avoids overhead of frequent process creation and destruction.
- **LoadBalancer**: Load balancer, supports 8 scheduling algorithms for distributing tasks across processes.
- **AlgorithmType**: Algorithm type constants.
- **Registry**: `registerJobManager()` / `killAllJobs()` for central cleanup on application exit.

## Import
Both CJS and ESM:
```javascript
// ESM
import { ChildJob, ChildPoolJob, LoadBalancer, AlgorithmType } from 'ee-core/jobs';
import type { LoadBalancerTarget, LoadBalancerParams, LoadBalancerOptions } from 'ee-core/jobs';
import type { JobProcessOptions, JobMessage, ProcessMessage } from 'ee-core/jobs';
import type { ChildPoolOptions } from 'ee-core/jobs';
// CJS
const { ChildJob, ChildPoolJob, LoadBalancer, AlgorithmType } = require('ee-core/jobs');
```

## ChildJob

### Introduction
Single-task child process manager. Creates a new `child_process.fork()` for each `exec()` call. The child process must call `Ps.exit()` to terminate after task completion, otherwise it remains running and wastes resources.

Advantages: Acquire on demand, release when done, does not excessively occupy computer resources.

Limitations: Process startup overhead when creating tasks frequently and in large quantities. Use ChildPoolJob for that scenario.

### Usage
```javascript
import { ChildJob } from 'ee-core/jobs';
const job = new ChildJob();
```

### API

#### job.jobs
**描述**: Property. Collection of active `JobProcess` instances keyed by PID. Processes are automatically removed when they exit or encounter errors.
**返回值**: `Record<number, JobProcess>`

#### job.exec(filepath, params, opt)
**描述**: Start a new child process to execute a job file. Resolves the filepath to an absolute path via `getFullpath()`, creates a `JobProcess`, and dispatches a `run` command.
**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| filepath | `string` | 是 | Relative path from the `./jobs` directory, or an absolute path |
| params | `Record<string, unknown>` | 否 | Parameters passed to the task |
| opt | `JobProcessOptions` | 否 | Process options (processArgs, processOptions) |

**返回值**: `JobProcess` — the child process instance
**示例**:
```javascript
const myProc = job.exec('./jobs/example/timer', { jobId: 'task-1' });
```

#### job.execPromise(filepath, params, opt)
**描述**: Async syntax for `exec()`. Creates a new child process to execute a job file. Returns the same `JobProcess` as `exec()`.
**参数**: Same as `exec()`
**返回值**: `Promise<JobProcess>` — the child process instance
**示例**:
```javascript
job.execPromise('./jobs/example/timer', { jobId }).then(task => {
  const eventName = 'job-timer-progress-' + jobId;
  task.emitter.on(eventName, (data) => {
    // handle progress
  });
});
```

#### job.createProcess(opt)
**描述**: Create a `JobProcess` instance without dispatching a command. Useful when you need a child process but want to send commands manually later.
**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| opt | `JobProcessOptions` | 否 | Process options (processArgs, processOptions) |

**返回值**: `JobProcess` — the newly created process instance

#### job.getPids()
**描述**: Get the PID list of all active child processes.
**参数**: None
**返回值**: `string[]` — array of PID strings

#### job.on(eventName, callback)
**描述**: Listen to an event emitted by child processes. ChildJob inherits from `EventEmitter`. Note: When multiple child processes exist simultaneously, it may be difficult to distinguish the source — use `JobProcess.emitter` for per-process events.
**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| eventName | `string` | 是 | Event name |
| callback | `Function` | 是 | Event handler |

**示例**:
```javascript
// In service: ./electron/service/example.js
const eventName = 'job-timer-progress-' + jobId;
job.on(eventName, (data) => {
  // handle data from child process
});

// In job: ./electron/jobs/timer.js
import { childMessage } from 'ee-core/message';
const eventName = 'job-timer-progress-' + jobId;
childMessage.sendToMain(eventName, { jobId, number });
```

### JobProcess (Entity)

The process object returned by `exec()`, `execPromise()`, or `createProcess()`.

#### myProc.emitter
**描述**: An `EventEmitter` instance for per-process (forkProcess-level) events. Messages from the child process with `eventReceiver: 'forkProcess'` are emitted here.
**返回值**: `EventEmitter`

#### myProc.child
**描述**: The `ChildProcess` object returned by `child_process.fork()`. See [Node.js child_process docs](https://nodejs.org/api/child_process.html#child_processforkmodulepath-args-options).
**返回值**: `ChildProcess`

#### myProc.pid
**描述**: Child process PID.
**返回值**: `number | undefined`

#### myProc.dispatch(cmd, jobPath, ...params)
**描述**: Send a dispatch message to the child process. Generates a unique message ID and sends a `JobMessage`.
**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| cmd | `string` | 是 | Command (e.g. `'run'`) |
| jobPath | `string` | 否 | Task file path (default `''`) |
| params | `unknown[]` | 否 | Additional parameters |

**返回值**: `void`

#### myProc.callFunc(jobPath, funcName, ...params)
**描述**: Call a specified method on the task instance in the child process. Resolves the task file path to an absolute path and sends a `run` command with `jobFunc` and `jobFuncParams`.
**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| jobPath | `string` | 否 | Task file path (relative paths resolved to absolute) |
| funcName | `string` | 否 | Method name to call on the task instance |
| params | `unknown[]` | 否 | Method arguments |

**返回值**: `void`
**示例**:
```javascript
myProc.callFunc('./jobs/example/timer', 'pause', param1, param2);
```

#### myProc.kill(timeout)
**描述**: Terminate the child process. First sends `SIGINT` for graceful exit, then sends `SIGKILL` to force termination after timeout.
**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| timeout | `number` | 否 | Timeout in ms to wait for graceful exit (default: 1000) |

**返回值**: `void`

## ChildPoolJob

### Introduction
Child process pool manager. Maintains a set of resident child processes, distributes tasks through a `LoadBalancer`. Avoids overhead of frequent process creation and destruction.

- Pool size: min=3, max=6 (configurable)
- Load balancing: supports multiple algorithms, default is `polling` (round-robin)
- Process binding: binds specific IDs to fixed processes via `boundMap` (e.g. session binding)
- Auto cleanup: child processes are automatically removed from the pool on exit/error

### Usage
```javascript
import { ChildPoolJob } from 'ee-core/jobs';
const pool = new ChildPoolJob();
await pool.create(3); // Create 3 child processes
```

### Constructor
```javascript
new ChildPoolJob(opt?)
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| opt | `ChildPoolOptions` | 否 | Pool options |

`ChildPoolOptions`:

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| weights | `number[]` | 否 | Weight values for each process (used by weighted load balancing algorithms) |

### API

#### pool.children
**描述**: Property. Collection of active `JobProcess` instances keyed by PID.
**返回值**: `Record<number, JobProcess>`

#### pool.boundMap
**描述**: Property. Binding map that maps `boundId` to PID. Ensures the same boundId is always assigned to the same child process (e.g. session binding).
**返回值**: `Map<string | number, number>`

#### pool.min
**描述**: Property. Minimum number of processes in the pool (default: 3).
**返回值**: `number`

#### pool.max
**描述**: Property. Maximum number of processes in the pool (default: 6).
**返回值**: `number`

#### pool.strategy
**描述**: Property. Current load balancing strategy name (default: `'polling'`).
**返回值**: `string`

#### pool.weights
**描述**: Property. Weight array for each process position. Default weight is 1; can be customized via constructor options.
**返回值**: `number[]`

#### pool.LB
**描述**: Property. The `LoadBalancer` instance used for process selection.
**返回值**: `LoadBalancer`

#### pool.create(number)
**描述**: Create a specified number of child processes and add them to the pool and load balancer. The count is limited by `max`; excess processes will not be created.
**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| number | `number` | 否 | Number of processes to create (default: 3) |

**返回值**: `Promise<string[]>` — list of all process PIDs after creation
**示例**:
```javascript
pool.create(3).then(pids => {
  // pids: ['12345', '12346', '12347']
});
```

#### pool.run(filepath, params)
**描述**: Select a child process from the pool via the load balancer and execute a job file.
**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| filepath | `string` | 是 | Relative path from the `./jobs` directory, or an absolute path |
| params | `Record<string, unknown>` | 否 | Parameters to pass to the task |

**返回值**: `Promise<JobProcess>` — the child process instance executing the task
**示例**:
```javascript
const timerTask = await pool.run('./jobs/example/timer', { jobId });
```

#### pool.runPromise(filepath, params)
**描述**: Alias for `run()`. Async syntax for executing a task file via the pool.
**参数**: Same as `run()`
**返回值**: `Promise<JobProcess>`

#### pool.getBoundChild(boundId)
**描述**: Get a child process bound to a specific ID. The same `boundId` always returns the same process, suitable for session binding scenarios. If the bound process no longer exists, a new one is assigned and the binding is updated.
**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| boundId | `string | number` | 是 | Binding identifier (e.g. user ID, session ID) |

**返回值**: `Promise<JobProcess>` — the bound child process instance

#### pool.getChildByPid(pid)
**描述**: Get a child process by its PID.
**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| pid | `number` | 是 | Process ID |

**返回值**: `JobProcess | null`

#### pool.getChild()
**描述**: Get an available child process from the pool. If the pool is empty, automatically creates one. Uses the load balancer to select a process.
**参数**: None
**返回值**: `Promise<JobProcess>` — the selected child process instance

#### pool.getPids()
**描述**: Get the PID list of all child processes in the pool.
**参数**: None
**返回值**: `string[]` — array of PID strings

#### pool.killAll(type)
**描述**: Kill all child processes in the pool.
**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| type | `'parallel' | 'sequence'` | 否 | Kill mode: `'parallel'` kills all at once (default), `'sequence'` kills one per second |

**返回值**: `void`

#### pool.on(eventName, callback)
**描述**: Listen to pool-level events. ChildPoolJob inherits from `EventEmitter`. For per-process events, use `JobProcess.emitter`.

### JobProcess (Entity)
Same entity as described in the ChildJob section above. Returned by `run()`, `runPromise()`, `getBoundChild()`, `getChild()`.

## LoadBalancer

### Introduction
Load balancer for distributing tasks across child processes. Supports 8 scheduling algorithms. Based on the electron-re project with modifications.

### Usage
```javascript
import { LoadBalancer, AlgorithmType } from 'ee-core/jobs';

const lb = new LoadBalancer({
  targets: [{ id: 1, weight: 1 }, { id: 2, weight: 2 }],
  algorithm: AlgorithmType.polling,
});
```

### Constructor
```javascript
new LoadBalancer(options)
```

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| options | `LoadBalancerOptions` | 是 | Configuration options |
| options.targets | `LoadBalancerTarget[]` | 是 | Target list `[{id, weight}]` |
| options.algorithm | `string` | 否 | Algorithm name (default: `AlgorithmType.polling`) |

`LoadBalancerTarget`:

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | `string | number` | 是 | Target identifier (usually child process PID) |
| weight | `number` | 否 | Weight value (used by weighted algorithms) |

`LoadBalancerOptions`:

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| targets | `LoadBalancerTarget[]` | 是 | Target list |
| algorithm | `string` | 否 | Algorithm name (default: `'polling'`) |

### API

#### LoadBalancer.Algorithm
**描述**: Static property. Reference to the `AlgorithmType` constants object for easy access to algorithm names.
**返回值**: `AlgorithmType`

#### lb.pickOne(...params)
**描述**: Elect a single target process using the configured algorithm.
**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| params | `unknown[]` | 否 | Additional parameters for the algorithm (e.g. ID for `specify` algorithm) |

**返回值**: `LoadBalancerTarget | null` — the selected target, or null if no targets available

#### lb.pickMulti(count, ...params)
**描述**: Elect multiple target processes by calling `pickOne()` repeatedly.
**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| count | `number` | 否 | Number of targets to select (default: 1) |
| params | `unknown[]` | 否 | Additional parameters passed to each `pickOne()` call |

**返回值**: `(LoadBalancerTarget | null)[]` — array of selected targets

#### lb.add(task)
**描述**: Add a target to the load balancer. If the target ID already exists, logs a warning and does nothing. Recalculates weight totals after adding.
**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| task | `LoadBalancerTarget` | 是 | Target to add `{id, weight}` |

**返回值**: `void`

#### lb.del(target)
**描述**: Delete a target from the load balancer. Removes from targets list, cleans up connection/occupancy maps, recalculates indices and weights. If the target ID is not found, logs a warning.
**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| target | `LoadBalancerTarget` | 是 | Target to delete `{id, weight}` |

**返回值**: `void`

#### lb.wipe()
**描述**: Remove all targets and reset all internal state (connections, occupancy maps, indices).
**参数**: None
**返回值**: `void`

#### lb.setTargets(targets)
**描述**: Replace the entire targets list. Cleans up state for targets that are no longer in the new list. Recalculates weight totals.
**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| targets | `LoadBalancerTarget[]` | 是 | New targets list |

**返回值**: `void`

#### lb.setAlgorithm(algorithm)
**描述**: Change the scheduling algorithm. Resets weight index. Throws an error if the algorithm name is not in `AlgorithmType`.
**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| algorithm | `string` | 是 | Algorithm name from `AlgorithmType` |

**返回值**: `void`
**示例**:
```javascript
lb.setAlgorithm(AlgorithmType.weightsPolling);
```

#### lb.refreshParams(pidMap)
**描述**: Refresh CPU and memory occupancy data from process info. Used by minimum-connection and weighted-minimum-connection algorithms.
**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| pidMap | `Record<string | number, PidInfo>` | 是 | Process info map with CPU/memory data |

`PidInfo`:

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| pid | `string | number` | 是 | Process ID |
| cpu | `number` | 是 | CPU usage |
| memory | `number` | 是 | Memory usage |

**返回值**: `void`

#### lb.updateParams(object)
**描述**: Update specific load balancer runtime parameters (currentIndex, weightIndex, weightTotal, connectionsMap, etc.). Only updates keys that exist in `LoadBalancerParams`.
**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| object | `Partial<LoadBalancerParams>` | 是 | Parameters to update |

**返回值**: `void`

#### lb.clean(id)
**描述**: Clean up internal state. If `id` is provided, removes connection/occupancy data for that specific target. If no `id`, resets all internal maps while preserving weight totals.
**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | `string | number` | 否 | Target ID to clean (omit to clean all) |

**返回值**: `void`

## Constants

### AlgorithmType
**描述**: Load balancing algorithm type constants. Used to configure the `LoadBalancer` algorithm and `ChildPoolJob` strategy.

| Constant | Value | Description |
|----------|-------|-------------|
| `AlgorithmType.polling` | `'polling'` | Round-robin: select the next target in sequence |
| `AlgorithmType.weights` | `'weights'` | Weighted: select by weight ratio |
| `AlgorithmType.random` | `'random'` | Random: randomly select a target |
| `AlgorithmType.specify` | `'specify'` | Specify: select a target by ID (requires `id` param) |
| `AlgorithmType.weightsPolling` | `'weightsPolling'` | Weighted round-robin: combines weight and sequential selection |
| `AlgorithmType.weightsRandom` | `'weightsRandom'` | Weighted random: combines weight and random selection |
| `AlgorithmType.minimumConnection` | `'minimumConnection'` | Least connections: select the target with fewest current connections |
| `AlgorithmType.weightsMinimumConnection` | `'weightsMinimumConnection'` | Weighted least connections: combines weight and least connections |

**示例**:
```javascript
import { AlgorithmType } from 'ee-core/jobs';

// Use in LoadBalancer
const lb = new LoadBalancer({
  targets: [{ id: 1, weight: 2 }, { id: 2, weight: 1 }],
  algorithm: AlgorithmType.weightsPolling,
});

// Use in ChildPoolJob
const pool = new ChildPoolJob({ weights: [2, 1, 1] });
pool.LB.setAlgorithm(AlgorithmType.weightsRandom);
```

## Registry

### registerJobManager(job)
**描述**: Register a job manager instance (ChildJob or ChildPoolJob) for centralized cleanup. Called after creating a ChildJob or ChildPoolJob to ensure all child processes are properly terminated when the application exits.
**参数**:

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| job | `Killable` | 是 | Job manager instance (must have `killAll()` method, or `getPids()` + `kill()` methods) |

**返回值**: `void`
**示例**:
```javascript
import { ChildJob, registerJobManager } from 'ee-core/jobs';

const job = new ChildJob();
registerJobManager(job);
```

### killAllJobs()
**描述**: Kill all registered job managers. Called in the Electron `before-quit` event to ensure all child processes are properly terminated and prevent process leaks. Prefers `killAll()` method; falls back to killing individual processes via `getPids()` + `kill()`.
**参数**: None
**返回值**: `void`
**示例**:
```javascript
import { killAllJobs } from 'ee-core/jobs';

// Called during app quit lifecycle
killAllJobs();
```
**参见**: Registered automatically during framework lifecycle via `before-quit` event

## Types

### JobProcessOptions
Options when creating a JobProcess.

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| processArgs | `Record<string, unknown>` | 否 | Arguments object passed to the child process |
| processOptions | `ForkOptions` | 否 | Fork options (cwd, env, stdio, etc.) |

### JobMessage
Message format: Main process -> Child process.

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| mid | `string` | 是 | Message ID (random string) |
| cmd | `string` | 是 | Command (e.g. `'run'`) |
| jobPath | `string` | 否 | Task file path |
| jobParams | `unknown[]` | 否 | Task constructor arguments |
| jobFunc | `string` | 否 | Instance method name to call |
| jobFuncParams | `unknown[]` | 否 | Instance method arguments |

### ProcessMessage
Message format: Child process -> Main process. Extends `MessageData`.

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| eventReceiver | `string` | 是 | Message receiver type: `'forkProcess'`, `'childJob'`, or `'all'` |
| event | `string` | 是 | Event name |
| data | `unknown` | 否 | Event data |

### LoadBalancerParams
Load balancer runtime parameters.

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| currentIndex | `number` | 是 | Current polling index |
| weightIndex | `number` | 是 | Current weight index (for weighted round-robin) |
| weightTotal | `number` | 是 | Sum of all weights |
| connectionsMap | `Record<string | number, number>` | 是 | Connections map: pid -> current connection count |
| cpuOccupancyMap | `Record<string | number, number>` | 是 | CPU occupancy map: pid -> CPU usage |
| memoryOccupancyMap | `Record<string | number, number>` | 是 | Memory occupancy map: pid -> memory usage |
