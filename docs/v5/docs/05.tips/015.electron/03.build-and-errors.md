`electron` build and error summary

### Common Issues
1. Project directories must not contain Chinese characters, otherwise build dependencies cannot be downloaded, leading to build failures
2. When packaging the application, it must not be running. Close it from the task manager first, otherwise the build will fail
3. asar encryption ignore configuration for packaging
```javascript
"asarUnpack": [
  "**/node_modules/iconv-lite/**/*",
  "**/node_modules/ip/**/*",
  "**/node_modules/ws/**/*",
  "**/node_modules/koa/**/*",
  "**/node_modules/koa-router/**/*",
  "**/node_modules/koa-static/**/*",
  "**/node_modules/path/**/*",
  "**/web/**/*"
],
"extraResources": [
  {
    "from": "./web",
    "to": "app.asar.unpacked/web"
  }
],
```
4. Disable asar encryption for packaging
```javascript
"build": { "asar": false }
```

5. Add packaging file configuration
```javascript
"build": {
    "files": [
      "web/**",
      "main.js",
      "preload.js",
      "config.js"
    ]
}
```

6. Error message `was compiled against a different Node.js version using`

Cause: Node.js version mismatch with the Electron version
```javascript
// Check versions in Electron
console.log("node:", process.versions.node)
console.log("electron:", process.versions.electron)
console.log("modules:", process.versions.modules)

// npm rebuild --runtime=electron --target=15.3.0 --disturl=https://atom.io/download/atom-shell --abi=98 
// 15.3.0 is your Electron version, 98 is the modules version
// node 16.5.0 electron 15.3.0 modules 98 (this version resolved successfully)
```

7. electron-builder packaging command error
```javascript
Error: C:\Users\Administrator\AppData\Local\electron-builder\Cache\nsis\nsis-

3.0.3.2\Bin\makensis.exe exited with code 1Output:
```
Solution: The Electron project path must not contain Chinese characters

8. Packaging with support for Chinese file paths

Solution: Open the file `C:\Users\Administrator\AppData\Roaming\npm\node_modules\electron-builder\node_modules\app-builder-lib\out\targets\nsis/NsisTarget.js` and add the required parameter `args.push("-INPUTCHARSET", "UTF8");` in the `executeMakensis` method.

9. Error `Fatal error: Unable to commit changes`

Cause: Antivirus software is running or the C drive has insufficient storage space

Solution: Disable all antivirus software, clean up disk space, and rebuild.

10. Application crashes, unable to open

Error `gpu_data_manager_impl_private.cc(231)] SetSecurityInfo returned 5`

Solution: Uninstall 360 security software

### Missing Files During Packaging
```javascript
cannot resolve https://xxxxx.org/mirrors/electron/15.3.4/electron-v15.3.4-win32-x64.zip: status code 404
```

- Failed to download `electron-v.xxxx.zip` during packaging

Solutions:
1. Download the corresponding version and platform file directly from the source repository.
2. After downloading, place it in the `C:\Users\<username>\AppData\Local\electron\Cache` directory

#### Failed to download `winCodeSign-v.xxx.7z` during packaging

1. Download URL: [winCodeSign-2.5.0](https://github.com/electron-userland/electron-builder-binaries/releases/download/winCodeSign-2.5.0/winCodeSign-2.5.0.7z)
2. After downloading, extract and place in the `C:\Users\<username>\AppData\Local\electron-builder\Cache\winCodeSign` directory

#### Failed to download nsis-v.xxx.7z during packaging

1. Download URL: [nsis-3.0.4.1](https://github.com/electron-userland/electron-builder-binaries/releases/download/nsis-3.0.4.1/nsis-3.0.4.1.7z) 
2. After downloading, extract and place in `C:\Users\<username>\AppData\Local\electron-builder\Cache\nsis`

#### Failed to download nsis-resources-v.xxx.7z during packaging

1. Download URL: [nsis-resources-3.4.1](https://github.com/electron-userland/electron-builder-binaries/releases/download/nsis-resources-3.4.1/nsis-resources-3.4.1.7z)

2. After downloading, extract and place in `C:\Users\<username>\AppData\Local\electron-builder\Cache\nsis\nsis-resources-3.4.1`

::: tip
If you need a different version of the downloaded file, simply change the version number in the URL.
:::
