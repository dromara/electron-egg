electron-egg 框架的窗口管理模块负责创建 BrowserWindow、加载页面内容，并处理页面加载逻辑。通过 `windowsOption` 配置，开发者可以完全控制窗口的行为和外观。

## windowsOption 配置

BrowserWindow 的所有配置通过 `windowsOption` 传入：

```javascript
// config.default.js
module.exports = {
  openDevTools: false,
  windowsOption: {
    width: 960,
    height: 640,
    minWidth: 600,
    minHeight: 400,
    maxWidth: 1920,
    maxHeight: 1080,

    title: 'My Application',
    icon: '/public/images/icon.png',

    show: false,                // 创建时不显示（防白屏）
    center: true,               // 屏幕居中
    resizable: true,            // 可调整大小
    movable: true,              // 可移动
    minimizable: true,          // 可最小化
    maximizable: true,          // 可最大化
    closable: true,             // 可关闭
    alwaysOnTop: false,         // 不总是置顶
    fullscreen: false,          // 不全屏
    kiosk: false,               // 不锁定模式

    frame: true,                // 有边框
    titleBarStyle: 'default',   // 标题栏风格（macOS: default/hidden/hiddenInset）
    transparent: false,         // 不透明

    backgroundColor: '#fff',    // 背景色

    webPreferences: {
      contextIsolation: true,    // 上下文隔离（必须）
      nodeIntegration: false,   // 禁止 Node.js（必须）
      sandbox: true,            // 沙箱模式
      preload: path.join(app.getAppPath(), 'electron/preload/bridge.js'),
      webSecurity: true,        // Web 安全策略
      devTools: true,           // 开发者工具
    },
  },
};
```

### 关键字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `show` | `boolean` | 创建时是否立即显示。建议 `false` 配合防白屏策略 |
| `frame` | `boolean` | 是否显示窗口边框。`false` 为无边框窗口 |
| `titleBarStyle` | `string` | macOS 标题栏风格：`default`/`hidden`/`hiddenInset` |
| `transparent` | `boolean` | 窗口是否透明（配合无边框制作自定义 UI） |
| `backgroundColor` | `string` | 窗口背景色（减少白屏闪烁） |
| `contextIsolation` | `boolean` | 上下文隔离（必须为 true） |
| `nodeIntegration` | `boolean` | Node.js 集成（必须为 false） |
| `preload` | `string` | preload 脚本路径 |
| `minWidth/minHeight` | `number` | 最小窗口尺寸 |
| `maxWidth/maxHeight` | `number` | 最大窗口尺寸 |
| `alwaysOnTop` | `boolean` | 是否总是置顶 |
| `fullscreen` | `boolean` | 是否全屏 |
| `kiosk` | `boolean` | Kiosk 模式（全屏锁定） |

## 多窗口创建模式

electron-egg 默认创建一个主窗口，但开发者可以根据需要创建额外的窗口：

```javascript
// electron/controller/window.js
const { BrowserWindow } = require('electron');

class WindowController {
  // 创建辅助窗口
  async createAssistant(args) {
    const win = new BrowserWindow({
      width: 400,
      height: 300,
      parent: getMainWindow(),  // 设置父窗口
      modal: false,             // 非模态窗口
      show: true,
      webPreferences: {
        contextIsolation: true,
        nodeIntegration: false,
      },
    });

    win.loadURL('https://assistant.example.com');
    return { windowId: win.id };
  }

  // 创建设置窗口
  async openSettings(args) {
    const win = new BrowserWindow({
      width: 800,
      height: 600,
      title: 'Settings',
      show: true,
      webPreferences: {
        contextIsolation: true,
        nodeIntegration: false,
        preload: path.join(__dirname, '../preload/bridge.js'),
      },
    });

    win.loadFile(path.join(__dirname, '../settings.html'));
    return { windowId: win.id };
  }
}
```

<Note>
子窗口通过 `parent` 选项设置父窗口关联。模态窗口（`modal: true`）会阻止用户操作父窗口直到子窗口关闭。
</Note>

## 无边框窗口

通过 `frame: false` 创建无边框窗口，配合自定义 UI 实现独特外观：

```javascript
// config.default.js
module.exports = {
  windowsOption: {
    frame: false,              // 无边框
    transparent: true,         // 透明背景
    backgroundColor: '#00000000',
    titleBarStyle: 'hiddenInset',  // macOS 隐藏标题栏但保留控制按钮
    webPreferences: {
      contextIsolation: true,
      nodeIntegration: false,
    },
  },
};
```

### 自定义拖拽区域

无边框窗口需要通过 CSS 定义可拖拽区域：

```css
/* 前端 CSS */
.title-bar {
  -webkit-app-region: drag;     /* 可拖拽区域 */
  height: 40px;
  background: #333;
}

.title-bar-button {
  -webkit-app-region: no-drag;  /* 不可拖拽的按钮 */
}

.no-drag {
  -webkit-app-region: no-drag;  /* 其他不可拖拽区域 */
}
```

<Warning>
无边框窗口需要自行实现窗口控制按钮（最小化、最大化、关闭）。如果没有自定义拖拽区域，窗口将无法移动。
</Warning>

## 防白屏策略

electron-egg 使用 `show: false` + 加载页面 的策略防止白屏：

### 工作流程

```
1. new BrowserWindow({ show: false }) → 创建窗口但不显示
2. win.loadFile(loading.html) → 先加载启动页面
3. 页面加载完成 → win.show() → 显示窗口
4. 主页面就绪 → win.loadURL(mainUrl) → 加载正式页面
```

### 开发环境启动页面

```javascript
// loadServer() 中的启动页面逻辑（开发环境）
if (load === 'url') {
  // 加载启动页面
  _loadingPage('public/html/loading.html');

  // 等待前端开发服务器就绪
  const frontendReady = await waitForUrl(url, { retries: 60 });

  if (!frontendReady) {
    // 前端未就绪 → 加载失败页面
    win.loadFile('public/html/failure.html');
    return;
  }

  // 前端就绪 → 加载正式页面
  loadMainUrl('spa', url);
}
```

<Note>
`_loadingPage()` 在窗口创建后立即加载一个轻量级的 HTML 页面，避免用户看到空白窗口。当主页面内容就绪后，再加载正式内容。
</Note>

### 生产环境启动页面

```javascript
// config.default.js
module.exports = {
  mainServer: {
    loadingPage: '/public/html/loading.html',  // 启动等待页面路径
  },
};
```

## 窗口状态恢复

`restoreMainWindow()` 用于在第二个实例启动时激活已有实例的窗口：

```javascript
// electron/app/index.ts - 单实例锁逻辑
const gotTheLock = electronApp.requestSingleInstanceLock();
if (singleLock && !gotTheLock) {
  electronApp.quit();  // 另一个实例已运行，退出当前实例
  return;
}

// 第二个实例启动时，激活已有实例的窗口
electronApp.on('second-instance', () => {
  restoreMainWindow();
});
```

### restoreMainWindow 实现

```javascript
export function restoreMainWindow(): void {
  if (Instance.mainWindow) {
    if (Instance.mainWindow.isMinimized()) {
      Instance.mainWindow.restore();  // 从最小化恢复
    }
    Instance.mainWindow.show();       // 显示窗口
    Instance.mainWindow.focus();      // 聚焦窗口
  }
}
```

## closeAndQuit 标志

`closeAndQuit` 控制窗口关闭后是否退出整个应用：

```javascript
// 默认值
Instance.closeAndQuit = true;

// 设置标志
setCloseAndQuit(true);   // 窗口关闭 → 应用退出
setCloseAndQuit(false);  // 窗口关闭 → 应用继续运行

// 获取标志
getCloseAndQuit();  // true / false
```

### 使用场景

```javascript
// 在 before-quit 事件中设置标志
electronApp.on('before-quit', () => {
  setCloseAndQuit(true);  // 确认应用退出
});

// 窗口关闭事件中检查标志
mainWindow.on('close', (event) => {
  if (!getCloseAndQuit()) {
    event.preventDefault();  // 阻止关闭，应用继续运行（macOS 行为）
  }
});
```

<Note>
macOS 上通常不需要在所有窗口关闭后退出应用（用户可以通过 Dock 重新打开窗口）。而在 Windows 和 Linux 上，所有窗口关闭意味着用户想退出应用。
</Note>

## window-all-closed 事件

`window-all-closed` 事件在所有窗口关闭时触发：

```javascript
// electron/app/index.ts
electronApp.on('window-all-closed', () => {
  if (!is.macOS()) {
    coreLogger.info('[lib/eeApp] window-all-closed quit');
    electronApp.quit();  // 非 macOS 平台直接退出
  }
  // macOS 保持应用运行，等待用户通过 Dock 重新打开
});
```

<Note>
macOS 的应用行为与其他平台不同：关闭所有窗口后，应用仍可在 Dock 中保持活跃状态，用户可以重新创建窗口。这是 macOS 的标准用户体验模式。
</Note>

## before-quit 事件

`before-quit` 事件在应用退出前触发，用于清理资源：

```javascript
// electron/app/index.ts
electronApp.on('before-quit', () => {
  setCloseAndQuit(true);
  eventBus.emitLifecycle(BeforeClose);  // 触发 before-close 生命周期事件
  cross.killAll();     // 终止所有跨进程服务（Go/Python 后端）
  killAllJobs();       // 终止所有子任务进程
});
```

<Warning>
before-quit 事件中必须清理所有子进程和外部进程。如果没有正确清理，会导致孤儿进程在应用退出后继续运行。
</Warning>

## 完整配置示例

```javascript
// config.default.js - 窗口管理完整配置
module.exports = {
  openDevTools: false,  // false / true / { mode: 'bottom' }

  singleLock: true,  // 单实例锁（防止多开）

  windowsOption: {
    width: 960,
    height: 640,
    minWidth: 600,
    minHeight: 400,

    show: false,     // 防白屏：创建时不显示
    center: true,

    title: 'My App',
    icon: '/public/images/icon.png',
    backgroundColor: '#ffffff',

    webPreferences: {
      contextIsolation: true,
      nodeIntegration: false,
      sandbox: true,
      preload: path.join(app.getAppPath(), 'electron/preload/bridge.js'),
    },
  },

  mainServer: {
    protocol: 'http://',
    indexPath: 'index.html',
    loadingPage: '/public/html/loading.html',
    takeover: '',  // 空字符串表示不使用接管模式
    options: {},   // loadURL/loadFile 选项
  },
};
```
