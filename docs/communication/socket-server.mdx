SocketServer 是 electron-egg 框架内置的 SocketIO 服务器，专为第三方软件与 Electron 主进程通信而设计。与 IpcServer（主进程与渲染进程通信）不同，SocketServer 支持本地其他进程（Go/Python 后端）和远程客户端通过 WebSocket/HTTP 轮询与主进程交互。

## 适用场景

- **本地后端通信**：Go/Python 后端服务与 Electron 主进程交互
- **远程控制面板**：远程管理终端操作应用程序
- **跨进程协作**：非 Electron 渲染进程的程序调用主进程控制器方法

<Note>
SocketServer 不用于渲染进程通信。渲染进程应使用 IpcServer 的 ipcRenderer 通道。SocketServer 主要服务于非 Electron 环境（Go 程序、Python 脚本、远程浏览器等）。
</Note>

## 通信协议

SocketServer 使用 `cmd + args` JSON 格式进行通信：

```json
{
  "cmd": "controller/user/add",
  "args": { "name": "Alice", "age": 25 }
}
```

- **cmd**：控制器方法路径（格式与 IPC 通道命名规则一致）
- **args**：传递给控制器方法的参数对象

服务端解析 cmd 路径，调用对应的控制器方法，并通过回调函数返回执行结果。

## SocketIO 配置

```javascript
// config.default.js
module.exports = {
  socketServer: {
    enable: true,
    port: 7071,              // 端口号（自动动态分配可用端口）
    channel: 'socket-channel', // 自定义通信频道名称
    path: '/socket.io',      // SocketIO 服务路径
    transports: ['websocket', 'polling'],  // 传输方式
    cors: {
      origin: '*',
      methods: ['GET', 'POST'],
    },
    pingTimeout: 60000,      // 心跳超时时间（毫秒）
    maxHttpBufferSize: 1e6,  // 最大消息缓冲区大小
  },
};
```

### 配置项说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `enable` | `true` | 是否启用 SocketServer |
| `port` | `7071` | 端口号，运行时动态分配 |
| `channel` | `'socket-channel'` | SocketIO 通信频道 |
| `transports` | `['websocket', 'polling']` | 传输方式 |
| `cors` | `{ origin: '*' }` | CORS 跨域配置 |
| `pingTimeout` | `60000` | 心跳超时时间 |
| `maxHttpBufferSize` | `1e6` | 最大消息缓冲区 |

## 频道自定义

SocketServer 支持自定义通信频道名称。默认使用 `'socket-channel'`，可通过配置修改：

```javascript
// config.default.js
module.exports = {
  mainServer: {
    channelSeparator: '/',  // cmd 路径分隔符
  },
  socketServer: {
    channel: 'my-custom-channel',  // 自定义频道名称
  },
};
```

<Note>
频道名称需要客户端和服务端一致。如果修改了频道名称，客户端连接时必须使用相同的频道名称来发送/接收消息。
</Note>

## 连接与断开事件

SocketServer 在客户端连接时监听指定频道上的消息，客户端断开时 SocketIO 自动清理连接。

```javascript
// 服务端连接处理（源码简化）
io.on('connection', (socket) => {
  console.log('客户端已连接:', socket.id);

  socket.on('socket-channel', async (message, callback) => {
    const result = await fn.call(controller, message.args);
    if (callback) callback(result);  // 通过回调返回结果
  });

  socket.on('disconnect', () => {
    console.log('客户端已断开:', socket.id);
  });
});
```

## 端口动态分配

与 HttpServer 一样，SocketServer 使用工厂模式创建并动态分配端口：

```javascript
// 启动时获取可用端口
const port = await getPort({ port: this.config.port });
process.env.EE_SOCKET_PORT = String(port);  // 写入环境变量
this.config.port = port;
```

端口信息写入 `process.env.EE_SOCKET_PORT`，其他进程可以读取该变量获取实际端口。

## Go 客户端实现示例

以下是一个 Go 程序通过 SocketIO 与 Electron 主进程通信的示例：

```go
// Go 客户端示例
package main

import (
    "encoding/json"
    "fmt"
    "github.com/zishang520/socket.io/v2/socket"
)

func main() {
    client := socket.NewClient(&socket.ClientConfig{
        Transports: []string{"websocket"},
    })

    client.On("connection", func(clients ...interface{}) {
        fmt.Println("已连接到 Electron 主进程")

        // 调用控制器方法
        cmd := map[string]interface{}{
            "cmd": "controller/user/add",
            "args": map[string]interface{}{
                "name": "Alice",
                "age": 25,
            },
        }

        client.Emit("socket-channel", cmd, func(result ...interface{}) {
            fmt.Printf("调用结果: %v\n", result[0])
        })
    })

    client.Connect("http://127.0.0.1:7071")
}
```

## Python 客户端实现示例

```python
# Python 客户端示例
import socketio

sio = socketio.Client()

@sio.on('connect')
def on_connect():
    print('已连接到 Electron 主进程')
    # 调用控制器方法
    sio.emit('socket-channel', {
        'cmd': 'controller/user/add',
        'args': {'name': 'Alice', 'age': 25}
    }, callback=lambda result: print(f'调用结果: {result}'))

@sio.on('disconnect')
def on_disconnect():
    print('已断开连接')

sio.connect('http://127.0.0.1:7071')
sio.wait()
```

## Node.js 客户端实现示例

```javascript
// Node.js 客户端示例
const { io } = require('socket.io-client');

const socket = io('http://127.0.0.1:7071', {
  transports: ['websocket'],
});

socket.on('connect', () => {
  console.log('已连接到 Electron 主进程');

  // 调用控制器方法
  socket.emit('socket-channel', {
    cmd: 'controller/user/add',
    args: { name: 'Alice', age: 25 },
  }, (result) => {
    console.log('调用结果:', result);
  });
});

socket.on('disconnect', () => {
  console.log('已断开连接');
});
```

## 错误处理

当控制器方法不存在或执行出错时，SocketServer 会在日志中记录错误信息：

```javascript
// 源码中的错误处理
try {
  const fn = resolveControllerFn(controller, cmd, this.channelSeparator);
  if (!fn) throw new Error('function not exists');
  const result = await fn.call(controller, args);
  if (callback) callback(result);
} catch (err) {
  coreLogger.error('[socket/socketServer] throw error:', err);
}
```

<Warning>
当前版本中，SocketServer 在控制器执行出错时不会通过 callback 返回错误信息给客户端，仅在服务端日志记录。如需将错误信息传递给客户端，需要自定义错误处理逻辑。
</Warning>
