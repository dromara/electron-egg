`ee-bin move` 命令负责将构建产物从源目录复制到目标目录，最典型的用途是将前端构建输出（`frontend/dist`）复制到 `public/dist`，使 Electron 能以静态资源方式加载前端页面。

<Note>
尽管命令名为 "move"，实际执行的是复制操作（copy）。源文件在操作完成后仍然保留在原位置，不会被删除。
</Note>

## MoveConfig 配置参考

`cmd/bin.js` 中的 `move` 配置段定义了资源复制规则：

```js
module.exports = {
  move: {
    frontend_dist: {
      src: './frontend/dist',
      dest: './public/dist',
    },
  },
};
```

### 字段详解

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `src` | `string` | 是 | 源路径（相对于项目根目录） |
| `dest` | `string` | 是 | 目标路径（相对于项目根目录） |
| `dist` | `string?` | 否 | `src` 的别名覆盖，设置后优先于 `src` |
| `target` | `string?` | 否 | `dest` 的别名覆盖，设置后优先于 `dest` |

### dist 和 target 别名

`dist` 和 `target` 是 `src` 和 `dest` 的别名字段，设置后会覆盖对应的原始字段：

```js
move: {
  frontend_dist: {
    src: './frontend/dist',       // 基础值
    dest: './public/dist',        // 基础值
    dist: './frontend/dist-new',  // 覆盖 src
    target: './public/dist-new',  // 覆盖 dest
  },
}
```

解析逻辑：

```js
const source = dist || src;        // dist 优先于 src
const destination = target || dest; // target 优先于 dest
```

<Note>
别名机制允许在不同环境下覆盖路径而不修改基础配置。例如在 CI 环境中通过环境变量动态设置 `dist` 和 `target`。
</Note>

## 原子化复制策略

为防止复制过程中出错导致数据丢失，ee-bin 实现了近似原子化的复制策略：

<Steps>
### 检查源目录

验证源路径是否存在。不存在时跳过并打印警告：

```js
if (!fs.existsSync(srcResource)) {
  console.log(`Warning: ${source} does not exist, skipping`);
  continue;
}
```

这通常发生在前端尚未构建的情况下。

### 备份目标目录

如果目标目录已存在，将其重命名为 `.bak` 备份：

```js
const backupDest = destResource + '.bak';
if (fs.existsSync(destResource)) {
  // 清除可能残留的旧备份
  if (fs.existsSync(backupDest)) rm(backupDest);
  // 将当前目标重命名为备份
  fs.renameSync(destResource, backupDest);
}
```

如果存在上次操作残留的 `.bak` 备份，先清除再创建新备份。

### 复制源到目标

使用 `copyDirSync` 将源目录内容复制到目标路径：

```js
copyDirSync(srcResource, destResource);
```

`copyDirSync` 支持文件和目录两种类型：
- 文件 → 直接复制，自动创建目标目录
- 目录 → 递归复制所有子目录和文件

### 删除备份

复制成功后删除 `.bak` 备份：

```js
if (fs.existsSync(backupDest)) {
  rm(backupDest);
  console.log('[ee-bin] [move] Cleaned previous resources:', destResource);
}
```
</Steps>

<Warning>
如果复制过程中失败（如磁盘空间不足），`.bak` 备份会保留在磁盘上。下次执行 move 命令时，ee-bin 会先清除残留备份再重新操作。但请注意，在失败场景下目标目录可能不完整。
</Warning>

## 多资源配置

`move` 配置支持多个复制规则，每个规则独立执行：

```js
module.exports = {
  move: {
    frontend_dist: {
      src: './frontend/dist',
      dest: './public/dist',
    },
    custom_assets: {
      src: './assets',
      dest: './public/assets',
    },
    data_files: {
      src: './data',
      dest: './public/data',
      dist: './data/production',  // 覆盖 src
      target: './public/data/prod', // 覆盖 dest
    },
  },
};
```

执行时通过 `--flag` 指定要执行的规则：

```bash
# 执行所有 move 规则
ee-bin move --flag frontend_dist,custom_assets,data_files

# 只复制前端产物
ee-bin move --flag frontend_dist
```

## 命令选项

```bash
ee-bin move --flag frontend_dist
ee-bin move --flag frontend_dist,custom_assets
ee-bin move --config ./cmd/bin.custom.js --flag frontend_dist
```

| 选项 | 类型 | 说明 |
|------|------|------|
| `--config <folder>` | string | 自定义配置文件路径 |
| `--flag <flag>` | string | 逗号分隔的 move 配置键名 |

<Warning>
`--flag` 必须指定。未指定时 ee-bin 打印错误信息并退出。`--flag` 的值必须与 `move` 配置中的键名匹配，不存在的键名会被跳过并打印错误。
</Warning>

## 与构建流程的关系

move 命令是构建流程中的关键步骤，通常在前端构建之后执行：

### 标准构建流程

```
1. build frontend   → 生成 frontend/dist
2. move             → 复制 frontend/dist → public/dist
3. build electron   → 打包 Electron 代码
4. encrypt          → 代码加密（可选）
```

### 快捷命令

项目 `package.json` 中通常配置了组合命令：

```json
{
  "scripts": {
    "build-frontend": "ee-bin build --cmds frontend && ee-bin move --flag frontend_dist",
    "build": "pnpm build-frontend && pnpm build-electron && pnpm encrypt"
  }
}
```

`build-frontend` 命令包含了前端构建和资源移动两个步骤，确保 Electron 在打包时能找到前端产物。

### 为什么需要 move

Electron 应用需要将前端构建产物放在 `public/dist` 目录下，以便：
- dev 模式下前端开发服务器提供静态资源
- prod 模式下 Electron 直接从本地文件加载（`file://` 协议）

前端构建工具（Vite）默认输出到 `frontend/dist`，而 Electron 期望在 `public/dist` 找到资源。move 命令填补了这个路径差异。

## 默认配置

ee-bin 内置的默认 move 配置：

```js
move: {
  frontend_dist: {
    src: './frontend/dist',
    dest: './public/dist',
  },
}
```

用户可以在 `cmd/bin.js` 中覆盖此配置，添加新的复制规则，或修改路径。

## 执行日志

move 命令执行时会打印详细日志：

```bash
[ee-bin] [move] Start moving resources
[ee-bin] [move] Move flag: frontend_dist
[ee-bin] [move] config: { src: './frontend/dist', dest: './public/dist' }
[ee-bin] [move] Copy /path/to/frontend/dist to /path/to/public/dist
[ee-bin] [move] Cleaned previous resources: /path/to/public/dist
[ee-bin] [move] End
```

如果源目录不存在：

```bash
[ee-bin] [move] Warning: ./frontend/dist does not exist, skipping
```

如果 flag 对应的配置不存在：

```bash
[ee-bin] [move] Error: custom_assets config does not exist
```
