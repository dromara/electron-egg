ee-bin 是 electron-egg 项目的 CLI 构建工具，基于 Commander.js 框架实现，提供 9 个子命令覆盖开发、构建、加密、资源管理等全流程。bin 命令为 `ee-bin`，当前版本 v5.0.0-beta.3。

## 命令总览

| 命令 | 说明 | 主要选项 |
|------|------|----------|
| `dev` | 启动开发模式（前端 + Electron） | `--config`, `--serve` |
| `build` | 构建多种资源 | `--config`, `--cmds`, `--env` |
| `start` | 生产模式启动 | `--config` |
| `exec` | 执行自定义命令 | `--config`, `--cmds` |
| `move` | 移动/复制资源文件 | `--config`, `--flag` |
| `encrypt` | 代码加密（混淆/字节码） | `--config`, `--out` |
| `clean` | 清除加密产物 | `-d/--dir` |
| `icon` | 生成应用图标 | `-i`, `-o`, `-s`, `-c`, `-m` |
| `updater` | 生成增量更新包 | `--config`, `--asar-file`, `--platform`, `--force` |

不带子命令运行 `ee-bin` 时，会显示绿色帮助文本。

## 命令详解

### dev — 开发模式

启动前端开发服务器和 Electron 进程，支持文件监听与自动重建。

```bash
ee-bin dev
ee-bin dev --serve frontend
ee-bin dev --serve electron
ee-bin dev --serve frontend,electron
ee-bin dev --config ./cmd/bin.custom.js
```

| 选项 | 类型 | 说明 |
|------|------|------|
| `--config <folder>` | string | 自定义 bin.js 配置文件路径，默认 `./cmd/bin.js` |
| `--serve <mode>` | string | 逗号分隔的服务列表，如 `"frontend,electron"`；不指定则启动所有 dev 配置中的命令 |

### build — 构建模式

构建前端、打包 Electron、执行平台打包命令。

```bash
ee-bin build --cmds frontend,electron
ee-bin build --cmds frontend,electron,win64
ee-bin build --cmds frontend,electron,mac
ee-bin build --cmds frontend,electron,mac_arm64
ee-bin build --cmds frontend,electron,linux
ee-bin build --env dev --cmds electron
```

| 选项 | 类型 | 说明 |
|------|------|------|
| `--config <folder>` | string | 自定义配置文件路径 |
| `--cmds <flag>` | string | 逗号分隔的构建命令，如 `"frontend,electron,win64"` |
| `--env <env>` | string | Node 环境，默认 `prod`；可设为 `dev` 启用调试模式 |

<Note>
`--cmds` 必须指定，否则 ee-bin 会提示修改 bin 配置文件。`electron` 命令仅触发 esbuild 打包，不会启动 Electron 进程。
</Note>

### start — 生产启动

直接运行已构建的 Electron 应用。

```bash
ee-bin start
ee-bin start --config ./cmd/bin.custom.js
```

| 选项 | 类型 | 说明 |
|------|------|------|
| `--config <folder>` | string | 自定义配置文件路径 |

### exec — 自定义命令

执行 bin.js 中 `exec` 配置段定义的自定义命令。

```bash
ee-bin exec --cmds mytask
ee-bin exec --cmds task1,task2
```

| 选项 | 类型 | 说明 |
|------|------|------|
| `--config <folder>` | string | 自定义配置文件路径 |
| `--cmds <flag>` | string | 逗号分隔的自定义命令名 |

### move — 资源移动

按照 bin.js 中 `move` 配置段复制资源文件。

```bash
ee-bin move --flag frontend_dist
ee-bin move --flag frontend_dist,custom_assets
```

| 选项 | 类型 | 说明 |
|------|------|------|
| `--config <folder>` | string | 自定义配置文件路径 |
| `--flag <flag>` | string | 逗号分隔的 move 配置键名 |

### encrypt — 代码加密

对前端和 Electron 代码分别执行混淆或字节码编译。

```bash
ee-bin encrypt
ee-bin encrypt --config ./cmd/bin.custom.js
ee-bin encrypt --out ./release
```

| 选项 | 类型 | 说明 |
|------|------|------|
| `--config <folder>` | string | 自定义配置文件路径 |
| `--out <folder>` | string | 输出目录覆盖，默认取配置中的 `encryptDir` |

### clean — 清除加密产物

删除加密产生的临时文件和目录。

```bash
ee-bin clean
ee-bin clean -d ./public/electron
ee-bin clean -d ./public/electron -d ./public/dist
```

| 选项 | 类型 | 说明 |
|------|------|------|
| `-d, --dir <folder>` | string/array | 要清除的目录，默认 `./public/electron`；可多次指定 |

### icon — 图标生成

从源图片生成各平台所需的图标文件。

```bash
ee-bin icon
ee-bin icon -i ./public/images/logo.png
ee-bin icon -i ./logo.png -o ./build/icons/ -s 16,32,64,256,512 -c
```

| 选项 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `-i, --input <file>` | string | `/public/images/logo.png` | 源图片文件路径 |
| `-o, --output <folder>` | string | `/build/icons/` | 输出目录 |
| `-s, --size <flag>` | string | `16,32,64,256,512` | 逗号分隔的图标尺寸 |
| `-c, --clear` | boolean | false | 生成前清除输出目录 |
| `-m, --images <flag>` | string | `/public/images/` | 系统托盘/窗口图标输出路径 |

### updater — 增量更新

生成增量更新 zip 包及 JSON 元数据文件。

```bash
ee-bin updater --platform mac
ee-bin updater --platform win64
ee-bin updater --platform mac --asar-file ./dist/mac/app.asar --force true
```

| 选项 | 类型 | 说明 |
|------|------|------|
| `--config <folder>` | string | 自定义配置文件路径 |
| `--asar-file <file>` | string | asar 包路径，覆盖配置中的 `asarFile` |
| `--platform <flag>` | string | 目标平台（如 `mac`、`win64`、`linux`） |
| `--force <flag>` | string | 强制全量更新，设为 `"true"` 启用 |

## 常见命令组合

<Steps>
### 开发调试

```bash
# 启动前端 + Electron 开发
pnpm dev

# 仅启动 Electron（前端独立启动）
ee-bin dev --serve electron

# 调试模式（启用 DEBUG 日志）
cross-env DEBUG=ee-* ee-bin dev
```

### 构建打包

```bash
# 构建前端 + Electron
pnpm build-frontend && pnpm build-electron

# 或一步到位
pnpm build

# 打包 Windows 64 位安装包
ee-bin build --cmds frontend,electron,win64

# 打包 macOS ARM64
ee-bin build --cmds frontend,electron,mac_arm64
```

### 加密与清理

```bash
# 构建后加密
pnpm build && pnpm encrypt

# 清除加密产物重新构建
ee-bin clean -d ./public/electron && pnpm build-electron
```

### 增量更新

```bash
# 先打包，再生成增量更新包
ee-bin build --cmds frontend,electron,mac
ee-bin updater --platform mac
```
</Steps>

## 配置文件

所有命令的行为由 `./cmd/bin.js` 控制。ee-bin 先加载用户配置，再与内置默认配置（`bin_default.ts`）深度合并，用户值始终覆盖默认值。

```js
// cmd/bin.js 示例
module.exports = {
  dev: {
    frontend: {
      directory: './frontend',
      cmd: 'npm',
      args: ['run', 'dev'],
    },
    electron: {
      directory: './',
      cmd: 'electron',
      args: ['.', '--env=local'],
    },
  },
  build: {
    frontend: { directory: './frontend', cmd: 'npm', args: ['run', 'build'] },
    electron: { bundleType: 'bundle', external: [] },
  },
  move: {
    frontend_dist: { src: './frontend/dist', dest: './public/dist' },
  },
  encrypt: {
    frontend: { type: 'confusion', files: ['./public/dist/**/*.js'] },
    electron: { type: 'bytecode', files: ['./public/electron/**/*.js'] },
  },
};
```

配置文件支持 `.js`、`.json`、`.json5` 格式。`.js` 文件支持导出函数（工厂模式）或对象。
